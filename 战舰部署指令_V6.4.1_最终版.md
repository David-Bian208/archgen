## 🚀 部署步骤（严格按顺序）

### 1. 上传部署包到服务器

```bash
# 在本地执行
scp behavior_recorder_v641_final.tar.gz user@your-server:/path/to/

# 或在服务器上下载
wget [下载链接] -O behavior_recorder_v641_final.tar.gz
```

### 2. 解压到目标目录

```bash
cd /path/to/
tar -xzf behavior_recorder_v641_final.tar.gz
```

### 3. 配置环境变量

```bash
cd deploy_v641_final
cp .env.example .env
nano .env
```

**必须填入实际的 API Key：**
```env
LLM_API_KEY=sk-你的实际API密钥
LLM_BASE_URL=https://api.deepseek.com/v1
LLM_MODEL=deepseek-chat
```

### 4. 安装 Python 依赖

```bash
pip3 install -r requirements.txt
```

**关键依赖版本（不能错）：**
- `chainlit>=1.1.0,<2.0.0`（必须1.x，2.x有ContextVar bug）
- `pydantic<2.0.0`（必须1.x，2.x不兼容Chainlit 1.x）

### 5. 停止旧服务

```bash
# 强制停止3000端口的旧服务
fuser -k 3000/tcp 2>/dev/null
echo "旧服务已停止"
```

### 6. 清理 Python 缓存（必须）

```bash
find . -type d -name "__pycache__" -exec rm -rf {} +
find . -name "*.pyc" -delete
echo "缓存已清理"
```

### 7. 启动服务

```bash
cd chainlit_v62
bash start_chainlit.sh
```

或直接运行（使用3000端口）：
```bash
python3 -m chainlit run app.py --port 3000 --host 0.0.0.0
```

### 8. 验证服务

```bash
# 检查端口监听
netstat -tlnp | grep 3000

# 浏览器访问
# http://your-server-ip:3000
```

## 🔧 常见问题

### 问题1：端口被占用
```bash
fuser -k 3000/tcp
sleep 2
# 重新启动
```

### 问题2：背景是黑色
说明主题配置未生效，请检查：
```bash
cat chainlit_v62/config.yaml | grep default_theme
# 应显示：default_theme: light

cat chainlit_v62/.chainlit/config.toml | grep default_theme
# 应显示：default_theme = "light"
```
如不正确，说明文件未完整上传。

### 问题3：关键词数量不对
```bash
grep "total_keywords" app/knowledge/intent_keywords.json
# 应显示：283
```
如不是283，说明文件被修改过。

### 问题4：报告秒生成
说明LLM未调用，请检查：
1. `.env` 中 `LLM_API_KEY` 是否正确
2. 网络是否能访问 DeepSeek API
3. 启动日志是否有 LLM 调用记录
