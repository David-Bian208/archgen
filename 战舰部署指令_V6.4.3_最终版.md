```bash

cd /opt/behavior-recorder/behavior_recorder_service

# 停止 Chainlit 服务（3000端口）
fuser -k 3000/tcp 2>/dev/null

# 停止中台服务（8000端口，如果有）
```

### Step 3: 清理 Python 缓存

```bash
cd /opt/behavior-recorder/behavior_recorder_service

# 清理所有 Python 缓存
find . -type d -name '__pycache__' -exec rm -rf {} + 2>/dev/null
find . -name '*.pyc' -delete 2>/dev/null
```

### Step 4: 启动服务

```bash

# 确保 data 目录存在
mkdir -p data

# 启动 Chainlit（3000端口）
cd /opt/behavior-recorder/behavior_recorder_service/chainlit_v62
nohup python3 -m chainlit run app.py --port 3000 --host 0.0.0.0 > ../logs/chainlit.log 2>&1 &
echo "Chainlit started, PID: $!"

# 启动中台（8000端口）
cd /opt/behavior-recorder/behavior_recorder_service
nohup python3 admin_dashboard.py > logs/admin_dashboard.log 2>&1 &
echo "Admin Dashboard started, PID: $!"

```

### Step 5: 验证服务


### 数据库结构

```sql
-- conversations 表（对话记录）
CREATE TABLE conversations (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id TEXT,
    session_id TEXT,
    role TEXT,              -- 'user' 或 'assistant'
    content TEXT,
    timestamp DATETIME DEFAULT CURRENT_TIMESTAMP,
    response_time_seconds REAL,
    scene_type TEXT,        -- A/B/C/D/E
    hypothesis TEXT
);

-- sessions 表（会话追踪）
CREATE TABLE sessions (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    session_id TEXT UNIQUE,
    status TEXT DEFAULT 'in_progress',  -- in_progress / completed / error
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    completed_at DATETIME,
    error_message TEXT,
    total_turns INTEGER DEFAULT 0
);
```

##  注意事项

1. **数据库是共享的** - app.py 和 admin_dashboard.py 读写同一个 `data/conversations.db`
2. **Python 缓存** - 每次部署后务必清理 `__pycache__` 和 `.pyc` 文件
3. **防火墙** - 确保 3000 和 8000 端口在阿里云安全组中开放
4. **日志位置** - `logs/chainlit.log` 和 `logs/admin_dashboard.log`
5. **环境变量** - 确保 `.env` 文件中有正确的 `LLM_API_KEY`
