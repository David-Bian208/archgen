# 阿里云部署指南

**最后更新：** 2026-03-25  
**版本：** V4.10.6

---

## 📋 部署流程

### 1️⃣ 拉取最新代码

```bash
cd /opt/behavior-recorder/behavior_recorder_service
git pull origin clean-release
```

### 2️⃣ 安装依赖（首次部署）

```bash
pip install -r requirements.txt
cd frontend && npm install && cd ..
```

### 3️⃣ 配置环境变量

**方式 A：使用环境变量文件（推荐）**

```bash
# 加载环境变量
source .env.aliyun

# 启动前端（使用环境变量中的 API 地址）
cd frontend && npm run dev
```

**方式 B：直接修改 vite.config.js（临时）**

```bash
cd frontend
sed -i "s|http://localhost:8000|http://8.130.148.166:8000|g" vite.config.js
npm run dev
```

### 4️⃣ 启动服务

**使用自动启动脚本：**

```bash
# 加载环境变量
source .env.aliyun

# 启动所有服务
./start-all.sh
```

**或手动启动：**

```bash
# 启动后端（Gunicorn 4 worker）
cd /opt/behavior-recorder/behavior_recorder_service
nohup gunicorn -w 4 -k uvicorn.workers.UvicornWorker --bind 0.0.0.0:8000 main:app > server.log 2>&1 &

# 启动前端（配置 API 代理）
cd /opt/behavior-recorder/behavior_recorder_service/frontend
source ../.env.aliyun
nohup npm run dev -- --host 0.0.0.0 --port 3000 > ../frontend.log 2>&1 &
```

### 5️⃣ 验证服务

```bash
# 检查后端
curl http://localhost:8000/api/health

# 检查前端
curl http://localhost:3000

# 检查进程
ps aux | grep -E "gunicorn|vite" | grep -v grep
```

## 🔧 配置文件说明

### start-all.sh
- ✅ **自动检测路径** - 本地和阿里云通用
- ✅ **Gunicorn 并发** - 4 worker 支持多用户
- ✅ **健康检查** - 启动失败自动退出

### .env.aliyun
- `VITE_API_URL` - API 后端地址（公网 IP）
- `DEPLOY_DIR` - 部署目录
- `PORT` - 前端端口

### vite.config.js
- ✅ **支持环境变量** - `VITE_API_URL`
- ✅ **默认 localhost** - 本地开发无需配置
- ✅ **生产环境** - 通过环境变量覆盖

## 🚨 常见问题

### Q1: 前端无法访问
```bash
# 检查端口
netstat -tlnp | grep 3000

# 检查防火墙/安全组
# 阿里云控制台 → 安全组 → 入站规则 → 3000 端口是否开放
```

### Q2: API 超时
```bash
# 检查后端日志
tail -50 server.log

# 检查 DeepSeek API
curl -X POST http://localhost:8000/api/v4/chat \
  -H "Content-Type: application/json" \
  -d '{"session_id": "test", "user_input": "测试"}'
```

### Q3: 路径错误
```bash
# 确认当前目录
pwd

# start-all.sh 会自动检测路径，无需手动修改
```

## 📝 更新流程

**日常更新（代码修改后）：**

```bash
cd /opt/behavior-recorder/behavior_recorder_service
git pull origin clean-release
./stop-all.sh
./start-all.sh
```

**依赖更新（requirements.txt 修改后）：**

```bash
cd /opt/behavior-recorder/behavior_recorder_service
git pull origin clean-release
pip install -r requirements.txt
./stop-all.sh
./start-all.sh
```

## 🎯 核心原则

1. **代码通过 Git 同步** - 不直接在阿里云修改代码
2. **配置通过环境变量** - 环境差异用配置文件隔离
3. **脚本自动检测路径** - 本地和阿里云使用同一套脚本
4. **生产分支保持干净** - `clean-release` 分支仅包含生产必需文件
