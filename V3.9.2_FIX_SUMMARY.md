## 问题背景

在 V3.9.1 及之前版本中，引导式记录员 Agent 存在严重的"重复提问"问题：
- 用户已回答"任务太难"后，AI 仍追问时间点或是否寻求关注
- 缺乏对对话状态的记忆和理解
- 问题生成机械，无法基于已有信息智能决策

这是"行为记录员"Agent 能否被用户接受的**生死线问题**。

---

## 核心修复

### 1. 会话状态追踪 (`session_state`)

在 `ConversationSession` 中新增 `session_state` 字典：

```python
session_state: dict = {
    "evidence": [],  # 用户已提供的关键事实
    "hypothesis_confidence": {
        "逃避难度": 0.33,
        "寻求关注": 0.33,
        "感觉逃避": 0.33,
    },  # 动态更新的置信度
    "current_focus": None,  # 当前聚焦的假设
    "clarified_hypotheses": [],  # 已厘清的假设
}
```

### 2. 状态更新函数 (`_update_state`)

每轮用户回答后，调用 `_update_state(user_input)`：
- 使用 LLM 解析输入，提取新证据
- 动态更新各假设的置信度 (0-1)
- 当某假设置信度 > 0.7 时，标记为 `current_focus`

### 3. 决策函数 (`_decide_next_question`)

重构问题生成逻辑：

```python
def _decide_next_question(state):
    conf = state["hypothesis_confidence"]
    
    # 规则 1: 如果任一假设置信度 > 0.7，转向深度挖掘
    if max(conf.values()) > 0.7:
        top_hyp = max(conf, key=conf.get)
        state["current_focus"] = top_hyp
        return f"deep_dive_{top_hyp}"
    
    # 规则 2: 否则，生成鉴别性问题区分接近的假设
    else:
        # 找出置信度接近的两个假设
        return f"discriminate_{hyp1}_vs_{hyp2}"
```

### 4. 深度挖掘问题库 (`_get_deep_dive_question`)

为每个假设预置深度挖掘问题：

- **逃避难度**: "您觉得是任务中的哪一部分让他感到最难？是读题不理解，还是动手操作有困难？"
- **寻求关注**: "当您或老师注意到他时，他的行为会有什么变化吗？"
- **感觉逃避**: "当时周围环境有什么特别的吗？比如噪音、灯光？"

### 5. 强制总结与聚焦的 Prompt

新的 `CONVERSATION_SYSTEM_PROMPT` 强制要求 LLM：

1. **总结**: "基于我们目前的对话，我了解到..."
2. **决策**: "为了进一步明确，现在最需要厘清的一点是..."
3. **提问**: 提出唯一的、精准的问题

### 6. 删除硬编码循环逻辑

- 移除所有可能导致循环提问的硬编码
- 问题生成完全由状态机驱动
- 每轮对话先更新状态，再决策，最后生成问题
