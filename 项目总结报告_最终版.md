## 🎯 核心功能（最终版）

### 3000 端口 - 主应用（Chainlit）
1. **开场引导** - 用户友好界面，清晰使用指引
2. **语义判断** - 自动识别输入质量（完整/部分/模糊/无效）
3. **ABC 引导式对话** - A→B→C 结构化收集行为场景
4. **ABC 智能识别** - B 回答中包含 ≥2 个后果关键词时自动提取 C 信息
5. **5 步临床推理** - 场景识别 → 假设生成 → 证据检验 → 机制解释 → 干预策略
6. **可折叠报告** - 推理过程折叠，核心报告清晰展示
7. **完整对话记录** - 每一轮用户↔AI 对话都记录到中台数据库

### 8000 端口 - 中台管理（FastAPI）
1. **紫色顶栏界面** - 渐变紫色头部 + 统计卡片 + 表格列表 + 分页
2. **统计卡片** - 总对话数、今日对话、平均响应（秒）、用户数
3. **简洁表格** - 时间 | 场景 | 用户问题 | 助手回答 | 耗时
4. **搜索功能** - 按关键词搜索问题或回答内容
5. **场景筛选** - 按 A/B/C/D/E 场景类型筛选
6. **导出 CSV** - 一键导出所有对话数据
7. **详情查看** - 点击单元格可查看完整内容
8. **未生成回答处理** - 红色高亮显示"未生成回答"

### 数据库结构
```sql
-- conversations 表（对话记录）
conversations (id, user_id, session_id, role, content, timestamp, response_time_seconds, scene_type, hypothesis)

-- sessions 表（会话追踪）
sessions (id, session_id, status, created_at, completed_at, error_message, total_turns)
```

## ⚠️ 重要教训

### 部署相关
1. **版本号统一** - 代码和文档必须一致，部署前检查
2. **部署文件完整性** - 列出所有依赖文件，不能只列主文件
3. **Python 缓存清理** - 每次部署后必须清理 `__pycache__` 和 `.pyc`
4. **SSH 密钥可能失效** - 服务器重建或 authorized_keys 被清掉后会断开，需要通过阿里云控制台重新添加
5. **用户名可能变化** - 服务器可能没有 admin 用户，需要用 root 登录

### 开发相关
1. **文本 > 大脑** - 所有重要决策必须写入文件
2. **安全 > 完成** - 临床系统优先保证输出安全
3. **用户用自然语言** - 系统必须理解自然语言，不能要求用户适应术语
4. **分析正确 ≠ 开对药** - 诊断与干预必须严格匹配
5. **界面方向一致** - 开发前确认界面设计方向，避免不同版本并行开发
