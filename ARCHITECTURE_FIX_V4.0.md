## 🔧 核心修改

### 1. 引导提问逻辑修复

**文件:** `app/agents/guided_recorder_agent_v3_v4.py`

**修改前 (V3.0):**
```python
CONVERSATION_SYSTEM_PROMPT = """
【后续追问话术参考】
- 了解情境："孩子是在什么活动或什么情况下出现这个情况的呢？"
- 了解行为："当时你观察到孩子具体做了什么..."
"""
```

**修改后 (V4.0):**
```python
CONVERSATION_SYSTEM_PROMPT = """
【V4.0 动态追问逻辑】
根据用户已描述的情境和行为，生成个性化的鉴别性问题：

**如果用户已描述"做操/体操/集体活动" + "发呆/走神"：**
→ 追问："在做操时，其他小朋友是都在认真做，还是也在走神？他的眼神是看起来在迷茫寻找提示，还是完全放空？"
→ 目的：鉴别提示依赖 vs 自我刺激

**如果用户已描述"学习/任务" + "抗拒/不要/不肯"：**
→ 追问："在学习时，是任务本身有难度（比如他不会做），还是他学了一会儿后才开始抗拒？抗拒时他会观察你的反应吗？"
→ 目的：鉴别逃避难度 vs 寻求关注
"""
```

**关键改进:**
- ✅ 追问必须引用用户已提到的具体活动（如"做操"、"学习"）
- ✅ 追问直接服务于行为功能鉴别
- ✅ 禁止通用问题（如"在什么活动中"）

### 2. 干预计划生成器重写

**文件:** `app/agents/intervention_planner_v4.py`

**核心输入 (V4.0 新增):**
```python
session_context = {
    "environment": session.environment,
    "context": session.context,
    "child_behavior": session.child_behavior,
    "others_response": session.others_response,
    # V4.0 新增：传递专家分析结果
    "primary_hypothesis": expert.get("functional_hypothesis", "提示依赖"),
    "capability_gap": expert.get("capability_gap", "工作记忆中的视觉 - 动作序列保持"),
    "behavior_pattern": expert.get("behavior_pattern", ""),
}
```

**动态生成逻辑:**

| 组件 | 生成逻辑 | 示例 |
|------|----------|------|
| **计划名称** | 基于活动 + 行为动态生成 | "针对做操中发呆的'建立锚点'计划" vs "针对学习启动困难的'情绪温度计'计划" |
| **核心思路** | 基于能力缺口生成 | 提示依赖→"外部提示转内部提示"；逃避难度→"分解步骤降低门槛" |
| **成功画面** | 基于行为基线生成 | 发呆（高频）→"降低频率至 XX%"；学习抗拒→"成功启动并完成短时任务" |
| **第一步行动** | 基于具体策略生成 | 做操发呆→"共创'大鹏展翅'暗号"；学习抗拒→"设计 3 分钟'火箭启动倒计时'" |

---

### 3. 知识库使用方式变更

**修改前:**
```
知识库 → 直接返回静态模板 → 用户
```

**修改后:**
```
知识库 → 提取干预原则 → 结合具体案例实例化 → 用户
```

**示例:**
- 知识库原则："通过创建'内化提示'搭建从外部依赖到内部执行的桥梁"
- 做操案例实例化："从做操动作序列中选择'大鹏展翅'作为暗号"
- 学习案例实例化："从学习任务中选择'火箭倒计时'作为启动游戏"

## 📊 技术架构

### V4.0 数据流
```
用户输入 → 情境提取 → 动态追问生成 → 功能鉴别 → 
专家分析 → 能力缺口识别 → 个性化计划生成 → 用户
```

### 关键函数
- `_evaluate_conversation()`: 情境感知追问生成
- `_generate_intervention_plan()`: 传递完整分析结论
- `_generate_four_step_plan_v4()`: 动态四步结构生成
- `_generate_phase_name_v4()`: 个性化计划命名
