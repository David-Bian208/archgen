# 行为观察助手 V6.4.2 中台修复部署指令

**部署时间：** 2026-05-06  
**版本：** V6.4.2（中台修复版）  
**核心修复：** 用户提问后关闭页面，中台也能看到记录

---

##  本次修复内容

| 项目 | 修复 | 说明 |
|------|------|------|
| **app.py** | ✅ 新增 | `on_message` 函数开头立即记录用户输入 |
| **admin_dashboard.py** | ✅ 修复 | 统计数据改为以 `role='user'` 为准 |

---

##  问题原因

之前用户输入"你好"或"我的孩子脾气不好"后关闭页面，中台看不到这些记录的原因是：

1. **app.py 旧版本** 没有在 `on_message` 开头记录用户输入
2. **admin_dashboard.py** 统计数据只计算 `role='assistant'` 的记录

---

##  部署步骤

### Step 1: 上传文件到服务器

需要上传以下 2 个文件到服务器 `/opt/behavior-recorder/behavior_recorder_service/`：

| 文件 | 本地路径 | 说明 |
|------|----------|------|
| `app.py` | `~/behavior_recorder_service/chainlit_v62/app.py` | 新增用户输入记录功能 |
| `admin_dashboard.py` | `~/behavior_recorder_service/admin_dashboard.py` | 修复统计数据计算逻辑 |

### Step 2: 停止旧服务

```bash
# 停止 Chainlit 服务（3000端口）
fuser -k 3000/tcp 2>/dev/null

# 停止中台服务（8000端口）
fuser -k 8000/tcp 2>/dev/null

sleep 2
```

### Step 3: 清理 Python 缓存

```bash
cd /opt/behavior-recorder/behavior_recorder_service

# 清理所有 Python 缓存
find . -type d -name '__pycache__' -exec rm -rf {} + 2>/dev/null
find . -name '*.pyc' -delete 2>/dev/null
```

### Step 4: 启动服务

```bash
# 启动 Chainlit（3000端口）
cd /opt/behavior-recorder/behavior_recorder_service/chainlit_v62
nohup chainlit run app.py --port 3000 --host 0.0.0.0 > ../logs/chainlit.log 2>&1 &
echo "Chainlit started, PID: $!"

sleep 5

# 启动中台（8000端口）
cd /opt/behavior-recorder/behavior_recorder_service
nohup python3 admin_dashboard.py > logs/admin_dashboard.log 2>&1 &
echo "Admin Dashboard started, PID: $!"
```

### Step 5: 验证服务

```bash
# 检查3000端口
curl -s http://localhost:3000/ | head -3

# 检查8000端口
curl -s http://localhost:8000/api/stats
```
