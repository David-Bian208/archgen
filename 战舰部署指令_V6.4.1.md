## 🚀 部署步骤

### 1. 上传部署包

```bash
# 方式A：使用 scp 上传
scp -r deploy_v641_final/* user@your-server:/path/to/behavior-recorder/

# 方式B：上传压缩包后解压
scp behavior_recorder_v641_final.tar.gz user@your-server:/path/to/
# 服务器上执行：
cd /path/to/
tar -xzf behavior_recorder_v641_final.tar.gz
```

### 2. 配置环境变量

```bash
cd /path/to/behavior-recorder/
cp .env.example .env
nano .env
```

**填入实际的 API Key：**
```env
LLM_API_KEY=sk-你的实际API密钥
LLM_BASE_URL=https://api.deepseek.com/v1
LLM_MODEL=deepseek-chat
```

### 3. 安装依赖

```bash
pip3 install -r requirements.txt
```

**关键依赖版本：**
- chainlit >=1.1.0,<2.0.0（必须1.x，2.x有ContextVar bug）
- pydantic <2.0.0（必须1.x，2.x不兼容Chainlit 1.x）

### 4. 清理旧服务

```bash
# 停止旧服务
fuser -k 8005/tcp 2>/dev/null

# 清理 Python 缓存
find . -type d -name "__pycache__" -exec rm -rf {} +
find . -name "*.pyc" -delete
```

### 5. 启动服务

```bash
cd chainlit_v62
python3 -m chainlit run app.py --port 8005 --host 0.0.0.0
```

或使用启动脚本：
```bash
cd chainlit_v62
bash start_chainlit.sh
```
（注意：脚本默认端口是8002，如需8005请修改脚本）

### 6. 验证服务

```bash
# 检查端口是否监听
netstat -tlnp | grep 8005

# 检查进程
ps aux | grep chainlit

# 浏览器访问
# http://your-server-ip:8005
```

##  常见问题

### 问题1：服务启动失败
```bash
# 检查端口占用
fuser -k 8005/tcp

# 清理缓存
find . -type d -name "__pycache__" -exec rm -rf {} +
find . -name "*.pyc" -delete
```

### 问题2：关键词数量不对
检查文件是否完整：
```bash
# 查看假设数量
grep -c '"id":' app/knowledge/hypothesis_network.json
# 应返回：18

# 查看关键词总数
grep "total_keywords" app/knowledge/intent_keywords.json
# 应返回：283
```

### 问题3：UI显示黑色背景
```bash
# 检查配置
cat chainlit_v62/config.yaml | grep default_theme
# 应看到：default_theme: light

cat chainlit_v62/.chainlit/config.toml | grep default
# 应看到：default = "light"
```

### 问题4：报告秒生成（模板化）
这说明LLM调用未生效，请检查：
1. `.env` 文件中 `LLM_API_KEY` 是否正确
2. 网络连接是否正常
3. 日志中是否有LLM调用记录
