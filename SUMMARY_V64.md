## 版本信息
- **当前版本**: V6.4（基于 V6.3 迭代）
- **运行端口**: http://localhost:8014（8001 为 FastAPI 后端，保留不动）
- **技术栈**: Chainlit 2.11.1 + 临床推理引擎 V6.2

## 一、V6.4 新增/修改项

### 1. 开场白精简
- 标题从"行为观察助手 V6.3"改为"行为观察助手"
- 文案从技术化改为用户友好型：
  - "您好！我是专注于儿童行为支持的观察助手。"
  - 列出 3 点核心能力（分步剖析、探讨原因、提供思路）
  - 底部备注"📷 注：当前版本暂不支持图片上传"

### 2. 报告正文优化
- **恢复"场景理解"区块**（之前删除了），但去掉了类型代码（如"A类"），只显示场景名称和核心挑战
- **"可能的解释" → "行为假设"**（标题改名）
- **删除"专业解释："前缀**（假设内容中去除冗余前缀）
- **"假设H1/H2/H3" → "1./2./3."**（编号简化）

### 3. ABC 引导流程优化
- **ABC 教育话术大幅精简**（从 ~25 行缩到 ~10 行），适配手机端阅读
- **修复引导逻辑 Bug**：用户第一次输入触发了 ABC 引导后，如果在引导过程中一次性回复完整内容（>80字），之前会继续引导。现在加了智能判断，检测到长回复直接跳过引导进入推理
- 保留了弹性上限（5 轮）

### 4. UI 交互优化
- **推理过程可折叠**：使用 `<details>` HTML 标签，推理详情默认折叠在"🧠 推理过程（点击查看）"下拉框中，用户点击可展开查看，灰色小字（`color: #888; font-size: 0.82em`）
- **推理进度流式展示**：推理过程中显示 "🤔 正在分析中..." + "Step 1~5" 进度提示
- **隐藏文件上传按钮**：`config.toml` 中 `enabled = false`
- **隐藏右上角"说明"和"调节颜色"按钮**：通过 `custom.css` + `custom.js` 实现（JS 定时扫描 + MutationObserver 监听 DOM 变化）
- **浅色主题默认**：`.chainlit/config.toml` 中 `default_theme = "light"`

### 5. Chainlit 2.x 配置文件修复
- 发现 `config.yaml` 不生效，Chainlit 2.x 实际读取的是 `.chainlit/config.toml`
- 开启了 `unsafe_allow_html = true`（用于 `<details>` 折叠功能）
- 配置了 `custom_css = "/public/custom.css"` 和 `custom_js = "/public/custom.js"`

## 四、V6.4 完整交互流程

1. 用户打开页面 → 看到精简版开场白（白色背景）
2. 用户输入行为描述 → 语义判断：
   - **完整输入**：直接推理（显示进度 + 折叠的推理过程 + 正文报告）
   - **部分输入**：启动 ABC 引导（精简版话术）
   - **模糊输入**（如"孩子一直有情绪"）：ABC 教育 + 引导
   - **无效输入**（问候语/太短）：直接拒绝
3. 引导过程中用户一次性给完整内容（>80字）→ 自动跳过引导，直接推理
4. 推理过程：进度流式展示（Step 1~5）→ 完成后折叠显示 + 正文报告自动展示
5. 正文报告结构：场景理解 → ABC 分析 → 行为假设 → 深层机制 → 安全提示 → 建议策略 → 免责声明
