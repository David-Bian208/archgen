**测试场景：** 4.5 岁女孩，迪士尼与玩伴争抢队长，拥抱不知轻重

**关键洞察：** 系统像"能准确读验血报告但会乱开药方的机器人医生"——诊断对，处方错。

### 3. V4.5.20 P0 修复：临床逻辑断裂修复

**根因：** 场景选择逻辑只有 2 分支（逃避难度/提示依赖），没有社交技能不足分支

**核心修复：**
```python
# 能力缺口优先级高于功能判断！
if capability_gap and any(kw in capability_gap for kw in ["社交", "换位思考", "灵活性", "情绪识别"]):
    scenario_key = "social_skills_deficit"
    hypothesis_id = "H_SOCIAL_SKILLS"

# 即使 functional_judgment 是"寻求关注"，也二次检查能力缺口
elif functional_judgment and "关注" in functional_judgment:
    if capability_gap and "社交" in capability_gap:
        scenario_key = "social_skills_deficit"  # 覆盖！
```

**新增干预模板：** `_generate_social_skills_plan()`
- 核心原则：社交技能是"教"出来的，不是"管"出来的
- 策略：社交故事 + 角色扮演 + 视觉提示 + 现场辅助
- 成功标准：孩子主动使用社交策略，而非问题行为减少

## 关键学习

### 技术层面
1. **能力缺口 > 功能判断**：capability_gap 比 functional_judgment 更可靠，因为前者直接来自 InsightAnalyzer 的临床分析
2. **阶段优先于推理引擎**：工作流阶段未完成时，推理引擎不应介入提问
3. **会话 ID 持久化**：`_create_session` 必须支持传入 session_id 参数

### 临床安全层面
1. **诊断 - 干预一致性是底线**：诊断是"能力缺失"，干预必须是"技能教学"，不能是"行为管理"
2. **社交技能不足的干预关键词**：教学、示范、练习、社交故事、角色扮演——而非关注、忽略、强化
3. **潜在危害**：错误干预可能让孩子因社交尝试屡屡受挫而更加困惑、焦虑，甚至引发社交回避

### 用户体验层面
1. **无效追问损害信任**：用户已提供详细信息还机械追问，会让用户感到困惑和不被理解
2. **共情先于纠正**：用户表达困难时，先认可感受，再换方式问
3. **信息充分性检测**：80% 必填字段完成即可生成报告，不必 100%

## 反思

这次修复让我深刻认识到：
1. **临床安全是底线**：P0 问题不是 bug，是潜在的安全风险。诊断对、处方错，比完全不诊断更危险。
2. **能力缺口比行为标签更可靠**：孩子"争抢队长"是行为标签，"换位思考困难"是能力缺口。干预应该针对能力缺口，而非行为标签。
3. **专家反馈的价值**：没有专家的详细评估，我可能只修复了表面的对话僵化问题，而错过了最危险的临床逻辑断裂。
