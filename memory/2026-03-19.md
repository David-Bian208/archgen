# 2026-03-19 能力缺口分析编号层级修复

## 问题诊断

用户反馈：报告中"二、多角度理解与能力分析"部分的编号显示错误

**实际显示：**
```
1、鉴别与排除：...
1、核心假设：...
1、能力缺口分析：
   4. 认知灵活性不足：...
   5. 情绪调节困难：...
```

**期望显示：**
```
1. 鉴别与排除：...
2. 核心假设：...
3. 能力缺口分析：
   • 认知灵活性不足：...
   • 情绪调节困难：...
```

## 根本原因

1. **LLM 忽略 Prompt 要求**：尽管 Prompt 明确要求使用 1. 2. 3. 编号，LLM 仍可能生成错误的编号（全部用"1."）
2. **嵌套列表渲染问题**：前端 Markdown 解析器将嵌套的有序列表渲染为连续编号（4. 5.）

## 修复方案（双重保障机制）

### 第一层：Prompt 优化

在 `ANALYSIS_SYSTEM_PROMPT` 中明确编号层级规则：
- 主标题编号：1. 鉴别与排除 → 2. 核心假设 → 3. 能力缺口分析
- 子列表编号：能力缺口分析内部必须用 • 符号
- 提供详细的格式示例

### 第二层：后处理修复（核心）

新增 `_fix_capability_gap_format()` 方法：

**修复逻辑：**
1. **主标题编号修复**：检测三个主标题（鉴别与排除、核心假设、能力缺口分析），强制使用 1. 2. 3. 编号
2. **子列表编号修复**：在"能力缺口分析"部分，将有序列表（1. 2. 3.）强制转换为无序列表（•）

**代码实现：**
```python
def _fix_capability_gap_format(self, result: dict) -> dict:
    clinical_diff = result.get("clinical_differential", "")
    
    # 修复 1: 主标题编号
    titles = ["鉴别与排除", "核心假设", "能力缺口分析"]
    lines = clinical_diff.split('\n')
    fixed_lines = []
    title_index = 0
    
    for line in lines:
        title_match = re.match(r'^(\d+)[.、]\s*\*\*([^*]+)\*\*', line)
        if title_match:
            title_text = title_match.group(2).strip()
            if title_text in titles and title_index < len(titles):
                correct_num = title_index + 1
                fixed_line = re.sub(r'^\d+[.、]', f'{correct_num}.', line)
                title_index += 1
        fixed_lines.append(fixed_line)
    
    clinical_diff = '\n'.join(fixed_lines)
    
    # 修复 2: 能力缺口分析的子列表
    if "能力缺口分析" in clinical_diff:
        parts = clinical_diff.split("能力缺口分析")
        prefix = parts[0] + "能力缺口分析"
        suffix = parts[1]
        # 将有序列表转换为无序列表
        suffix_fixed = re.sub(r'\n\s+(\d+)[.、]\s*(\*\*[^\*]+\*\*)', r'\n   • \2', suffix)
        clinical_diff = prefix + suffix_fixed
    
    result["clinical_differential"] = clinical_diff
    return result
```

## 核心教训

1. **LLM 输出不可靠**：即使 Prompt 明确要求，LLM 仍可能生成错误格式
2. **防御性编程必要**：后处理是确保格式正确的关键保障
3. **双重保障机制**：Prompt 优化 + 后处理修复，缺一不可
4. **调试日志重要**：添加修复前后的日志对比，快速定位问题
