## 🔧 部署步骤

### Step 1：停止旧服务

```bash
# 停止 3000 端口的 Chainlit 服务
fuser -k 3000/tcp 2>/dev/null
sleep 2

# 停止 8000 端口的中台服务（如果存在）
fuser -k 8000/tcp 2>/dev/null
sleep 2
```

### Step 2：上传文件

将以下文件上传到服务器 `~/behavior_recorder_service/chainlit_v62/` 目录：

| 文件 | 说明 |
|------|------|
| `app.py` | V6.4.2 主应用（ABC 智能引导 + 数据记录） |

将以下文件上传到服务器 `~/behavior_recorder_service/` 目录：

| 文件 | 说明 |
|------|------|
| `admin_dashboard.py` | 中台管理服务（端口 8000） |

### Step 3：清理 Python 缓存

```bash
cd ~/behavior_recorder_service
find . -type d -name '__pycache__' -exec rm -rf {} + 2>/dev/null
find . -name '*.pyc' -delete 2>/dev/null
```

### Step 4：启动中台服务（端口 8000）

```bash
cd ~/behavior_recorder_service
nohup python admin_dashboard.py > logs/admin_dashboard.log 2>&1 &
echo "中台服务 PID: $!"

# 验证
sleep 3
curl http://localhost:8000/api/stats
```

**预期输出：**
```json
{"total": 0, "today": 0, "avg_time": 0, "users": 0}
```

### Step 5：启动 Chainlit 服务（端口 3000）

```bash
cd ~/behavior_recorder_service/chainlit_v62
nohup chainlit run app.py --port 3000 > logs/chainlit.log 2>&1 &
echo "Chainlit 服务 PID: $!"

# 验证
sleep 5
curl http://localhost:3000/health
```

### Step 6：验证服务

```bash
# 检查端口占用
lsof -i:3000
lsof -i:8000

# 查看日志
tail -20 logs/chainlit.log
tail -20 logs/admin_dashboard.log
```

## 🎯 功能验证

### 1. ABC 引导智能识别测试

**测试场景：** 用户在回答 B 时已经包含了 C 的信息

**输入示例：**
```
A: 在一起上课的时候，他分不清严厉还是凶。然后会很生气
B: 我一般两种处理方式，第一种是我退让一步，然后他会寻求安慰，想希望我安慰她，还有一种方式，我不退让，他就会生气发脾气，但发脾气的过程当中会克制，说我不想当好孩子啦，我想发脾气啦
```

**预期行为：** 系统识别到 B 中包含"处理方式"、"退让"、"安慰"、"发脾气"等关键词（≥2个），自动提取 C 信息，直接进入推理，不再追问。

### 2. 中台管理测试

1. 访问 `http://服务器IP:8000` 打开中台首页
2. 在 3000 端口的主应用中提交一个行为观察
3. 刷新中台页面，应该能看到新的对话记录
4. 点击用户问题/助手回答可以查看完整内容
5. 点击"导出 CSV"可以下载数据

## 📊 中台功能说明

### 统计数据
- 总对话数
- 今日对话数
- 平均响应时间（秒）
- 用户数（不同 session）

### 数据列表
- 时间、场景类型、用户问题、助手回答、耗时
- 支持搜索（问题/回答内容）
- 支持按场景类型筛选（A/B/C/D/E）
- 分页显示（每页 20 条）

### 数据导出
- 导出为 CSV 格式
- 包含：时间、场景类型、假设、用户问题、助手回答、响应时间
