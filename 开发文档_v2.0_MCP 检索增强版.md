## 🎯 新定位与目标

### ArchGen v2.0 定位

**一句话定义：**
> 本地运行的 Web App：前端浏览器界面 → 连本机后端服务 → 后端用 MCP 式文件直读处理逻辑 → 调 LLM 返回结果

**核心价值：**
- **输入端**：本地文件夹（AI-Pulse 产出的 MD 精选文件）
- **处理端**：MCP 文件直读 + LLM 调用（零向量、零切块、零 embed）
- **输出端**：总结/整理稿/口播稿/任意 Skill
- **壳**：localhost 后端 + 浏览器前端（看起来像网页，实际是本机软件）

### 与 AI-Pulse 的关系

```
┌─────────────────┐         ┌─────────────────┐
│   AI-Pulse      │         │    ArchGen      │
│   (采集侧)       │         │   (消费侧)       │
│                 │         │                 │
│  抓取 AI 资讯     │   MD    │  扫描文件夹      │
│  → 精选评分      │ ──────▶ │  → MCP 检索      │
│  → 存储为 MD     │  文件    │  → LLM 总结      │
│                 │         │  → 返回结果      │
└─────────────────┘         └─────────────────┘
     独立软件                     独立软件
     通过文件夹松耦合
```

**关键：两个软件完全独立，只通过文件夹耦合。**

### 模块 2：添加 MCP 检索接口

**文件：** `api.py`

**新增接口：**
```python
@router.post("/api/mcp/search")
async def mcp_search(request: Dict = Body(...)):
    """
    MCP 检索：扫描→过滤→读取→LLM 总结
    
    请求体：
    {
      "folder_path": "/path/to/knowledge_base",
      "topic": "Agent",
      "tag": "AI"  // 可选
    }
    
    返回：
    {
      "summary": "LLM 生成的总结（Markdown 格式）",
      "source_files": ["agent_intro.md", "agent_workflow.md"],
      "file_count": 5
    }
    """
    # Step 1: 扫描 + 过滤
    reader = LocalFolderReader(folder_path)
    files = reader.list_files(keyword=topic, tag=tag)
    
    if not files:
        return _error(code=404, msg="没找到相关文件")
    
    # Step 2: 读取全文（限制最多 5 个文件）
    target_paths = [f["path"] for f in files[:5]]
    contents = reader.read_files(target_paths)
    
    # Step 3: 拼接上下文
    context = "\n---\n".join([
        f"## {Path(c['path']).name}\n{c['content']}"
        for c in contents
    ])
    
    # Step 4: 调用 LLM 总结
    llm_config = _get_config("llm")
    prompt = f"""基于以下资料，整理一份关于「{topic}」的综合摘要：

{context}

要求：
1. 结构化输出（分点/分层级）
2. 引用原文时标注文件名
3. 不要编造资料中没有的内容
"""
    
    # 调用 DeepSeek API
    import httpx
    response = httpx.post(
        f"{llm_config['base_url']}/chat/completions",
        headers={"Authorization": f"Bearer {llm_config['api_key']}"},
        json={
            "model": llm_config["model"],
            "messages": [{"role": "user", "content": prompt}],
            "max_tokens": 4096,
        },
        timeout=llm_config["timeout"]
    )
    
    result = response.json()
    summary = result["choices"][0]["message"]["content"]
    
    return _success({
        "summary": summary,
        "source_files": [f["name"] for f in files[:5]],
        "file_count": len(files),
    })
```

### 模块 3：前端对话式 UI

**文件：** `frontend/src/views/MCPSearch.vue`（新建）

**UI 设计：**
```vue
<template>
  <div class="mcp-search">
    <!-- 顶部：搜索框 -->
    <div class="search-bar">
      <a-input-search
        v-model="topic"
        placeholder="帮我整理 Agent 相关内容..."
        size="large"
        @search="handleSearch"
      />
    </div>
    
    <!-- 左侧：文件列表（可选） -->
    <a-layout>
      <a-layout-sider width="300">
        <a-list :data="files">
          <template #item="{ item }">
            <a-list-item>
              {{ item.name }}
              <a-tag v-for="tag in item.tags" :key="tag">{{ tag }}</a-tag>
            </a-list-item>
          </template>
        </a-list>
      </a-layout-sider>
      
      <!-- 右侧：结果展示 -->
      <a-layout-content>
        <a-spin :loading="loading">
          <div v-if="result" class="result">
            <h3>📄 来源文件</h3>
            <a-tag v-for="f in result.source_files" :key="f">{{ f }}</a-tag>
            
            <h3>📝 总结报告</h3>
            <div class="markdown-body" v-html="renderMarkdown(result.summary)" />
          </div>
        </a-spin>
      </a-layout-content>
    </a-layout>
  </div>
</template>

<script setup>
import { ref } from 'vue'
import { message } from '@arco-design/web-vue'
import { marked } from 'marked'  // npm install marked

const topic = ref('')
const loading = ref(false)
const result = ref(null)
const files = ref([])

const handleSearch = async () => {
  loading.value = true
  try {
    const res = await fetch('/api/mcp/search', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        folder_path: '/path/to/knowledge_base',
        topic: topic.value,
      }),
    })
    const data = await res.json()
    if (data.code === 0) {
      result.value = data.data
    } else {
      message.error(data.msg)
    }
  } catch (e) {
    message.error('搜索失败：' + e.message)
  } finally {
    loading.value = false
  }
}

const renderMarkdown = (text) => {
  return marked.parse(text)
}
</script>

<style scoped>

}

}

  line-height: 1.8;
  font-size: 15px;
}
</style>
```

## 📝 附录：MCP vs 向量知识库对比

| 维度 | MCP（文件直读） | 向量知识库 |
|------|----------------|-----------|
| **原理** | 直接读取文件全文 | 向量化后模糊召回 |
| **保真度** | 高（完整逻辑骨架） | 低（切碎后拼图） |
| **适用场景** | 少而精的结构化文件 | 海量非结构化文本 |
| **更新成本** | 零（文件改了下次就读新的） | 高（需重新向量化） |
| **复杂度** | 低（文件系统操作） | 高（向量数据库 +embed） |
| **ArchGen 选择** | ✅ **采用** | ❌ **不采用** |

**决策理由：**
- AI-Pulse 产出的是精选 L2 内容（少而精）
- 文件带 Front Matter 元数据（易过滤）
- 需要保真度（完整逻辑，不是碎末）
- MCP 更符合"操作系统"感觉（文件是真实实体）
