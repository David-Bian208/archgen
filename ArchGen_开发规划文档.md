## 项目定位

AI 写作全流程工作台——从选题到成文的工业化流水线。

**核心价值：**
- **素材高效利用** → MCP 知识库 + AI-Pulse 信源 + 联网搜索 + 用户输入四源合一
- **结构化写作** → 方向分析 → 框架匹配 → 槽位填充 → 提纲整合 → 文章生成
- **透明可追查** → AI 思考日志 + 素材来源标记 + 降级链提示
- **人设一致** → 六维身份模型控制 AI 输出的风格和观点

**非目标：**
- ~~不做社交/用户体系/复杂推荐~~
- ~~不做通用聊天~~
- ~~不做手动设计 UI（照搬已定设计）~~

- [x] 工作手册已更新（v2.0）
- [x] 接口层不再需要改动

### 下一步：Phase 1 — 内部执行精炼期

**目标：** 按 v3.1 定义精炼每个步骤的内部执行逻辑，覆盖 Trae 可直接编码的粒度

**优先级：**
1. **Step 3 槽位填充** — 补全缺失的 `ask_followup` 和 `analyze` 的对话历史逻辑
2. **Step 5 文章生成** — 润色阶段添加 source_tag 约束校验
3. **补充系统** — 前端降级提示的交互细节（用户点"不满意"→降级链触发→UI 变化）
4. **前端状态管理** — 各 composable 内部状态转换规则

### Phase 1.5 — 稳定性加固期

- **LLM 调用统一入口**（P0 任务，待实施）
- **全链路 E2E 测试**（mock LLM 输出）
- **Session 持久化**（重启不丢 session）
- **限流保护**（防止 LLM 超预算）

### 后续 P1-P2

- **P1**：原文缓存、订阅功能、导出功能（AI-Pulse 周边）
- **P2**：4 阶段独立 LLM Pipeline、source_tag 硬约束、知识库同步
