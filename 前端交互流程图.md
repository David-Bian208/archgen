## 二、关键交互状态

### 2.1 按钮状态机

```
┌──────────────┐
│  初始状态     │
│  (可点击)     │
└──────┬───────┘
       │ 用户点击
       ↓
┌──────────────┐
│  Loading 状态 │
│  (禁用 + 转圈) │
└──────┬───────┘
       │ API 返回
       ↓
┌──────────────┐
│  完成状态     │
│  (可点击)     │
└──────────────┘
```

**状态说明：**

| 状态 | 按钮文本 | 禁用 | Loading | 说明 |
|------|---------|------|---------|------|
| 初始 | "AI 智能补充全部问题" | ❌ | ❌ | 可点击 |
| Loading | "补充中..." | ✅ | ✅ | 禁用 + 转圈 |
| 完成 | "AI 智能补充全部问题" | ❌ | ❌ | 恢复可点击 |

**异常处理：**
- API 超时（>15 秒）：按钮恢复，显示错误提示
- API 失败：按钮恢复，显示错误提示
- 网络错误：按钮恢复，显示"网络错误，请重试"

## 四、弹窗状态管理

### 4.1 补充弹窗状态机

```
┌──────────────┐
│  closed      │ ← 初始状态
└──────┬───────┘
       │ 用户点击"AI 智能补充"
       ↓
┌──────────────┐
│  loading     │ ← 显示 loading
└──────┬───────┘
       │ API 返回
       ↓
┌──────────────┐
│  showing     │ ← 显示补充结果
└──────┬───────┘
       │ 用户操作
       ├─ 确认 → saved → 关闭弹窗 → 跳转 Step 6
       ├─ 跳过 → skipped → 关闭弹窗 → 跳转 Step 6
       └─ 重新补充 → loading → (循环)
```

### 4.2 状态变量定义

```vue
<script setup>
const supplementState = ref('closed') // closed | loading | showing
const supplementResults = ref([]) // 补充结果数组
const degradationCount = ref(0) // 降级次数
const currentStep = ref(4) // 当前步骤

// 状态转换函数
const startSupplement = async () => {
  supplementState.value = 'loading'
  try {
    const result = await api.supplement(sessionId, missingItems)
    supplementResults.value = result.data.supplements
    supplementState.value = 'showing'
  } catch (error) {
    supplementState.value = 'closed'
    showError(error)
  }
}

const confirmSupplement = async () => {
  await api.confirmSupplement(sessionId, supplementResults.value)
  supplementState.value = 'closed'
  currentStep.value = 6 // 跳转 Step 6
}

const skipSupplement = async () => {
  await api.skipSupplement(sessionId)
  supplementState.value = 'closed'
  currentStep.value = 6 // 跳转 Step 6
}

const retrySupplement = async () => {
  if (degradationCount.value >= 2) {
    showMaxDegradationAlert()
    return
  }
  degradationCount.value++
  await startSupplement()
}
</script>
```

### 5.2 SupplementCard.vue 组件

```vue
<template>
  <div class="supplement-card">
    <!-- 降级提示 Alert -->
    <a-alert
      v-if="showAlert"
      :type="alertType"
      show-icon
      :message="alertMessage"
    />
    
    <!-- 内容区域 -->
    <div class="content">
      <!-- L0/L1: 显示内容 -->
      <div v-if="isL0OrL1" v-html="content" />
      
      <!-- L2: 显示问题列表 -->
      <div v-else-if="isL2" class="questions">
        <h4>建议你先思考这些问题：</h4>
        <ul>
          <li v-for="q in questions" :key="q">{{ q }}</li>
        </ul>
      </div>
      
      <!-- L3: 显示类比 -->
      <div v-else-if="isL3" class="analogy">
        <h4>类比推导：</h4>
        <p>{{ analogy }}</p>
      </div>
      
      <!-- L4: 显示框架 -->
      <div v-else-if="isL4" class="framework">
        <h4>思考维度：</h4>
        <ul>
          <li v-for="dim in framework.dimensions" :key="dim.name">
            <strong>{{ dim.name }}:</strong> {{ dim.hint }}
          </li>
        </ul>
      </div>
    </div>
    
    <!-- 来源标签 -->
    <div class="meta">
      <a-tag :color="tagColor">{{ tagText }}</a-tag>
      <span v-if="evidenceQuote" class="evidence">📖 {{ evidenceQuote }}</span>
    </div>
    
    <!-- 操作按钮 -->
    <div class="actions">
      <a-button type="primary" @click="confirm">确认</a-button>
      <a-button @click="skip">跳过</a-button>
      <a-button @click="retry" :disabled="canNotRetry">重新补充</a-button>
    </div>
  </div>
</template>

<script setup>
const props = defineProps({
  item: Object,
  knowledgeLevel: String,
  content: String,
  questions: Array,
  analogy: String,
  framework: Object,
  evidenceQuote: String,
  degradationCount: Number
})

const emit = defineEmits(['confirm', 'skip', 'retry'])

const isL0OrL1 = computed(() => ['L0', 'L1'].includes(props.knowledgeLevel))
const isL2 = computed(() => props.knowledgeLevel === 'L2')
const isL3 = computed(() => props.knowledgeLevel === 'L3')
const isL4 = computed(() => props.knowledgeLevel === 'L4')

const showAlert = computed(() => props.knowledgeLevel !== 'L0')

const alertType = computed(() => {
  const map = { L1: 'info', L2: 'warning', L3: 'info', L4: 'warning' }
  return map[props.knowledgeLevel] || 'info'
})

const alertMessage = computed(() => {
  const map = {
    L1: '💡 基于通用模式推导，部分内容需要你补充',
    L2: '⚠️ 这个话题 AI 知识有限，以下基于通用模式推导',
    L3: '💡 这个话题超出 AI 知识范围，以下基于类比推导',
    L4: '⚠️ 这个话题超出 AI 知识范围，建议你先提供相关资料'
  }
  return map[props.knowledgeLevel] || ''
})

const tagColor = computed(() => {
  const map = { L0: 'green', L1: 'blue', L2: 'orange', L3: 'gray', L4: 'gray' }
  return map[props.knowledgeLevel] || 'blue'
})

const tagText = computed(() => {
  const map = {
    L0: '🔍 API + AI 推断',
    L1: '🤖 AI 推理',
    L2: '🤖 AI 推理',
    L3: '🤖 AI 类比',
    L4: '🤖 AI 框架'
  }
  return map[props.knowledgeLevel] || '🤖 AI 推理'
})

const canNotRetry = computed(() => props.degradationCount >= 2)

const confirm = () => emit('confirm', props.item)
const skip = () => emit('skip', props.item)
const retry = () => emit('retry', props.item)
</script>
```

## 九、测试用例

### 9.1 功能测试

| 测试项 | 操作步骤 | 预期结果 |
|--------|---------|---------|
| 正常补充 | 点击补充→等待→确认 | 显示结果，确认后跳转 Step 6 |
| 跳过补充 | 点击补充→等待→跳过 | 不保存结果，跳转 Step 6 |
| 重新补充 | 点击补充→等待→重新补充 | 降级一级，重新显示结果 |
| 降级上限 | 重新补充 2 次→再次重新补充 | 显示"建议手动输入"提示 |
| API 超时 | 模拟 API 超时 15 秒 | 显示超时错误，按钮恢复 |
| API 失败 | 模拟 API 返回 500 | 显示错误提示，按钮恢复 |

### 9.2 UI 测试

| 测试项 | 验证点 | 验收标准 |
