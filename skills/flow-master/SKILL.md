# flow-master Skill

**流程主控 Skill - 标准开发团队流程**

---

## 🎯 核心职责

1. **流程执行监督** - 确保每个任务按 6 阶段执行
2. **架构设计** - 技术方案、接口定义
3. **代码审查** - trae 代码审查 + 修改建议
4. **测试验证** - 集成测试、回归测试、覆盖率审计
5. **部署运维** - 部署检查、执行部署、监控
6. **文档更新** - 自动更新 TASK_CURRENT.md 和 memory/

---

## 🎭 团队角色

| 角色 | 人员 | 职责 |
|------|------|------|
| 产品经理 (PM) | DAVID | 需求、优先级、最终判定 |
| 架构师 + 技术负责人 | 战舰 | 架构、方案、审查、测试、部署 |
| 开发 (Dev) | trae | 编码实现、单元测试 |
| 测试 (QA) | 战舰 | 测试用例、回归测试 |
| 运维 (Ops) | 战舰 | 部署、监控 |

---

## 🔄 标准流程（6 阶段）

```
阶段 1：需求（PM） → DAVID 需求 → 战舰澄清 → DAVID 确认
         ↓
阶段 2：架构（架构师） → 战舰技术方案 → DAVID 审批
         ↓
阶段 3：开发（Dev） → 战舰任务拆解 → trae 编码 → 战舰审查
         ↓
阶段 4：测试（QA） → 战舰集成测试 → 覆盖率审计
         ↓
阶段 5：部署（Ops） → 战舰部署检查 → DAVID 高危确认 → 战舰部署
         ↓
阶段 6：判定（PM） → DAVID 判定（通过/修改/重做）
```

---

##  trae 卡壳/代码审查处理

### trae 卡壳

```
trae 编码中
    ↓
检测卡壳（>10 分钟 OR 失败≥3 次）
    ↓
战舰技术指导 或 接管
    ↓
继续执行
```

### 代码审查（Code Review）

```
trae 完成编码
    ↓
战舰代码审查
    ↓
┌─────────────────────────────────────┐
│ 审查结果 | 处理                       │
├─────────────────────────────────────┤
│ ✅ 通过   │ 进入阶段 4：测试            │
│ ⚠️ 小修改  │ trae 修复 → 再审            │
│ ❌ 重做   │ 战舰指导 → trae 重做        │
└─────────────────────────────────────┘
```

## 🎤 唤醒词（14 个）

### 流程启动
- `开始新任务：[需求描述]` - 启动标准流程
- `继续之前的任务` - 继续中断的任务

### 流程检查
- `检查流程执行` - 检查当前阶段
- `下一步是什么` - 显示下一步

### 阶段推进
- `阶段 1 完成` - 需求确认完成 → 阶段 2
- `阶段 2 完成` - 架构设计完成 → 阶段 3
- `阶段 3 完成` - 开发完成 → 阶段 4
- `阶段 4 完成` - 测试完成 → 阶段 5
- `阶段 5 完成` - 部署完成 → 阶段 6

### trae 相关
- `trae 卡壳了` - 技术指导/接管
- `代码审查` - 审查 trae 代码

### 判定
- `提交判定` - 提交给 DAVID 判定
- `判定结果：通过/修改/重做` - 执行判定
