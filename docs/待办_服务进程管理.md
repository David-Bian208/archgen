**问题背景：** 测试后服务进程容易退出，需要手动重启

---

## 问题描述

**现状：**
- API 后端 (`python3 main.py`) 和前端 (`npm run dev`) 进程容易退出
- 每次测试后需要手动重启
- 影响测试效率和部署稳定性

**需求：**
1. 服务崩溃后自动重启
2. 开机自动启动
3. 方便手动重启（一条命令）

---

## 解决方案（选一种实现）

### 方案 A：systemd 服务（推荐，生产环境标准）

**创建文件：**

1. `/etc/systemd/system/behavior-api.service`
```ini
[Unit]
Description=Behavior Recorder API Service
After=network.target

[Service]
Type=simple
User=admin
WorkingDirectory=/home/admin/.openclaw/workspace/behavior_recorder_service
ExecStart=/usr/bin/python3 main.py
Restart=always
RestartSec=5
StandardOutput=append:/home/admin/.openclaw/workspace/behavior_recorder_service/logs/api.log
StandardError=append:/home/admin/.openclaw/workspace/behavior_recorder_service/logs/api.log

[Install]
WantedBy=multi-user.target
```

2. `/etc/systemd/system/behavior-frontend.service`
```ini
[Unit]
Description=Behavior Recorder Frontend Service
After=behavior-api.service network.target

[Service]
Type=simple
User=admin
WorkingDirectory=/home/admin/.openclaw/workspace/behavior_recorder_service/frontend
ExecStart=/usr/bin/npm run dev
Restart=always
RestartSec=5
Environment=NODE_ENV=production
StandardOutput=append:/home/admin/.openclaw/workspace/behavior_recorder_service/logs/frontend.log
StandardError=append:/home/admin/.openclaw/workspace/behavior_recorder_service/logs/frontend.log

[Install]
WantedBy=multi-user.target
```

**启用命令：**
```bash
sudo systemctl daemon-reload
sudo systemctl enable behavior-api
sudo systemctl enable behavior-frontend
sudo systemctl start behavior-api
sudo systemctl start behavior-frontend
```

**管理命令：**
```bash
# 查看状态
systemctl status behavior-api
systemctl status behavior-frontend

# 重启服务
sudo systemctl restart behavior-api
sudo systemctl restart behavior-frontend

# 查看日志
journalctl -u behavior-api -f
journalctl -u behavior-frontend -f
```

### 方案 B：简单重启脚本（开发环境快速方案）

**创建文件：** `scripts/restart.sh`

```bash
#!/bin/bash

echo "🔄 重启行为观察助手服务..."

# 杀掉旧进程
echo "📌 停止旧进程..."
pkill -9 -f "python3 main.py" 2>/dev/null || true
pkill -9 -f "node.*vite" 2>/dev/null || true
sleep 2

# 清理缓存
echo "🧹 清理 Python 缓存..."
find /home/admin/.openclaw/workspace/behavior_recorder_service -type d -name __pycache__ -exec rm -rf {} + 2>/dev/null || true
find /home/admin/.openclaw/workspace/behavior_recorder_service -name "*.pyc" -delete 2>/dev/null || true

# 确保日志目录存在
mkdir -p /home/admin/.openclaw/workspace/behavior_recorder_service/logs

# 启动后端
echo "🚀 启动 API 后端..."
cd /home/admin/.openclaw/workspace/behavior_recorder_service
nohup python3 main.py > logs/api.log 2>&1 &
API_PID=$!
sleep 3

# 检查后端是否启动成功
if curl -s http://localhost:8001/health > /dev/null; then
    echo "✅ API 后端启动成功 (PID: $API_PID)"
else
    echo "❌ API 后端启动失败，检查 logs/api.log"
    exit 1
fi

# 启动前端
echo "🎨 启动前端服务..."
cd /home/admin/.openclaw/workspace/behavior_recorder_service/frontend
nohup npm run dev > ../logs/frontend.log 2>&1 &
FRONTEND_PID=$!
sleep 3

# 检查前端是否启动成功
if curl -s http://localhost:5173 > /dev/null; then
    echo "✅ 前端服务启动成功 (PID: $FRONTEND_PID)"
else
    echo "❌ 前端服务启动失败，检查 logs/frontend.log"
    exit 1
fi

echo ""
echo "================================"
echo "✅ 服务重启完成！"
echo "================================"
echo "API 后端：http://localhost:8001"
echo "前端页面：http://localhost:5173"
echo "Swagger:  http://localhost:8001/docs"
echo ""
echo "查看日志：tail -f logs/api.log"
echo "          tail -f logs/frontend.log"
echo "================================"
```

**使用命令：**
```bash
chmod +x scripts/restart.sh
./scripts/restart.sh
```
