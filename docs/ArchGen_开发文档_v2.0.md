## 项目概述

### 项目定位

**ArchGen（Architecture Generator）** - 云端部署的逻辑框架可视化引擎。

用户输入自然语言需求 + 选择文件（已同步/上传/图片/文字） → 系统引导式收集信息 → 生成结构化逻辑架构图。

### 核心价值

| 问题 | 传统方案 | ArchGen 方案 |
|------|----------|-------------|
| **框架选择** | 用户自己找模板 | 系统智能推荐 + 用户确认 |
| **数据收集** | 用户手动整理 | 引导式追问补充 |
| **逻辑可视化** | 手动画图 | 自动按框架渲染 |
| **迭代成本** | 重画整个图 | 调整数据秒出 |
| **文件来源** | 手动上传 | AI-Pulse 自动下载 + 同步工具上传 |

### 设计原则

1. **引导式交互** - 系统引导 + 用户确认，决策权在用户
2. **逻辑归逻辑，视觉归视觉** - 拆解不准调 Prompt，图太丑改 CSS
3. **云端部署** - 后端运行在云服务器，支持多用户访问
4. **四元素输入** - 已同步文件/单个文件/图片/文字，四选一或多选

### 引导式工作流（9 步）

```
① 用户输入（自然语言 + 可选文件/图片/文字）
    ↓
② 语义理解 → 识别大类（自动）
    ↓
③ 【可选】大类确认（置信度<0.7 时）
    ↓
④ 框架推荐（Top 5 匹配度排序）
    ↓
⑤ 【必选】用户选择框架
    ↓
⑥ 数据完整性检查（自动）
    ↓
⑦ 【可选】补充缺失数据（逐个追问）
    ↓
⑧ 生成预览（自动）
    ↓
⑨ 【必选】用户确认/调整
    ↓
输出最终结果
```

### 状态机

```
初始状态 → 语义理解 → 大类确认 (可选) → 框架推荐 → 用户选择框架
                                                        ↓
最终输出 ← 用户确认 ← 生成预览 ← 补充数据 (可选) ← 数据检查
```

## 核心模块设计

### 1. 文件存储模块

**文件：** `src/storage.py`

**职责：** 管理云端文件存储，支持三种文件来源

**核心方法：**
- `upload_file(file, user_id)` - 上传文件
- `list_files(user_id)` - 列出用户文件列表
- `read_file(file_id)` - 读取文件内容
- `delete_file(file_id)` - 删除文件
- `sync_files_from_local(user_id, file_list)` - 同步本地文件列表（ArchGen Sync 工具调用）

**文件来源（三种）：**
| 来源 | 说明 | 优先级 |
|------|------|--------|
| **ArchGen Sync** | 本地同步工具上传 | 1（优先） |
| **手动上传** | 网页端手动上传单个文件 | 2（备选） |
| **临时上传** | 单次分析临时上传 | 3（兜底） |

**存储策略：**
- **临时文件：** 24 小时后自动清理
- **同步文件：** 永久存储（P1 阶段）
- **元数据：** 文件名、路径、大小、类型、上传时间

---

### 2. 分类器模块

**文件：** `src/classifier.py`

**职责：** 语义理解用户意图，识别业务大类

**核心方法：**
- `classify_by_intent(user_input)` - 语义理解分类
- `classify_by_button(category)` - 前端按钮选择
- `get_categories()` - 获取所有可用分类

**大类定义（7 个）：**
1. `business` - 商业分析
2. `finance` - 财务分析
3. `product` - 产品设计
4. `operations` - 运营优化
5. `personal` - 个人成长
6. `risk` - 风险管理
7. `project` - 项目管理

**置信度阈值：** 0.7
- ≥0.7：自动进入下一步
- <0.7：显示大类选项让用户确认

---

### 3. 框架匹配模块

**文件：** `src/framework_matcher.py`

**职责：** 根据大类和用户输入，匹配具体框架

**三层匹配策略：**
| 层级 | 权重 | 方法 | 说明 |
|------|------|------|------|
| **关键词** | 40% | 关键词匹配 | 快速、可解释 |
| **语义** | 50% | LLM 评分 | 理解深层意图 |
| **规则** | 10% | 规则兜底 | 处理特殊情况 |

**P0 框架（10 个）：**

| 框架 | 来源 | 业务领域 | layout_style |
|------|------|----------|--------------|
| SWOT 分析 | v2.0 新增 | business/personal/project | 2x2_matrix |
| 商业模式画布 | v2.0 新增 | business | 9_grid |
| PESTEL 分析 | v2.0 新增 | business/risk | 6_columns |
| 用户旅程地图 | v2.0 新增 | product/operations | timeline |
| 时间管理矩阵 | v2.0 新增 | personal/project | 2x2_matrix |
| 主张型分析 | v1.0 复用 | business/personal | center_radial |
| 因果分析 | v1.0 复用 | business/risk/project | flow_chart |
| 系统架构分析 | v1.0 复用 | product/project | layered_arch |
| 对比分析 | v1.0 复用 | business/product/finance | comparison_table |
| 流程分析 | v1.0 复用 | operations/project | timeline |

**返回格式：**
```json
[
    {
        "framework_key": "business_canvas",
        "framework_name": "商业模式画布",
        "score": 92,
        "reason": "用户提到'商业模式'，与 9 模块画布高度匹配",
        "layout_style": "9_grid",
        "output_preview": "九宫格布局"
    }
]
```

## API 接口设计

### 基础接口

```python
# 获取所有分类
GET /api/categories
→ {"code": 0, "data": {"categories": [{"key": "business", "name": "商业分析"}]}}

# 语义理解分类
POST /api/classify/intent
Request: {"user_input": "分析一下腾讯的商业模式"}
→ {"code": 0, "data": {"category": "business", "confidence": 0.92, ...}}

# 框架推荐
POST /api/frameworks/match
Request: {"category": "business", "user_input": "...", "content": "..."}
→ {"code": 0, "data": [{"framework_key": "business_canvas", "score": 92, ...}]}

# 数据完整性检查
POST /api/data/check
Request: {"framework_key": "business_canvas", "content": "..."}
→ {"code": 0, "data": {"completeness": 0.67, "questions": [...]}}

# 补充数据
POST /api/data/supplement
Request: {"framework_key": "business_canvas", "field": "客户细分", "value": "..."}
→ {"code": 0, "data": {"field": "客户细分", "status": "completed"}}

# 生成架构图
POST /api/generate
Request: {"framework_key": "business_canvas", "data": {...}, "style": "minimal", "size": "wechat"}
→ PNG 文件（同步模式）或 {"task_id": "..."}（异步模式）

# 文件列表（已同步 + 手动上传）
GET /api/files/list
→ {"code": 0, "data": {"files": [{"id": "1", "filename": "...", "source": "sync", ...}]}}

# 上传文件
POST /api/files/upload
Request: multipart/form-data (file)
→ {"code": 0, "data": {"file_id": "123", "filename": "..."}}

# 同步文件列表（ArchGen Sync 工具调用）
POST /api/files/sync
Request: {"files": [{"filename": "...", "path": "...", "size": 1234, ...}]}
→ {"code": 0, "data": {"synced_count": 156}}

# 读取文件内容
GET /api/files/{file_id}/content
→ {"code": 0, "data": {"content": "..."}}
```
