# P3 多源素材接入方案

> 版本：v1.0 | 2026-07-04 | 状态：方案定稿待开发

---

## 核心原则

- **不增加 LLM 调用次数** — 所有新增逻辑都不涉及额外 LLM 调用
- **不增加用户等待时间** — 网络 IO 预拉取到后台，用户交互路径零额外延迟
- **不改现有匹配算法** — `_match_materials_internal` 的双字词匹配不动

---

## 5 项改动

### 改动 1：Step 1 确认方向时异步预拉 AIPULSE

**位置**：`api.py` `generate_slots()` (L7239)

**当前行为**：保存 direction 到 session，然后用 SSE 流式生成槽位。

**改后行为**：SSE 流开始后，用 `BackgroundTasks` 异步拉 AIPULSE：

```python
# 在 generate_slots 签名中添加 background_tasks
async def generate_slots(request: Dict = Body(...), background_tasks: BackgroundTasks):
    ...
    # 启动后台任务拉取 AIPULSE
    topic = request.get("topic", "")
    background_tasks.add_task(_prefetch_aipulse, session_id, topic)
    
    # 继续现有的 SSE 流...
```

**新增函数**：

```python
async def _prefetch_aipulse(session_id: str, topic: str):
    """后台异步拉取 AIPULSE 内容到 session"""
    try:
        from src.ai_pulse_client import AIPulseClient
        client = AIPulseClient()
        results = client.fetch_latest_cases(query=topic, take=20)
        if results:
            session = _get_session(session_id)
            session["aipulse_items"] = results
            logger.info(f"[AIPULSE] 预拉取完成: {len(results)}条, session={session_id}")
    except Exception as e:
        logger.warning(f"[AIPULSE] 预拉取失败(不阻塞): {e}")
```

**注意**：`time_filter=month` 必须传，否则默认返回空。

### 改动 2：batch_fill_v4 prompt 加 search_keywords

**位置**：`api.py` `batch_fill_v4()` prompt (L7704)

**当前 prompt** 要求 LLM 输出 `{section_content, key_points, materials}`

**改后 prompt** 多输出一个字段：

```
同时为每个槽位生成 3-5 个中文搜索关键词，用于在素材库中匹配相关内容。
搜索关键词应精准、多样，涵盖槽位核心概念的近义词和相关概念。
```

输出示例：

```json
{
  "slot_1": {
    "outline": ["技术壁垒是企业核心护城河"],
    "search_keywords": ["技术壁垒", "专利护城河", "研发能力", "技术门槛"]
  },
  "slot_2": {
    "outline": ["品牌溢价来自用户信任"],
    "search_keywords": ["品牌溢价", "用户信任", "品牌价值", "消费者认知"]
  }
}
```

### 改动 3：_match_materials_internal 拼接 search_keywords

**位置**：`api.py` `_match_materials_internal()` (L7151)

**当前**：`combined = label + " " + description` 作为匹配文本

**改后**：

```python
combined = label + " " + description
if slot.get("search_keywords"):
    combined += " " + " ".join(slot["search_keywords"])
```

search_keywords 以单倍权重参与所有评分计算，不做特殊加权。

### 改动 4：_ensure_material_pool 加 AIPULSE + 去重

**位置**：`api.py` `_ensure_material_pool()` (L7036)

**当前**：

```python
pool = mcp_summary + mcp_files + supplements
```

**改后**：

```python
pool = mcp_summary + mcp_files + supplements

# 追加 AIPULSE 预拉取结果（带去重）
session = _get_session(session_id)
aipulse_items = session.get("aipulse_items", [])
if aipulse_items:
    existing_titles = [item.get("filename", "") or item.get("text", "")[:30] 
                       for item in pool]
    for item in aipulse_items:
        title = item.get("title", "")
        if not title:
            continue
        # 标题去重
        if any(_titles_similar(title, et) for et in existing_titles if et):
            continue
        pool.append({
            "text": item.get("summary", "") or item.get("content", "")[:500],
            "source_type": "aipulse",
            "filename": f"aipulse_{item.get('id', '')}",
            "title": title,
            "url": item.get("url", ""),
            "score": 0,  # 由 _match_materials_internal 计算
        })
```

**去重函数**：

```python
from difflib import SequenceMatcher

def _titles_similar(t1: str, t2: str, threshold: float = 0.6) -> bool:
    """短标题相似度比较，用于去重"""
    if not t1 or not t2:
        return False
    return SequenceMatcher(None, t1.lower(), t2.lower()).ratio() > threshold
```

## AIPULSE 接口参考

**Base URL**: `http://8.130.148.166:8887`

### GET /api/public/items

| 参数 | 类型 | 说明 | 默认 |
|------|------|------|------|
| `q` | string | 关键词搜索 | 空（返回全部） |
| `take` | int | 返回条数 | 20 |
| `time_filter` | string | `week` / `month` / `all` | 默认可能空，建议传 `month` |
| `cursor` | string | 分页游标 | 无 |

**响应**：

```json
{
  "count": 3,
  "total": 98,
  "hasNext": true,
  "items": [
    {
      "id": "25",
      "title": "文章标题",
      "url": "https://...",
      "source": "公众号：xxx",
      "summary": "摘要文本...",
      "full_content": "完整正文...",
      "publishedAt": "2026-06-23T06:04:50",
      "final_score": 73.0,
      "category": "ai-models",
      "tags": ["开源", "多模态"]
    }
  ]
}
```

## 关键决策记录

| 决策 | 选项 | 选定 |
|------|------|------|
| 去重算法 | 200字符前缀 / SequenceMatcher / jieba余弦 | **SequenceMatcher**（stdlib自带，0依赖） |
| AIPULSE 拉取时机 | Step 1 确认方向 / Step 2 槽位确认 / 即时 | **Step 1 确认方向后台异步**（延迟隐藏） |
| search_keywords 权重 | 等权 / 加权 / 独立查询 | **等权拼接**（不改现有评分逻辑） |
| 视觉区分 | badge + 边框 / 独立区域 / tooltip | **绿色左边框 + [AI-Pulse] badge** |

## 验证方法

1. 启动后端后手动调 AIPULSE API 确认有数据返回
2. `generate_slots` 触发后检查 `session["aipulse_items"]` 有值
3. `batch_fill_v4` 调用后检查 `_match_materials_internal` 匹配到 AIPULSE 项目
4. 前端检查 AIPULSE 素材卡片显示绿色样式
5. 同一篇文章在 KB 和 AIPULSE 中同时存在 → pool 里只出现一次
