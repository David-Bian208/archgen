## 📋 测试用例

### 用例 1：薯片盒子游戏（观点采择）

**输入：**
```
老师好，想跟您分享一下 OK 在家的小趣事。
今天爸爸和 OK 玩了一个小游戏：我先让她看到一个薯片盒子，
她猜里面是薯片，结果打开发现装的是糖果。
盖上盒子后，她可以准确说出里面是糖果。
这时我走进房间，问她"妈妈会看到什么"，
她以为我看到盒子里还是糖，不太理解我看到的是薯片的意思。
```

**预期流程：**
1. ABC Skills 提取：
   - 前因：爸爸和 OK 玩薯片盒子游戏
   - 行为：OK 说"糖"（当被问妈妈会看到什么）
   - 后果：妈妈继续引导
   - 年龄：未明确

2. 行为分析 Agent：
   - 行为功能：寻求关注/其他
   - 能力缺口：心理理论（观点采择）
   - 问题识别：无法推断他人视角

3. 心伴解读 Agent：
   - 情感锚点：感谢家长观察
   - 发展地图：心理理论发展阶段
   - 优先级：观点采择能力
   - 建议：1+2+3 结构

4. 前端展示：
   - 📝 场景描述（蓝色）
   - 🔍 行为分析（黄色）
   - 💡 心伴解读（粉色）

---

### 用例 2：等待提示（提示依赖）

**输入：**
```
小明 5 岁，在幼儿园遇到困难就说"我不会"，
等着老师来教。其实他有能力完成，
但就是不愿意自己尝试。
```

**预期输出：**
- ABC：前因（遇到困难）、行为（说不会）、后果（等老师）
- 行为分析：执行功能（抑制控制）、心理理论
- 心伴解读：发展性语言、最小可行干预

---

### 用例 3：寻求关注（打断对话）

**输入：**
```
玥玥 4 岁 3 个月，老师看她时就故意发出奇怪声音，
打断老师和其他小朋友的对话。
```

**预期输出：**
- ABC：前因（老师看别人）、行为（发怪声）、后果（打断对话）
- 行为分析：寻求关注、心理理论
- 心伴解读：温暖解读、具体建议

## ✅ 验证标准

### ABC Skills 验证
- [ ] 年龄提取准确（支持 5 岁、4 岁 3 个月等格式）
- [ ] 姓名提取准确（OK、小明、玥玥等）
- [ ] 前因、行为、后果提取准确

### 行为分析 Agent 验证
- [ ] 行为功能判断合理
- [ ] 能力缺口分析准确
- [ ] 问题识别清晰（3-5 个）

### 心伴解读 Agent 验证
- [ ] 语言风格温暖、像朋友聊天
- [ ] 用"我们"视角
- [ ] 发展性语言（非标签化）
- [ ] 1+2+3 结构（1 原则 +2 场景 +3 小改变）
- [ ] 期待管理合理

### 前端展示验证
- [ ] 三部分结构清晰（📝🔍）
- [ ] 颜色区分明显（蓝黄粉）
- [ ] 移动端适配正常
- [ ] 无 CORS 错误、无 MIME type 错误

## 🧪 测试步骤

### Step 1：准备测试环境
```bash
# 1. 检查后端服务
curl http://localhost:8001/health

# 2. 检查前端服务
curl http://localhost:5173/

# 3. 检查环境变量
echo $LLM_API_KEY
```

### Step 2：发送测试消息
1. 打开前端页面（http://localhost:5173）
2. 输入测试用例 1（薯片盒子游戏）
3. 点击发送
4. 等待报告生成

### Step 3：验证输出
1. 检查 ABC 场景描述（蓝色部分）
2. 检查行为分析（黄色部分）
3. 检查心伴解读（粉色部分）
4. 检查语言风格

### Step 4：记录结果
填写测试报告：
- 测试用例
- 预期结果
- 实际结果
- 问题记录
