## 问题 2：能力缺口分析像复述 ❌

### 用户反馈

> "能力缺口分析，像是在复述，不是分析"

### 当前输出示例

```
2. 能力缺口分析：
   • 观点采择能力不足：'认为妈妈会看到糖果而不是薯片'→表明难以推断与自己视角不同（妈妈视角）时他人会看到什么。
   • 认知灵活性不足：'认为妈妈还是在用电脑，爸爸还是在用手机'→表明在信息（位置）已改变后，难以更新原有的认知脚本。
```

**问题：**
- 只是重复了孩子的行为
- 没有分析"为什么"会有这个困难
- 没有连接到发展心理学理论

---

### 期望输出示例

```
2. 能力缺口分析：
   • 观点采择能力不足：孩子在这个年龄阶段，大脑中负责"心理理论"（Theory of Mind）的神经网络还在发育中。他默认"我看到的就是你看到的"，这是典型的自我中心思维特点。需要更多"错误信念"类游戏来帮助他理解"每个人知道的信息可能不同"。
   
   • 认知灵活性不足：当位置交换后，孩子需要同时处理两套信息（"刚才的位置"和"现在的位置"），这对工作记忆和认知切换能力是挑战。他不是"记不住"，而是大脑还在练习如何快速更新心理地图。
```

**关键差异：**
- ✅ 解释"为什么"（神经发育、认知负荷）
- ✅ 连接到发展理论（心理理论、自我中心思维）
- ✅ 正常化（"这是典型的...特点"而非"缺陷"）
- ✅ 指出发展方向（"需要更多...游戏"）

---

### 修复方案

**文件：** `app/agents/insight_analyzer.py`

**修改位置：** `ANALYSIS_SYSTEM_PROMPT` 中的"能力缺口分析"部分

**修改前：**
```python
3. **能力缺口分析**：
   - 基于以上分析，指出 2-3 个核心能力缺口
   - 每个缺口用"行为表现→能力解释"的格式
```

**修改后：**
```python
3. **能力缺口分析**：
   - 基于以上分析，指出 2-3 个核心能力缺口
   - 每个缺口包含三层：
     1. **行为表现**：简短引用孩子的具体行为
     2. **发展解读**：解释这个表现反映了什么发展阶段特点（引用发展心理学理论，如心理理论、执行功能、工作记忆等）
     3. **正常化 + 方向**：说明这是发展过程中的常见现象，指出能力发展方向
   
   **示例格式**：
   ```
   • 观点采择能力不足：孩子在这个年龄阶段，大脑中负责"心理理论"的神经网络还在发育中。他默认"我看到的就是你看到的"，这是典型的自我中心思维特点。需要更多"错误信念"类游戏来帮助他理解"每个人知道的信息可能不同"。
   ```
   
   **避免**：
   - ❌ 只重复行为（"认为妈妈会看到糖果→表明难以推断..."）
   - ❌ 使用缺陷语言（"不足"、"困难"后不给发展方向）
   
   **提倡**：
   - ✅ 解释发展机制（"大脑还在发育"、"工作记忆容量有限"）
   - ✅ 正常化（"这是典型的...特点"、"常见现象"）
   - ✅ 指出方向（"需要更多...游戏"、"可以通过...练习"）
```
