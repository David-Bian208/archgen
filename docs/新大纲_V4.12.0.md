## 📊 系统定位

**行为观察助手 = 专业临床评估工具**

**核心价值：**
- 帮助家长理解孩子行为背后的原因
- 提供专业、温暖、可操作的洞察
- 不是诊断工具，是"理解桥梁"

---

## 🏗️ 核心功能模块

### 模块 1：心伴解读（核心）
**功能：** 深度解读孩子行为的心理意义

**输入：** 家长描述的行为场景
**输出：** 
- 行为功能判断（寻求关注/逃避/获得物品/自我刺激）
- 能力缺口分析（心理理论/执行功能/视觉空间/语言理解）
- 温暖解读（像朋友聊天，不是写论文）

**报告结构：**
```markdown
## 📋 核心洞察
孩子正在用这种方式表达他的需求。

## 🎯 行为功能判断
功能判断：寻求关注
核心能力目标：心理理论（观点采择）

## 🔍 临床鉴别思考
基于观察到的行为模式，我们考虑了多种可能性...

## 💡 家庭引导建议
在家可以这样引导...

## 🏫 幼儿园配合建议
在幼儿园可以这样配合...
```

---

### 模块 2：四维度归因（Skill）
**功能：** 从四个认知维度分析行为

**四维度：**
1. 心理理论（核心缺口）
2. 执行功能
3. 视觉 - 空间认知
4. 语言与概念理解

**输出：** 齿轮啮合比喻，描述四维度的协同关系

**使用场景：** 复杂行为分析（需要深度解读时）

---

### 模块 3：安全优先模式
**功能：** 危险行为自动触发安全干预

**触发条件：**
- 打人、咬人、发火等危险关键词
- 自伤、攻击他人等行为

**处理流程：**
1. 安全关键词检测
2. 强制启用安全优先模式
3. 输出安全干预建议
4. 记录人工审核队列

---

### 模块 4：干预建议（辅助参考）
**功能：** 提供家庭引导方向

**原则：**
- 通用框架，非具体方案
- 明确"仅供参考，建议咨询专业治疗师"
- 聚焦干预方向，不写具体游戏步骤

## 📐 架构设计

### 前端（Vite + Vue 3）
- 端口：5174
- 核心页面：对话界面、报告展示
- API 调用：相对路径 `/api/v4/analyze`

### 后端（FastAPI）
- 端口：8001
- 核心 API：
  - `POST /api/v4/analyze` - 行为分析
  - `POST /api/v4/feedback` - 反馈收集
  - `GET /health` - 健康检查

### LLM 服务
- 服务：DeepSeek
- API Key: `LLM_API_KEY`（环境变量）
- Base URL: `https://api.deepseek.com`

### 三方协作流程
```
DAVID（PM）→ 需求
  ↓
战舰（架构师）→ 技术方案 + 任务派发
  ↓
trae（5 个窗口）：
  - 小美（Dev-1）→ 前端开发
  - 小治（Dev-2）→ 后端开发
  - 小强（Dev-3）→ 性能测试
  - 小测试（Dev-4）→ 功能测试
  - 小文（Dev-5）→ 用户文档
  ↓
小智检测 → 小测测试 → 战舰提交 Git → DAVID 判定
```
