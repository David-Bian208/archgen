- ⑤ 生成配图：AI 根据文章内容生成配套插图

## 二、工作流详解

### 2.1 完整流程（5 步）

```
选题
  ↓ 用户输入创作主题
补充材料
  ↓ 系统自动检索知识库 + 网络搜索
  ↓ 用户可手动补充文本/文件/图片
  ↓ 输出：素材池（material_pool）
三列分析工作台  ← ✨ 核心改动
  ├─ 阶段 1：流式推理槽位
  │   ├─ AI 打字机输出 → 分析思路 + 推荐维度（4-6个）
  │   ├─ 用户：增删改维度名称 + 新增/删除维度
  │   └─ 流式完成后自动触发 ↓
  │
  ├─ 阶段 1.5：素材可行性预检 ← 🆕
  │   ├─ 快速统计每个槽位能匹配几条素材
  │   ├─ 🟢(充足) / 🟡(不足) / 🔴(空) 状态灯
  │   ├─ 🔴🟡 槽位：AI 推荐替代维度（基于素材池内容）
  │   ├─ 用户：[采纳建议] 替换 / [手动编辑] 改名 / [保留原槽位]
  │   └─ 确认后进入 ↓
  │
  ├─ 阶段 2：同时生成素材 + 提纲
  │   ├─ AI 从素材池匹配素材到每个槽位 → 第二列
  │   ├─ AI 根据主题+素材生成提纲要点 → 第三列
  │   └─ 一次性填充三列
  │
  ├─ 阶段 3：精调（可选）
  │   ├─ 点击槽位行的 [✏️ 编辑]
  │   ├─ 右侧滑出编辑面板
  │   ├─ 写作方案：用户描述这个槽位怎么写
  │   ├─ 补充素材：输入文本/上传文件/上传图片
  │   └─ 微调提纲：直接编辑提纲内容
  │
  └─ 全部确认 → 进入下一步
  ↓
生产全文
  ↓ AI 基于三列内容自动撰写完整文章
生成配图
```

### 2.2 三列的数据来源

| 列 | 谁生成 | 用户可以做什么 | 数据存储位置 |
|----|--------|---------------|-------------|
| ① 槽位 | AI 流式推理 | 增删改、新增维度 | `session.framework_slots[]` |
| ② 素材 | AI 从素材池匹配 | 手动补充、调整来源 | `session.framework_slots[].materials[]` |
| ③ 提纲 | AI 基于主题+素材生成 | 编辑要点 | `session.framework_slots[].outline[]` |

## 六、技术实现要点

### 6.1 SSE 流式推理

```javascript
// api.js 中新增
async function generateSlots(sessionId, topic, materialSummary, onChunk, onDone) {
  const response = await fetch('/api/workflow/slot/generate_slots', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ session_id: sessionId, topic, material_pool_summary: materialSummary })
  })

  const reader = response.body.getReader()
  const decoder = new TextDecoder()

  while (true) {
    const { done, value } = await reader.read()
    if (done) break

    const text = decoder.decode(value)
    const lines = text.split('\n').filter(l => l.startsWith('data: '))

    for (const line of lines) {
      const data = JSON.parse(line.replace('data: ', ''))
      if (data.type === 'done') {
        onDone(data.slots)
      } else {
        onChunk(data)
      }
    }
  }
}
```

### 6.2 素材匹配逻辑

```
输入：素材池（N 条文本） + M 个槽位（每个有关键词）

匹配策略：
  1. 关键词匹配：素材文本包含槽位关键词 → 高置信度(0.9+)
  2. 语义向量匹配：embedding 计算素材与槽位描述的余弦相似度 → 中置信度(0.6+)
  3. 超过阈值的素材分配到对应槽位
  4. 低于阈值的素材标记为「未匹配」，放入通用素材池

输出：每个槽位挂 0-N 条素材
```
