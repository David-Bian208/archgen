## 📖 核心实体

### 行为记录 (Behavior Record)

**定义：** 对孩子行为的结构化描述，包含 ABC 三要素

**结构：**
```json
{
  "antecedent": "前因（行为发生前的情境）",
  "behavior": "行为（具体可观察的行为描述）",
  "consequence": "后果（行为后的结果/他人反应）"
}
```

**示例：**
- 前因："幼儿园轮流玩玩具时"
- 行为："小明抢走小红的玩具"
- 后果："老师批评，小红哭了"

---

### 临床假设 (Clinical Hypothesis)

**定义：** 对行为功能的临床推断

**5 类核心假设：**

| ID | 名称 | 关键词 | 干预方向 |
|----|------|--------|----------|
| H1 | 提示依赖 | "不看提示"、"等待" | 建立视觉/动作锚点 |
| H2 | 逃避需求 | "不做"、"逃跑"、"拒绝" | 任务分解、降低起点 |
| H3 | 寻求关注 | "看大人"、"吸引注意" | 关注重定向 |
| H4 | 感觉寻求 | "盯着看"、"转圈"、"摸" | 提供安全替代刺激 |
| H5 | 坚持同一性 | "必须"、"固定"、"拒绝改变" | 预告 + 微小变化 |

---

### 场景分类 (Scene Category)

**定义：** 行为发生的情境类型

**5 类核心场景：**

| 类别 | 关键词 | 干预重点 |
|------|--------|----------|
| 共同游戏 | "一起"、"分享"、"轮流" | 社交信号监测 |
| 对话/介绍 | "名字"、"介绍"、"聊天" | 观察 - 反应练习 |
| 规则变化 | "规则"、"改变"、"新的" | 预告 + 小变化 |
| 身体边界 | "身体"、"触摸"、"距离" | 身体地图游戏 |
| 情绪识别 | "情绪"、"表情"、"开心" | 表情大侦探 |

---

### 干预建议 (Intervention Plan)

**定义：** 基于临床假设的个性化干预方案

**结构：**
```json
{
  "phase_name": "阶段名称",
  "goal": "目标",
  "core_principle": "核心原则",
  "four_step_plan": {
    "core_idea": "核心思路",
    "our_plan": "具体计划",
    "success_picture": "成功画面",
    "first_step": "第一步"
  },
  "observation_tool": "观察工具"
}
```

---

### 安全优先模式 (Safety First Mode)

**定义：** 检测到危险行为时触发的特殊模式

**触发条件：**
- 危险关键词：爬高、危险、利器、火、电、水、交通...
- 上下文确认：排除误报（如"发火"≠"玩火"）

**干预原则：**
1. 立即消除危险
2. 安全前提下最小化反应
3. 教授安全替代行为
4. 持续监控

**禁止：** 简单"忽视"危险行为

## 📝 报告语言风格原则

### 必须遵守

1. **避免通用比喻** - 不用"社交雷达"，用具体行为细节
2. **避免说教感** - 不用"临床上通常考虑"，直接分析
3. **用"我们"视角** - "从玥玥的表现来看，我们觉得..."
4. **能力发展视角** - "还没学会"而不是"不会"
5. **短句优先** - 每句<30 字，像聊天不像论文

### 示例对比

| ❌ 错误 | ✅ 正确 |
|--------|--------|
| "临床上通常考虑几种可能性" | "从玥玥的表现来看，我们觉得..." |
| "孩子不会社交" | "孩子还没学会这个社交技能" |
| "需要进行社交训练" | "可以一起练习这个技能" |
| "工作记忆缺陷" | "工作记忆还在发展中" |
