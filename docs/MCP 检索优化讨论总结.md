## 一、当前问题诊断

### 1.1 MCP 检索无方向性
- **问题**：搜索"agent"返回 4 篇资料，涵盖完全不同的主题（技术、商业、应用等）
- **后果**：LLM 总结缺乏聚焦，每次结果差异大
- **根因**：检索时没有意图引导，资料太宽泛

### 1.2 数据填充流程断裂
- **问题**：用户选择"流程分析"框架，但 MCP 检索的内容根本不是流程性质的
- **表现**：AI 自动填充时只能填一个标题，其他字段显示 `[object Object]`（JSON 解析错误）
- **后果**：完整度显示 100%，但实际无法生成架构图

### 1.3 缺少内容预览环节
- **问题**：用户填完数据后直接进入"生成架构图"
- **后果**：发现问题时已经晚了，没有文字预览确认环节

## 二、参考：行为观察助手的引导逻辑

### 2.1 ABC 引导式多轮对话

```
用户输入 → 语义判断 → 完整？→ 直接推理
                ↓ 不完整
            ABC 引导：
            A 前因 → B 行为 → C 后果
                ↓ 智能检测
            如果 B 已含 C → 跳过追问
```

### 2.2 四维度归因模式
从 4 个认知维度分析行为：
1. **心理理论**（理解他人想法）
2. **执行功能**（工作记忆/抑制控制）
3. **视觉 - 空间认知**（空间理解）
4. **语言 - 概念理解**（词汇/概念抽象）

### 2.3 输入质量判断
- 有行为动词 + 有场景词 → 完整
- 有其一 → 部分
- 都没有 → 模糊 → LLM 兜底判断

## 四、具体实现建议

### 4.1 意图引导（类比 ABC 引导）

**行为观察助手**问：
> "发生前孩子在哪里？正在做什么？"（A 前因）
> "孩子具体做了什么？"（B 行为）
> "您当时怎么处理的？"（C 后果）

**ArchGen**可以问：
> "你想写什么方向的文章？"
> - [ ] 技术深度分析（架构/算法/实现）
> - [ ] 商业趋势解读（市场/竞品/模式）
> - [ ] 应用场景拆解（案例/解决方案）
> - [ ] 竞品对比评估（选型/优劣分析）
> - [ ] 行业研究报告（宏观/趋势预测）

### 4.2 语义判断（类比输入质量判断）

**行为观察助手**判断：
- 有行为动词 + 有场景词 → 完整
- 有其一 → 部分
- 都没有 → 模糊

**ArchGen**可以判断：
- MCP 检索到的资料数量 ≥ 3 篇 → 资料充足
- 1-2 篇 → 资料偏少，建议补充
- 0 篇 → 无资料，建议换关键词

### 4.3 智能检测（类比 B 含 C 自动识别）

**行为观察助手**：
> 用户回答 B 时，检测 15 个后果关键词，匹配≥2 个 → 跳过 C 追问

**ArchGen**可以：
> 用户选择"技术向"后，系统检测资料中是否已包含商业内容
> 如果包含 → 自动提示"资料中也包含商业内容，是否也分析商业方向？"

## 五、待确认事项（需战舰决策）

### 5.1 方向建议的实现方式

| 方案 | 说明 | 优点 | 缺点 |
|------|------|------|------|
| 方案 A | AI 自动生成方向 | 智能、适配性强 | 不稳定，每次可能不同 |
| 方案 B | 预设方向模板 + AI 填充覆盖度 | 稳定、一致性好 | 不够灵活 |

**建议**：先用方案 B 快速验证，后续升级到方案 A

### 5.2 文字预览的展示方式

| 方案 | 说明 | 优点 | 缺点 |
|------|------|------|------|
| 方案 A | 独立文字预览页面 | 确认环节清晰 | 增加操作步骤 |
| 方案 B | 数据填充页面实时预览 | 所见即所得 | 页面可能拥挤 |

**建议**：先用方案 B（实时预览），用户反馈后再考虑独立页面

### 5.3 数据完整度的计算方式

| 方案 | 说明 | 优点 | 缺点 |
|------|------|------|------|
| 方案 A | 总字段完成 30% 即可生成 | 门槛低 | 可能内容不足 |
| 方案 B | 必填字段 100% 完成即可生成 | 质量有保障 | 用户可能觉得严格 |

**建议**：采用方案 B，必填字段必须 100% 完成

## 七、技术实现要点

### 7.1 后端修改
1. MCP 检索接口增加 `intent` 参数（技术/商业/应用等）
2. 新增方向建议接口，返回 3-5 个方向选项
3. 每个方向包含：名称、适合框架、资料覆盖度、需补充内容

### 7.2 前端修改
1. MCP 检索页面增加意图引导标签
2. 新增方向建议页面（Step 2）
3. 数据填充页面改为结构化表单
4. 新增文字预览页面（Step 5）

### 7.3 数据流
```
用户输入关键词 + 选择意图
  ↓
MCP 检索（带意图过滤）
  ↓
AI 分析资料，生成方向建议
  ↓
用户选择方向
  ↓
推荐框架 + 数据填充
  ↓
文字预览确认
  ↓
生成架构图
```

## 八、总结

这套方案的核心价值：

1. **意图引导** - 给 MCP 检索装上方向盘，避免资料太宽泛
2. **方向建议** - 类比 ABC 引导，帮助用户明确写作角度
3. **资料覆盖度** - 让用户清楚知道哪些内容充足、哪些需要补充
4. **结构化表单** - 解决 `[object Object]` 错误，提升填写体验
5. **文字预览** - 增加确认环节，避免生成后才发现问题
