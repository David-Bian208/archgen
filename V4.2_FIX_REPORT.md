### 问题 2：摘要机械复制首条消息（P1）✅ 已修复

**根本原因**：无摘要合成逻辑

- 原代码没有从对话历史中提取关键信息的逻辑
- 摘要字段直接为空或使用默认值

**修复方案**：
新增 `_generate_summary()` 方法：

```python
def _generate_summary(self, state: ClinicalInterviewState) -> str:
    """V4.2 新增：生成综合摘要"""
    # 1. 从收集的数据中提取关键事实
    # 2. 从对话历史中提取额外关键信息（关键词匹配）
    # 3. 构建叙述性摘要
    summary_parts = []
    
    # 环境/情境
    if environment:
        summary_parts.append(f"在{environment}的环境中")
    
    # 行为模式
    if behavior:
        summary_parts.append(f"观察到孩子{behavior}")
    
    # 关键补充信息（从对话历史提取）
    for fact in key_facts[:3]:
        summary_parts.append(f"进一步了解：{fact}")
    
    # 假设（如果有）
    if state.locked_hypothesis:
        summary_parts.append(f"初步判断可能与{hyp_name}有关")
    
    return "。".join(summary_parts)
```

### 问题 3：单方面结束对话（P2）✅ 已修复

**根本原因**：无协商结束状态

- 原代码在 `_should_complete()` 返回 True 后直接触发分析并结束
- 没有给用户确认是否有补充的机会

**修复方案**：
1. 新增 `READY_TO_CONCLUDE` 状态
2. 修改状态转移逻辑：`READY_TO_ANALYZE → READY_TO_CONCLUDE → ANALYZING`
3. 在 `READY_TO_CONCLUDE` 状态下询问用户确认

```python
# 新增状态
class InterviewState(Enum):
    READY_TO_CONCLUDE = "ready_to_conclude"  # V4.2 新增：准备结束，等待用户确认

# 状态转移逻辑
if current == InterviewState.READY_TO_ANALYZE:
    return InterviewState.READY_TO_CONCLUDE  # 先询问确认

if current == InterviewState.READY_TO_CONCLUDE:
    if any(kw in user_input.lower() for kw in ["没有", "完了", "可以", "好的"]):
        return InterviewState.ANALYZING  # 用户确认，开始分析
    elif len(user_input.strip()) > 5:
        return InterviewState.HYPOTHESIS_DRIVEN  # 用户有补充，继续收集
```

**验收结果**：
- 系统会在信息收集完毕后询问："请问还有没有其他重要的细节需要补充？"
- 用户确认后才会生成报告

### 额外修复：干预计划四步结构为空

**根本原因**：场景 key 为空字符串

```python
# 原代码
scenario_key = ""  # V4.0 暂不依赖场景匹配
plan = planner.generate_plan("H1", scenario_key, session_context)
```
