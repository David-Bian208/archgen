## 📱 小程序发布步骤

### 第 1 步：注册微信小程序

**网址：** https://mp.weixin.qq.com/

**需要：**
- 邮箱
- 身份证（个人）或 营业执照（企业）
- 手机号

**时间：** 10-15 分钟

### 第 2 步：获取 AppID

1. 登录小程序后台
2. 开发 → 开发管理 → 开发设置
3. 复制 AppID

### 第 3 步：下载微信开发者工具

**网址：** https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html

**安装后：**
1. 打开工具
2. 微信扫码登录
3. 导入项目（frontend 目录）
4. 填写 AppID

### 第 4 步：创建配置文件

在 `frontend/` 目录下创建 `project.config.json`：

```json
{
  "description": "行为观察伙伴",
  "appid": "你的 AppID",
  "projectname": "behavior-recorder",
  "libVersion": "2.19.4",
  "setting": {
    "urlCheck": true,
    "es6": true,
    "minified": true
  }
}
```

### 第 5 步：预览测试

1. 点击"预览" → 生成二维码
2. 手机微信扫码
3. 测试所有功能

### 第 6 步：上传代码

1. 点击"上传"
2. 填写版本号（如 1.0.0）
3. 填写备注（如"初版测试"）
4. 确认上传

### 第 7 步：设置为体验版

1. 登录小程序后台
2. 版本管理 → 开发版本
3. 点击"选为体验版"
4. 添加体验者（输入微信号）

### 第 8 步：分享给测试用户

1. 生成体验版二维码
2. 发送给测试用户
3. 收集反馈

**详细流程见：** `微信小程序发布流程.md`

## 📊 数据分析

### 查看今日测试数据

```sql
-- 查看今天的测试会话数
SELECT COUNT(DISTINCT session_id) 
FROM test_logs 
WHERE DATE(timestamp) = DATE('now');

-- 查看今天的反馈
SELECT * FROM user_feedback 
WHERE DATE(timestamp) = DATE('now');
```

### 分析用户满意度

```sql
-- 平均评分
SELECT AVG(rating) FROM user_feedback;

-- 评分分布
SELECT rating, COUNT(*) as count 
FROM user_feedback 
GROUP BY rating 
ORDER BY rating;

-- 准确性分布
SELECT accuracy, COUNT(*) as count 
FROM user_feedback 
GROUP BY accuracy;
```

### 找出问题最多的场景

```sql
-- 查找评分最低的会话
SELECT session_id, rating, feedback_text 
FROM user_feedback 
WHERE rating <= 2 
ORDER BY timestamp DESC;

-- 查看这些会话的对话内容
SELECT t.session_id, t.user_input, t.functional_judgment
FROM test_logs t
WHERE t.session_id IN (
  SELECT session_id FROM user_feedback WHERE rating <= 2
);
```

## 🐛 故障排查

### 问题 1：前端无法访问

**检查：**
```bash
curl http://localhost:3000
```

**解决：**
```bash
cd behavior_recorder_service/frontend
npm run build
cd ..
./start-all.sh
```

### 问题 2：后端 API 报错

**检查日志：**
```bash
tail -50 server.log
```

**常见错误：**
- LLM API key 错误 → 检查 `.env` 文件
- 数据库锁定 → 删除 `test_logs.db` 重启

### 问题 3：反馈提交失败

**检查：**
```bash
curl http://localhost:8000/api/v4/test/feedback-summary
```

**如果报错：**
```bash
# 查看后端日志
tail -50 server.log | grep feedback

# 检查数据库
sqlite3 test_logs.db ".tables"
```
