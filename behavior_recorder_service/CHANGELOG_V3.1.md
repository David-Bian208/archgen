## 新增功能

### 1. 专家拆解（Expert Breakdown）

在生成核心洞察前，增加专业的"专家拆解"步骤：

```json
{
  "expert_breakdown": {
    "behavior_pattern": "一句话命名该行为模式",
    "functional_hypothesis": "基于 ABC 的专业功能假设",
    "capability_gap": "反映的能力挑战"
  }
}
```

**示例输出**:
```
行为模式：任务参与度不足行为
功能假设：最可能的功能是 escape（逃避），因为孩子通过发呆来减少对早操任务的参与
能力缺口：这可能反映了执行功能方面的挑战，如注意力维持、任务跟随或社交参照能力
```

---

### 2. 五段式报告结构

**新增第二段"专业视角"**：

```
📝 我们看到了什么
[叙述性描述]

🔬 专业视角  ← V3.1 新增
[行为模式 + 功能假设 + 能力缺口]

💡 这可能意味着什么
[核心洞察 - 说进家长心坎里]

🌱 我们可以尝试什么
[基于干预原则的具体建议]

🤔 一个小思考
[开放性问题]
```

---

### 3. 视觉设计

**专家拆解模块**使用独特样式：
- 蓝色渐变背景 (`#e3f2fd` → `#bbdefb`)
- 蓝色边框 (`2px solid #2196f3`)
- 🔬 科学图标
- 与"核心洞察"的紫色渐变区分

---

### 4. 自然过渡语

在对话结束、报告生成前添加：

> "根据我们刚才的交流，我生成了一份简要的分析报告，希望能帮助您更好地理解孩子的行为。"

## 实测案例

### 完整对话流程

```
👤 排队跳操时发呆，老师没干预

💬 谢谢您告诉我这个情况，能注意到孩子在排队跳操时的状态，说明您很细心。
   我们一起来梳理一下。孩子是在什么活动或什么情况下出现这个情况的呢？

👤 幼儿园的早操活动

💬 您能注意到孩子在幼儿园早操活动中的情况，真的很细心。
   当时你观察到孩子具体做了什么，或者没做什么，让你比较在意呢？

👤 他眼睛不跟随老师，站着发呆

💬 您观察得很仔细。
   当孩子那样做的时候，你或周围人（比如老师）第一时间是怎么回应的呢？

👤 老师继续带着其他孩子做操，没有特别干预

💬 谢谢你这么详细的分享。我已经了解了情况，让我为你整理一份观察分析。
   
   根据我们刚才的交流，我生成了一份简要的分析报告，希望能帮助您更好地理解孩子的行为。

✨ 观察分析

📝 我们看到了什么
在幼儿园早操排队时，孩子出现了眼睛不跟随老师、站着发呆的表现。
当时，老师继续带着其他孩子做操，没有进行特别干预。

🔬 专业视角
这是一种"任务参与度不足行为"。基于 ABC 模式，最可能的功能是 escape（逃避）——
孩子通过发呆来减少对早操任务的参与。这可能反映了执行功能方面的挑战，
如注意力维持、任务跟随或社交参照能力。

💡 这可能意味着什么
孩子可能觉得早操有点难跟上，或者不太明白要做什么，所以用发呆来让自己轻松一点。

🌱 我们可以尝试什么
降低任务复杂度并增加个体化支持，如提供视觉提示或简化指令。
例如，可以在孩子面前贴一个简单的步骤图，或者在他看老师时立即给予微笑鼓励。

🤔 一个小思考
下次在类似情况下，如果我们提前给孩子一个简单的视觉提示（比如用手指一下要关注的人或物），
你猜他会有什么不同的反应吗？
```
