# 任务指令模板

**用途：** 规范给 trae 的任务指令，确保 trae 理解准确、执行高效

**重要性：** ⭐⭐⭐⭐⭐ **每次给 trae 任务时都必须使用此模板**

---

## 📋 模板格式

```markdown
# 任务：[任务名称]

## 🎯 目标
[用 1-2 句话清晰描述要做什么]

## 📋 背景
[为什么需要做这个任务，解决了什么问题]

## 🚫 禁令（必须遵守）
1. 禁止 [行为 1]，因为 [原因]
2. 禁止 [行为 2]，因为 [原因]
3. 禁止 [行为 3]，因为 [原因]

## ✅ 要求
1. [具体要求 1]
2. [具体要求 2]
3. [具体要求 3]

## 📄 必读文档
1. [文档 1 路径]
2. [文档 2 路径]

## 📋 验收标准
1. [标准 1]
2. [标准 2]
3. [标准 3]

## 🧪 测试要求
```bash
# 运行测试命令
python3 -m pytest tests/test_xxx.py -v
```

## ⏰ 时间估算
预计时间：[X 分钟/小时]

## 🆘 遇到问题时
1. 查看 [文档路径]
2. 运行 [诊断脚本]
3. 生成 [问题诊断包]
```

## 📝 完整示例

### 示例 1：修复干预建议字段缺失

```markdown
# 任务：补充 Fallback Plan 缺失字段

## 🎯 目标
在 `_generate_fallback_plan()` 方法中添加 `strategy_details` 和 `success_criteria` 字段

## 📋 背景
前端期望干预方案包含 4 个字段（goal, strategy_details, success_criteria, observation_tool），但 Fallback Plan 只返回了 2 个字段，导致前端显示异常

## 🚫 禁令（必须遵守）
1. 禁止不读取文件就直接修改 - 必须先读取 `intervention_planner.py` 第 220-248 行
2. 禁止加没被要求的东西 - 只添加这 2 个字段，不要修改其他部分
3. 禁止编造字段名 - 必须使用前端期望的字段名（`strategy_details`, `success_criteria`）

## ✅ 要求
1. 添加 2 个字段：`strategy_details` 和 `success_criteria`
2. 字段值要有实际内容（不能为空字符串）
3. 保持与其他干预模板一致的格式

## 📄 必读文档
1. `docs/API 契约.md` - 干预方案字段定义
2. `.trae/Claude_Code_禁令清单.md` - 第 2 条禁令

## 📋 验收标准
1. 运行测试，输出包含 `strategy_details` 和 `success_criteria`
2. 不修改其他干预模板
3. Git 提交信息清晰

## 🧪 测试要求
```bash
cd /home/admin/.openclaw/workspace/behavior_recorder_service
python3 -c "
from app.agents.intervention_planner import InterventionPlanner
planner = InterventionPlanner()
plan = planner._generate_fallback_plan({'child_behavior': '测试', 'context': '测试', 'functional_judgment': '测试'})
print('strategy_details:', plan.get('strategy_details', 'MISSING'))
print('success_criteria:', plan.get('success_criteria', 'MISSING'))
"
```

## ⏰ 时间估算
预计时间：10 分钟

## 🆘 遇到问题时
1. 查看 `.trae/问题诊断包.md`
2. 运行 `./scripts/问题诊断包收集.sh "字段缺失"`
3. 生成问题诊断包发送给 OpenClaw
```

### 示例 2：创建新 API 接口

```markdown
# 任务：创建儿童管理 API 接口

## 🎯 目标
创建添加儿童、查看儿童列表、更新儿童信息的 API 接口

## 📋 背景
用户需要管理多个儿童的信息，当前系统只支持单个儿童

## 🚫 禁令（必须遵守）
1. 禁止不读取现有代码就添加新接口 - 必须先查看 `api/endpoints_v4.py` 的现有结构
2. 禁止修改现有的 `/api/v4/analyze` 接口 - 保持向后兼容
3. 禁止硬编码数据库路径 - 必须使用配置中的路径

## ✅ 要求
1. 创建 3 个新接口：`/api/v4/children` (POST/GET), `/api/v4/children/{id}` (PUT/DELETE)
2. 添加数据验证（姓名必填，年龄 0-18 岁）
3. 添加单元测试
4. 更新 API 文档

## 📄 必读文档
1. `TECH_DESIGN.md` - 数据模型定义
2. `docs/API 契约.md` - API 设计规范
3. `AGENTS.md` - 开发规范

## 📋 验收标准
1. 3 个接口都能正常工作
2. 数据验证正确（无效数据返回 400）
3. 单元测试覆盖率>85%
4. API 文档已更新

## 🧪 测试要求

# 运行测试

# 手动测试

## ⏰ 时间估算
预计时间：2 小时

## 🆘 遇到问题时
1. 查看 `docs/常见问题.md`
2. 运行 `./scripts/问题诊断包收集.sh "儿童管理 API"`
3. 生成问题诊断包
```

## 🎯 使用指南

### 什么时候使用此模板？

**必须使用的场景：**
- ✅ 新功能开发
- ✅ Bug 修复
- ✅ 代码重构
- ✅ 性能优化

**可以简化的场景：**
- ⏳ 简单文字修改（只用目标和验收标准）
- ⏳ 紧急 Hotfix（只用目标、禁令、测试）

### 各部分说明

| 部分 | 重要性 | 说明 |
|------|--------|------|
| 目标 | ⭐⭐⭐⭐⭐ | 必须清晰，1-2 句话 |
| 背景 | ⭐⭐⭐⭐ | 帮助 trae 理解为什么做 |
| 禁令 | ⭐⭐⭐⭐⭐ | 必须写清楚，每条都有原因 |
| 要求 | ⭐⭐⭐⭐⭐ | 具体、可执行 |
| 必读文档 | ⭐⭐⭐⭐ | 避免 trae 遗漏关键信息 |
| 验收标准 | ⭐⭐⭐⭐⭐ | 明确完成标准 |
| 测试要求 | ⭐⭐⭐⭐⭐ | 必须提供测试命令 |
| 时间估算 | ⭐⭐⭐ | 帮助跟踪进度 |
| 遇到问题时 | ⭐⭐⭐ | 提供求助路径 |

## 📊 效果对比

### 不使用模板（低效）

```
你：修复干预建议的问题

trae：好的，我看看...
[trae 修改了 10 个文件，引入了新的 Bug]
你：不对，不是这样修...
[来回 5 轮，问题仍未解决]
```

### 使用模板（高效）

```
你：[使用任务指令模板]

trae：[阅读模板，理解目标、禁令、要求]
[trae 只修改了 1 个文件，按验收标准测试]
trae：✅ 完成，测试通过
你：✅ 验证通过
```

**效率提升：**
- 沟通轮次：5 轮 → 1 轮
- 修复时间：30 分钟 → 10 分钟
- 挫败感：高 → 低
