## 🚫 禁止盲从

**禁令：**
> ❌ 禁止盲目执行任务指令，不核实实际情况

**原因：**
- AI（包括 OpenClaw）可能不了解软件的真实情况
- AI 可能不知道历史遗留问题
- AI 可能不理解业务逻辑
- 盲从可能导致严重后果

**正确做法：**
1. 接收任务指令
2. **核实实际情况**（必须执行）
3. 如有疑问，**立即提出**（必须执行）
4. 确认无误后，再执行

## ✅ 核实清单

**接收任务指令后，必须核实：**

### 1. 技术可行性

- [ ] 这个修改在技术上是否可行？
- [ ] 是否会影响其他功能？
- [ ] 是否有技术债务或历史原因？
- [ ] 是否需要查看相关代码？

**核实方法：**
```bash
# 1. 查看相关代码
cat [文件路径]

# 2. 查看 Git 历史
git log -p [文件路径]

# 3. 查看相关测试
cat tests/test_xxx.py

# 4. 运行测试验证
python3 -m pytest tests/test_xxx.py -v
```

---

### 2. 业务逻辑

- [ ] 这个修改是否符合业务逻辑？
- [ ] 是否违反临床系统要求？
- [ ] 是否影响用户体验？
- [ ] 是否需要咨询 DAVID？

**核实方法：**
```markdown
# 1. 查看业务文档
- PRD.md
- TECH_DESIGN.md
- docs/API 契约.md

# 2. 查看临床要求
- AGENTS.md（临床系统特殊要求）
- .trae/Claude_Code_禁令清单.md（临床禁令）

# 3. 如有疑问，立即提问
🆘 需要确认：[问题描述]
```

---

### 3. 端口、路径、配置

- [ ] 端口号是否正确？（如 8003 vs 8000）
- [ ] 路径是否正确？（如实际路径 vs 文档路径）
- [ ] 配置是否正确？（如环境变量）

**核实方法：**
```bash
# 1. 查看实际运行的服务
ps aux | grep python
ps aux | grep node

# 2. 查看配置文件
cat config.yaml
cat .env

# 3. 查看实际端口
netstat -tlnp | grep python
netstat -tlnp | grep node
```

---

### 4. 依赖关系

- [ ] 是否会影响其他模块？
- [ ] 是否有依赖关系？
- [ ] 是否需要同时修改其他部分？

**核实方法：**
```bash
# 1. 搜索引用
grep -r "函数名" --include="*.py"
grep -r "组件名" --include="*.vue"

# 2. 查看依赖
cat requirements.txt
cat frontend/package.json

# 3. 运行完整测试
python3 -m pytest tests/ -v
```

## 📝 反驳示例

### 示例 1：端口号错误

```markdown
⚠️ 需要澄清：修改 API 端口

**指令内容：**
"修改 API 端口从 8003 改为 8000"

**核实结果：**
- 当前实际运行端口：8003
- 查看 config.yaml：port 配置为 8003
- 查看 Git 历史：8003 是有意设置的（避免与 8000 冲突）

**发现的问题：**
1. 8000 端口可能被其他服务占用
2. 8003 是 DAVID 特意选择的端口

**建议：**
1. 保持 8003 端口不变
2. 如果确实需要修改，请先咨询 DAVID

**请确认：**
- [ ] 继续执行原指令（改为 8000）
- [ ] 按建议修改指令（保持 8003）
- [x] 需要进一步讨论
```

---

```markdown

**建议：**

2. 可以优化措辞，使其更简洁

**请确认：**
- [ ] 继续执行原指令（删除）
- [x] 按建议修改指令（保留并优化）
- [ ] 需要进一步讨论
```

### 示例 3：简化复杂逻辑

```markdown
⚠️ 需要澄清：简化干预规划器逻辑

**指令内容：**
"为什么这里用这么复杂的逻辑？简化它"

**核实结果：**
- 查看 Git 历史：这个逻辑是为了解决 V4.5.21 的 P0 Bug
- 查看测试：test_p0_regression.py 专门测试这个逻辑

**发现的问题：**
1. 这个逻辑是有意设计的，不是过度设计
2. 之前简化过，导致了诊断 - 干预断裂（10% 错误率）
3. V4.5.21 P0 Fix 专门修复了这个问题

**建议：**
1. 保持现有逻辑不变
2. 可以添加更多注释说明原因

**请确认：**
- [ ] 继续执行原指令（简化）
- [x] 按建议修改指令（保持不变，添加注释）
- [ ] 需要进一步讨论
```
