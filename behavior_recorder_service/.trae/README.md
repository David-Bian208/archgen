##  17 条禁令（必须遵守）

**核心禁令（前 5 条必须牢记）：**

1. ❌ **禁止不读取文件就直接修改** - 因为会凭幻觉修改
2. ❌ **禁止加没被要求的东西** - 因为会引入 Bug
3. ❌ **禁止编造或预测结果** - 因为会产生幻觉
4. ❌ **禁止硬编码敏感信息** - 因为会泄露密钥
5. ❌ **禁止忽略异常** - 因为会隐藏问题

**完整清单：** `.trae/Claude_Code_禁令清单.md`

**违反后果：**
- ⚠️ 代码审查不通过
- ❌ 必须重新修改
- 📝 记录到故障案例库

## 📝 接收任务流程

### 步骤 1：等待任务指令

**DAVID 会使用任务指令模板给你发送任务：**

```markdown
# 任务：[任务名称]

## 🎯 目标
## 📋 背景
## 🚫 禁令（必须遵守）
## ✅ 要求
## 📄 必读文档
## 📋 验收标准
## 🧪 测试要求
```

**你的响应：**
```markdown
🛠️ 开始任务：[任务名称]

**预计时间：** [X 分钟]

**步骤：**
1. [步骤 1]
2. [步骤 2]
3. [步骤 3]
```

---

### 步骤 2：执行任务

**执行顺序：**
1. 阅读任务指令
2. 阅读必读文档
3. 理解禁令和要求
4. 开始执行

**沟通格式：**
```markdown
🕐 任务进行中

**当前进度：** X%

**已完成：**
- [完成项]

**进行中：**
- [当前正在做的]

**预计完成：** [时间]
```

---

### 步骤 3：提交代码

**提交前必须：**

1. **填写审查清单** - `.trae/代码审查清单.md`
2. **运行测试** - P0 测试必须 100% 通过
3. **更新文档** - 如有 API 变更

**提交格式：**
```markdown
✅ 任务完成：[任务名称]

**修改文件：**
- [文件路径] - [修改内容]

**测试结果：**
- P0 测试：✅ 通过（17/17）

**代码审查：**
✅ 全部通过（见审查清单）

**Git 提交：**
[提交信息]

**请审查：**
- [ ] 代码质量
- [ ] 测试结果
- [ ] 文档更新
```

## 🆘 遇到问题时

### 问题分级

**P0 - 紧急且重要：**
- 无法运行项目
- 无法理解核心逻辑
- 违反临床禁令

**处理：** 立即求助 OpenClaw

---

**P1 - 重要不紧急：**
- 代码审查未通过
- 测试失败
- 不理解需求

**处理：** 查看文档 → 尝试解决 → 求助

---

**P2 - 日常问题：**
- 小 Bug
- 代码优化
- 文档更新

**处理：** 自行解决 → 提交审查

---

### 求助格式

```markdown
🆘 需要帮助：[问题描述]

**问题级别：** P0/P1/P2

2. [方案 2] - [结果]

**错误信息：**
```
[粘贴错误信息]
```

**需要帮助：**
[具体需求]
```
