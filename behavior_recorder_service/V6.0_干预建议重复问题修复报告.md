## 📋 问题描述

### 验证失败的案例

| 案例 | 能力缺口 | 期望干预 | 实际干预 | 状态 |
|------|----------|----------|----------|------|
| 玥玥 | 社交信号监测弱和观点采择应用困难 | 社交信号侦探游戏 | ❌ 我们开始玩了吗信号练习 | 错误 |
| OK | 观点采择能力：难以理解他人视角 | 视角游戏：妈妈会看到什么 | ❌ 我们开始玩了吗信号练习 | 错误 |

### 问题现象

两个不同能力缺口的案例得到了**完全相同的干预建议**，违背了"不同能力缺口→不同干预"的核心原则。

---

## 🔍 根因分析

### 问题文件
`behavior_recorder_service/app/agents/intervention_planner.py`

### 问题方法
`_generate_social_skills_plan()`

### 根因

1. **没有优先使用 `capability_gap` 字段**
   - 原实现优先检查 `child_behavior` 中的关键词
   - 导致不同能力缺口被错误匹配到相同干预

2. **场景匹配条件过于宽泛**
   - 如 `"以为" in child_behavior` 或 `"监测" in child_behavior`
   - 玥玥和 OK 都有"以为"，都被匹配到"我们开始玩了吗信号练习"

3. **缺乏差异化干预逻辑**
   - 没有为 6 种能力缺口设计不同的干预游戏
   - 干预建议变成"万金油"，缺乏精准性

---

## ✅ 修复方案

### 1. 能力缺口优先匹配

在 `_generate_social_skills_plan()` 中实现以下匹配顺序：

```python
if capability_gap:
    # 1. 社交信号监测（优先检查）
    if any(kw in capability_gap for kw in ["社交信号", "监测", "观察反应", "眼神接触"]):
        game_name = "社交信号侦探游戏"
    
    # 2. 观点采择
    elif any(kw in capability_gap for kw in ["观点采择", "换位思考", "视角转换", "心理理论"]):
        game_name = "视角游戏：妈妈会看到什么"
    
    # 3. 工作记忆
    elif any(kw in capability_gap for kw in ["工作记忆", "记不住", "忘记步骤", "多步骤"]):
        game_name = "视觉提示卡游戏"
    
    # 4. 认知灵活性
    elif any(kw in capability_gap for kw in ["认知灵活性", "灵活性", "变化", "转换"]):
        game_name = "规则变变变游戏"
    
    # 5. 情绪识别
    elif any(kw in capability_gap for kw in ["情绪识别", "表情", "面部表情", "情绪"]):
        game_name = "情绪小侦探游戏"
    
    # 6. 共同调控
    elif any(kw in capability_gap for kw in ["共同调控", "同步", "节奏匹配", "互动"]):
        game_name = "我们开始玩了吗信号练习"
    
    else:
        # 降级到场景匹配
        game_name, game_steps, why_effective = self._match_social_scene(child_behavior)
```

### 2. 新增辅助方法

新增 `_match_social_scene()` 方法作为降级逻辑：

```python
def _match_social_scene(self, child_behavior: str) -> tuple[str, str, str]:
    """
    V6.0 新增：场景匹配辅助方法（降级用）
    
    当 capability_gap 不明确时，基于 child_behavior 进行场景匹配
    """
    # 身体边界
    if "拥抱" in child_behavior or "轻重" in child_behavior:
        return "轻柔的手练习游戏", "...", "..."
    
    # 规则理解
    elif "规则" in child_behavior or "僵化" in child_behavior:
        return "规则变变变游戏", "...", "..."
    
    # 介绍/任务化
    elif "介绍" in child_behavior or "任务化" in child_behavior:
        return "说完看脸游戏", "...", "..."
    
    # 轮流等待
    elif "轮流" in child_behavior or "等待" in child_behavior:
        return "轮流当队长游戏", "...", "..."
    
    else:
        return "社交技能假装游戏", "...", "..."
```

### 3. 6 种能力缺口→6 种不同干预

| 能力缺口 | 干预游戏 | 核心策略 |
|----------|----------|----------|
| 社交信号监测 | 社交信号侦探游戏 | 看动画片暂停提问、制作表情卡片、录像回放分析 |
| 观点采择 | 视角游戏：妈妈会看到什么 | 薯片盒游戏、逐步升级、录像回放 |
| 工作记忆 | 视觉提示卡游戏 | 制作步骤卡、按顺序排列、指卡提示 |
| 认知灵活性 | 规则变变变游戏 | 提前预告、微小变化、提供选择权 |
| 情绪识别 | 情绪小侦探游戏 | 制作表情卡片、看照片猜感受、真实情境引导 |
| 共同调控 | 我们开始玩了吗信号练习 | 设定专属信号、家庭演练、暂停重启 |
