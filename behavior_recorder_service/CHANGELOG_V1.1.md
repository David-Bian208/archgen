## 更新概述

V1.1 版本专注于提升行为记录员 Agent 的**专业判断准确性、稳定性和深度**，使其更像一位严谨的 BCBA 实习生，而不仅仅是一个关键词匹配工具。

---

## 核心变更

### 1. 步骤一：结构化提取 Prompt 优化

#### 问题
- 原 Prompt 可能导致格式混乱的输出（如 HTML 代码）
- 对"未提及"信息处理不佳

#### 解决方案
**新 System Prompt**:
```
你是一名严谨的儿童行为观察记录员。你的任务是从描述中提取精确的信息。
请确保输出纯净的文本，不包含任何 markdown、代码或额外格式。
```

**新 User Prompt**:
```
请分析以下关于儿童行为的描述：

"{description}"

请严格按照以下 JSON 格式输出，且仅输出此 JSON 对象：
{
    "antecedent": "行为发生前，孩子所处的具体情境、被给予的指令或活动。如果描述中未明确提及，则填入'未明确提及'。",
    "behavior": "孩子表现出的具体、可观察的动作或非动作（例如：跑开、哭闹、发呆、沉默）。请用简洁的短语描述。",
    "consequence": "行为发生后，他人（如老师、家长）的直接反应，或环境/任务的直接变化。如果描述中未明确提及，则填入'未明确提及'。"
}
```

#### 代码调整
- 使用 `json.loads()` 确保获得结构化字典
- 检查键值是否存在，缺失时使用"未明确提及"填充

---

### 2. 步骤二：功能假设 Prompt 优化（引入推理链）

#### 问题
- 原判断逻辑简单
- 对"逃避"和"自动强化"的区分不准
- 缺乏推理过程

#### 解决方案
**新 System Prompt**:
```
你是一名经验丰富的应用行为分析（BCBA）专家。
你的任务是根据行为 ABC 数据，遵循标准的决策流程，做出最合理的功能假设。
请逐步思考。
```

**新 User Prompt**（推理链）:
```
你正在分析一个行为案例。请严格遵循以下思考步骤：

【已知信息】
- 前因 (A): `{antecedent}`
- 行为 (B): `{behavior}`
- 后果 (C): `{consequence}`

【推理步骤】
1. **检查实物获取**：行为是否为了获得某个具体物品、食物或活动？后果是否为"获得了该物"？如果是，功能是 **tangible**。
2. **检查关注获取**：行为是否为了获得他人的注意（包括安慰、责备、对话）？后果是否为"他人给予了社交性反应"？如果是，功能是 **attention**。
3. **检查逃避**：前因是否包含"要求"、"任务"、"复杂情境"？后果是否为"任务/要求被取消、延迟或减轻"？如果是，功能是 **escape**。
4. **检查自动强化**：行为是否是重复的、刻板的，且**看起来**与当前社交环境无关（如摇晃、看手、喃喃自语），其后果通常不易被外部观察？如果是，功能是 **automatic**。
5. **特殊情况**：如果以上均不符合，或信息不足（如"未明确提及"），则功能为 **inconclusive**（无法确定）。

【你的输出】
请以以下 JSON 格式输出你的最终结论和简要推理：
{
    "hypothesized_function": "tangible | attention | escape | automatic | inconclusive",
    "reasoning": "一句话解释，说明是哪个推理步骤最关键。"
}
```

#### 代码调整
- 解析 LLM 返回的 JSON，读取 `hypothesized_function` 和 `reasoning` 字段
- 更新 API 响应格式，包含推理过程

---

### 3. 新增功能：inconclusive（无法确定）

当信息不足或不符合标准功能分类时，返回 `inconclusive`，避免强行判断。

---

### 4. 错误处理增强

- 两步 LLM 调用之间做好错误处理
- 第一步解析失败时，直接返回错误，不继续第二步
- 对 LLM 返回的非 JSON 或无效响应进行捕获
- 记录日志，返回用户友好的错误信息

## API 响应格式变更

### V1.0 响应
```json
{
  "antecedent": "...",
  "behavior": "...",
  "consequence": "...",
  "hypothesized_function": "tangible"
}
```

### V1.1 响应（新增 reasoning 字段）
```json
{
  "antecedent": "...",
  "behavior": "...",
  "consequence": "...",
  "hypothesized_function": "tangible",
  "reasoning": "行为后获得了想要的物品，符合实物获取特征。"
}
```

## 实测案例对比

### 案例 1：实物获取
**输入**: "不给他手机，他就打自己头，我赶紧把手机给他了。"

**V1.1 输出**:
```json
{
  "antecedent": "未明确提及",
  "behavior": "打自己头",
  "consequence": "家长把手机给他了",
  "hypothesized_function": "tangible",
  "reasoning": "行为后家长给了手机，符合步骤 1 中'获得了具体物品'的特征，表明功能可能是获取实物。"
}
```

### 案例 2：逃避（模糊后果）
**输入**: "在教室排队跳操时，他眼睛不跟随老师，站着发呆。"

**V1.1 输出**:
```json
{
  "antecedent": "在教室排队跳操时",
  "behavior": "眼睛不跟随老师，站着发呆",
  "consequence": "未明确提及",
  "hypothesized_function": "inconclusive",
  "reasoning": "后果未明确提及，无法根据标准步骤确定功能，信息不足。"
}
```

### 案例 3：关注（负面）
**输入**: "我在工作时，他不停跑来拍我笔记本，我每次都说别吵。"

**V1.1 输出**:
```json
{
  "antecedent": "未明确提及",
  "behavior": "跑来拍笔记本",
  "consequence": "被说'别吵'",
  "hypothesized_function": "attention",
  "reasoning": "行为后获得他人社交反应（被说'别吵'），符合关注获取特征。"
}
```

### 案例 4：逃避（明确）
**输入**: "要求他写作业，他就哭闹并逃跑，妈妈只好让他先去玩。"

**V1.1 输出**:
```json
{
  "antecedent": "要求他写作业",
  "behavior": "哭闹并逃跑",
  "consequence": "妈妈让他先去玩",
  "hypothesized_function": "escape",
  "reasoning": "前因为任务要求，后果为任务中断，符合逃避特征。"
}
```
