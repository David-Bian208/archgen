## 准备工作

### 1. 注册微信小程序账号

**网址：** https://mp.weixin.qq.com/

**所需材料：**
- 邮箱（未注册过微信公众平台）
- 身份证（个人主体）或 营业执照（企业主体）
- 手机号

**步骤：**
1. 打开 https://mp.weixin.qq.com/
2. 点击"立即注册"
3. 选择"小程序"
4. 填写邮箱、密码
5. 激活邮箱（登录邮箱点击激活链接）
6. 填写主体信息（个人/企业）
7. 填写管理员信息
8. 完成注册

**费用：**
- 个人主体：**免费**
- 企业主体：**300 元/年**（认证费用）

---

### 2. 获取 AppID

注册完成后：
1. 登录小程序后台
2. 进入 **开发 → 开发管理 → 开发设置**
3. 找到 **AppID(小程序 ID)**
4. 复制保存（后续配置需要）

## 代码打包

### 方案 A：使用微信开发者工具（推荐）

**1. 下载微信开发者工具**

网址：https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html

**2. 导入项目**

1. 打开微信开发者工具
2. 登录（用小程序管理员微信扫码）
3. 点击"+" 添加项目
4. 选择项目目录：`/home/admin/.openclaw/workspace/behavior_recorder_service/frontend`
5. 填写 AppID
6. 点击"确定"

**3. 配置项目**

在项目根目录创建 `project.config.json`：

```json
{
  "description": "行为观察伙伴",
  "packOptions": {
    "ignore": []
  },
  "setting": {
    "urlCheck": true,
    "es6": true,
    "enhance": true,
    "postcss": true,
    "preloadBackgroundData": false,
    "minified": true,
    "newFeature": true,
    "coverView": true,
    "nodeModules": false,
    "autoAudits": false,
    "showShadowRootInWxmlPanel": true,
    "scopeDataCheck": false,
    "uglifyFileName": false,
    "checkInvalidKey": true,
    "checkSiteMap": true,
    "uploadWithSourceMap": true,
    "compileHotReLoad": false,
    "babelSetting": {
      "ignore": [],
      "disablePlugins": [],
      "outputPath": ""
    }
  },
  "appid": "你的 AppID",
  "projectname": "behavior-recorder",
  "libVersion": "2.19.4",
  "condition": {}
}
```

**4. 预览/上传**

1. 点击工具栏"预览" → 生成二维码
2. 用手机微信扫码 → 在手机上测试
3. 点击"上传" → 填写版本号和备注 → 上传到微信后台

---

### 方案 B：使用 uni-app（如果项目是 uni-app）

如果你的项目是用 uni-app 开发的：

**1. 安装 HBuilderX**

网址：https://www.dcloud.io/hbuilderx.html

**2. 导入项目**

1. 打开 HBuilderX
2. 文件 → 导入 → 从本地目录导入
3. 选择项目目录

**3. 配置 manifest.json**

在 `manifest.json` 中填写：
- AppID（从微信后台获取）
- 版本信息
- 权限配置

**4. 发布到小程序**

1. 发行 → 小程序 - 微信
2. 填写 AppID
3. 点击"发行"
4. 生成小程序代码包
5. 用微信开发者工具打开并上传

## 提交审核

### 1. 登录小程序后台

https://mp.weixin.qq.com/

### 2. 版本管理

1. 进入 **版本管理**
2. 看到刚才上传的开发版本
3. 点击"提交审核"

### 3. 填写审核信息

**需要填写：**
- 功能介绍（100 字以内）
- 服务类目（选择"教育 - 教育工具"或"医疗 - 健康管理"）
- 测试账号（如果需要登录）
- 补充说明（可选）

**功能介绍示例：**
```
行为观察伙伴是一款帮助家长和教育工作者记录、分析儿童行为的工具。
基于 ABC 行为分析理论，提供专业的行为观察记录和干预建议。
```

**服务类目选择：**
- 主类目：教育
- 子类目：教育工具

### 4. 隐私政策（必须！）

**需要上传隐私政策链接**

可以用以下模板修改：

```markdown
# 行为观察伙伴 隐私政策

## 我们如何收集和使用您的信息

1. **您主动提供的信息**
   - 儿童行为描述（文本）
   - 儿童年龄、性别等背景信息
   
2. **我们如何使用**
   - 用于生成行为分析报告
   - 用于改进 AI 模型质量
   
3. **数据存储**
   - 数据存储在加密服务器
   - 不向第三方出售或共享数据
   
4. **数据安全**
   - 采用 HTTPS 加密传输
   - 数据库加密存储
   
5. **您的权利**
   - 您可以随时删除自己的数据
   - 可以联系我们导出数据

## 联系我们

如有隐私相关问题，请联系：your-email@example.com
```

**托管方式：**
- 放在项目官网
- 或用腾讯文档生成公开链接

### 5. 提交审核

确认信息无误后，点击"提交审核"

**审核时间：**
- 通常 1-3 个工作日
- 节假日可能延长

## 发布上线

### 1. 审核通过

审核通过后，会收到微信通知（公众号消息）

### 2. 发布

1. 登录小程序后台
2. 进入 **版本管理**
3. 看到"审核版本"
4. 点击"发布"

### 3. 设置可搜索

发布后默认不可搜索，需要手动设置：

1. 进入 **设置 → 基本设置**
2. 找到"搜索设置"
3. 开启"允许用户搜索到"

### 4. 获取小程序码

1. 进入 **设置 → 基本设置**
2. 找到"小程序码及线下物料下载"
3. 下载小程序码

## 测试版发布（推荐先做这个！）

### 为什么先做测试版？

- ✅ 无需审核或快速审核
- ✅ 可以邀请指定用户测试
- ✅ 方便快速迭代
- ✅ 发现问题后再正式发布

### 方式一：体验版（推荐）

**1. 上传代码**

用微信开发者工具上传代码（前面已讲）

**2. 设置为体验版**

1. 登录小程序后台
2. 进入 **版本管理**
3. 找到开发版本
4. 点击"选为体验版"

**3. 添加体验者**

1. 进入 **成员管理 → 体验成员**
2. 点击"添加"
3. 输入体验者的微信号
4. 保存

**4. 分享体验版**

1. 进入 **版本管理**
2. 点击体验版"详情"
3. 生成体验版二维码
4. 发送给体验者

**体验者使用方式：**
- 微信扫码 → 打开体验版小程序
- 可以正常使用所有功能
- 体验者不需要是开发者

### 方式二：开发版

**开发版 vs 体验版区别：**

| 特性 | 体验版 | 开发版 |
|------|--------|--------|
| 用户数 | 最多 15 人 | 不限 |
| 审核 | 无需 | 无需 |
| 适合 | 内部测试 | 公开内测 |

**开发版发布流程：**

1. 上传代码
2. 登录小程序后台
3. 版本管理 → 开发版本 → 详情
4. 生成开发版二维码
5. 分享给测试用户

## 小程序审核常见问题

### Q1: 个人主体能做什么类目？

**个人主体可选类目：**
- 工具 - 效率
- 教育 - 教育工具
- 医疗 - 健康管理（需要资质）

**建议：** 选择"教育 - 教育工具"

### Q2: 审核被拒怎么办？

**常见原因：**
1. 功能与类目不符 → 修改类目或功能介绍
2. 隐私政策缺失 → 补充隐私政策
3. 有 bug 无法使用 → 修复后重新提交
4. 涉及医疗建议 → 修改表述为"仅供参考"

**解决方法：**
1. 查看审核反馈
2. 按要求修改
3. 重新提交（无需重新上传代码）

### Q3: 需要办理 ICP 许可证吗？

**不需要。**

小程序由微信平台管理，不需要单独的 ICP 许可证。

## 测试数据收集

### 1. 查看测试日志

**API 接口：** `GET /api/v4/test/logs`

**示例：**
```bash
curl http://localhost:8000/api/v4/test/logs
```

**返回：**
```json
{
  "status": "success",
  "count": 10,
  "data": [
    {
      "session_id": "abc123",
      "user_input": "孩子在学校...",
      "ai_response": {...},
      "timestamp": "2026-03-19T18:00:00Z"
    }
  ]
}
```

### 2. 查看反馈汇总

**API 接口：** `GET /api/v4/test/feedback-summary`

**示例：**
```bash
curl http://localhost:8000/api/v4/test/feedback-summary
```

**返回：**
```json
{
  "status": "success",
  "data": {
    "rating_distribution": {"5": 10, "4": 5, "3": 2},
    "accuracy_distribution": {"accurate": 12, "partial": 5},
    "average_rating": 4.5,
    "total_feedback": 17
  }
}
```

### 3. 导出数据

**API 接口：** `POST /api/v4/test/export`

**示例：**
```bash
curl -X POST "http://localhost:8000/api/v4/test/export?output_path=my_data.csv"
```

**导出文件位置：** `/home/admin/.openclaw/workspace/behavior_recorder_service/test_logs_export.csv`
