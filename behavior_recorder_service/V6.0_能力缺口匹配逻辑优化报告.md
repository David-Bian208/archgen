## 📋 问题描述

### 当前问题

能力缺口匹配是简单的 if-else 关键词匹配，只匹配"标准术语"，不理解用户的自然语言描述。

| 用户描述 | 当前匹配 | 期望匹配 | 状态 |
|---------|---------|---------|------|
| "看不懂眼神" | ❌ 无匹配 | 社交信号监测 | 失败 |
| "总觉得别人应该懂他" | ❌ 无匹配 | 观点采择 | 失败 |
| "多步骤记不住" | ❌ 无匹配 | 工作记忆 | 失败 |
| "路线变了就崩溃" | ❌ 无匹配 | 认知灵活性 | 失败 |

### 根因

当前实现只匹配"标准术语"（如"社交信号监测"），不理解用户的自然语言描述（如"看不懂眼神"）

### 2. 新增匹配方法

**`_match_capability_keywords()`** - V6.0 P0 新增：关键词扩展匹配

```python
def _match_capability_keywords(self, capability_gap: str) -> Optional[str]:
    """
    V6.0 P0 新增：关键词扩展匹配
    
    使用同义词库匹配能力缺口，支持自然语言描述
    
    Args:
        capability_gap: 能力缺口描述（支持自然语言）
        
    Returns:
        干预游戏名称，如果无匹配则返回 None
    """
    for capability, synonyms in CAPABILITY_SYNONYMS.items():
        if any(syn in capability_gap for syn in synonyms):
            return INTERVENTION_MAP[capability]
    return None
```
