# 待办：能力缺口分析增加"发展阶段定位"

**优先级：** P0（专业深度升级）  
**分配给：** TRAE（小治）  
**时间：** 2026-04-08 18:25  
**背景：** 从"缺陷标签"转向"发展阶段定位"

---

## 问题描述

### 当前问题
**能力缺口分析太笼统：**
```
• 认知灵活性不足：这涉及到执行功能中的工作记忆和认知切换能力...
```

**缺少：**
- ❌ 孩子已掌握什么能力
- ❌ 具体卡在哪个规则/概念
- ❌ 处于什么发展阶段（如"从 X 向 Y 过渡"）

### 用户需求
> "执行功能是一个长期问题。他其实也会分不同的阶段。我觉得这里可能要说一下他当下是属于什么样的一个阶段，什么样的一个情况。"

---

## 修改方案

### 修改文件
`app/agents/insight_analyzer.py` - 系统提示词中的"能力缺口分析"部分

---

### 当前提示词（需要修改）
```python
3. **能力缺口分析**（从现象到生理/神经归因）
- 格式：每个缺口必须包含三层结构
  1. **行为表现**：极简，5-10 字以内
  2. **神经/发展归因**：解释**为什么**会有这个表现...
  3. **正常化 + 方向**：说明不是缺陷，指出练习方向
```

---

### 修改后提示词
```python
3. **能力缺口分析**（发展性评估，非缺陷标签）

请为每个能力缺口按以下结构组织（**必须包含三部分**）：

**1. 神经/发展归因**（解释为什么）
- 简要说明相关的脑区/认知功能（如"颞顶联合区"、"工作记忆"）
- 典型发展时间线参照（如"4-6 岁快速发育中"）

**2. 当前发展阶段定位**（核心新增）
- **已掌握的能力**：孩子已经能做到的具体能力（参考发展里程碑）
  - 示例："OK 已能理解'每个人知道的信息可能不同'（一级错误信念）"
- **具体的发展卡点**：卡在哪个具体规则、概念或处理步骤
  - 示例："但卡在'视角由位置决定'这条具体规则上——她应用了简单规则'妈妈总是看到电脑'"
- **发展阶段描述**：处于哪个发展过渡阶段
  - 示例："这恰恰是从'理解知识状态不同'迈向'掌握复杂视角采择'的关键过渡阶段"

**3. 支持方向**（指出去哪里）
- 下一阶段可期待的发展
- 具体的能力建设方向
  - 示例："干预重点是用身体化、游戏化的方式，帮她发现并内化'位置决定视角'这条规则"

**输出格式要求：**
- 用清晰的分点格式（如示例所示）
- 避免专业术语堆砌
- 结合具体观察证据
- 每个能力缺口总长度控制在 300-400 字以内

**示例输出：**
> **观点采择能力弱**：
>
> **发展归因**：这与心理理论（Theory of Mind）发展节奏有关。大脑中负责理解他人心理状态的区域（颞顶联合区）在 4-6 岁还在快速发育中。
>
> **当前阶段**：
> - **已掌握**：OK 已能理解"每个人知道的信息可能不同"（如知道爸爸打开过盒子，所以知道里面是糖）
> - **卡点**：但卡在"视角由位置决定"这条具体规则上——她应用了简单规则"妈妈总是看到电脑"，而非"坐在电脑椅的人会看到电脑"
> - **发展阶段**：这恰恰是从"理解知识状态不同"迈向"掌握复杂视角采择"的关键过渡阶段（4-6 岁典型发展）
>
> **支持方向**：干预重点是用身体化、游戏化的方式，帮她发现并内化"位置决定视角"这条规则（如"指挥家"游戏、"照相馆"游戏），逐步从"人物 - 物体"固定链接转向"位置 - 视角"动态链接。
```

## 测试要求

### 测试用例
修改后，用以下 3 个案例测试：

1. **薯片盒游戏案例**（观点采择）
   ```
   场景：在家玩小游戏，先玩薯片盒装糖果的游戏，然后问妈妈和爸爸交换位置后看到什么的问题
   孩子的表现：她以为我看到盒子里还是糖，不太理解我看到的是薯片的意思。她就不太明白了，还是以为妈妈是在用电脑，爸爸在用手机。
   ```

2. **抓人游戏案例**（感官/逃避）
   ```
   场景：幼儿园户外活动时间
   孩子的表现：小朋友要抓他，他就跑开，躲在滑梯后面
   ```

3. **排队推人案例**（社交技能）
   ```
   场景：幼儿园排队洗手
   孩子的表现：直接推前面小朋友，想插队
   ```

- ✅ "已掌握的能力"（具体里程碑）
- ✅ "具体的发展卡点"（非笼统描述）
- ✅ "发展阶段描述"（如"从 X 向 Y 过渡"）
- ✅ 输出格式稳定，无 Markdown 错误
- ✅ 现有功能不受影响（其他报告部分正常）
