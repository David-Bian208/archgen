## 🏗️ 系统架构

### 当前架构（流水线模式）

```
用户输入 → [意图检测] → [功能判断] → [归因] → [叙事解读] → 报告
```

**问题：** 模块孤立，信息流单向，无校验机制

**典型问题：**
- 功能判断误判为"逃避"，但叙事解读是"心理理论发展"
- 报告内部"精神分裂"，家长困惑

### 目标架构（推理引擎模式）

```
用户输入 → [假设生成] → [证据审查] → [得出结论] → [一致性输出] → 报告
                     ↑
                     └── 逐一验证/排除竞争性假设
```

**核心升级：**
1. **假设生成** - 列出 4 种竞争性假设
2. **证据审查** - 用证据逐一验证/排除
3. **得出结论** - 选择证据最强的假设
4. **一致性输出** - 功能判断、归因、解读一致

## 🔧 核心开发原则

### 1. 临床安全 > 功能完成

**原则：**
- ❌ 不输出可能误导家长的判断（如误判"逃避"）
- ✅ 证据不足时说"需要进一步观察"
- ✅ 优先使用发展性语言（"发展中的挑战"）而非功能标签（"逃避"）

**案例：**
```
❌ 错误：功能判断 = "提示依赖下的逃避"（孩子主动参与，无逃避证据）
✅ 正确：功能判断 = "需要进一步观察" 或 "观点采择能力发展中的挑战"
```

---

### 2. 发展性评估 > 缺陷标签

**原则：**
- ❌ 不说"障碍"、"缺陷"、"不足"
- ✅ 说"发展中的挑战"、"还没学会"、"需要练习"
- ✅ 先描述"已掌握的能力"，再谈"卡点"

**案例：**
```
❌ 错误：观点采择能力不足
✅ 正确：OK 已能理解"每个人知道的信息可能不同"（已掌握），
        但卡在"视角由位置决定"这条具体规则上（卡点），
        这恰恰是从"理解知识状态不同"迈向"掌握复杂视角采择"的关键过渡阶段（发展阶段）
```

---

### 3. 具体卡点 > 笼统归因

**原则：**
- ❌ 不说"执行功能不足"、"工作记忆有限"
- ✅ 说"卡在哪个具体规则/概念/步骤"
- ✅ 用发展里程碑描述当前位置（如"4-5 岁典型发展"）

**案例：**
```
❌ 错误：认知灵活性不足，工作记忆容量有限
✅ 正确：OK 目前能同时保持 2 个信息（如"妈妈在用电脑"），
        但当情境变化后（交换位置），还很难快速更新这 2 个信息，
        思维容易卡在初始状态。同龄孩子（5 岁）通常能同时处理 3-4 个信息并灵活切换。
```

---

### 4. 证据审查 > 条件反射

**原则：**
- ❌ 不基于关键词直接判断（如"不明白"→"逃避"）
- ✅ 列出竞争性假设，用证据逐一验证
- ✅ 有排除证据时必须排除

**案例：**
```
验证"提示依赖下的逃避"：
- ✅ 支持证据：孩子说"我不会"、等待成人提示、回避任务
- ❌ 排除证据：孩子主动参与、尝试回答、能陈述自己知识
- 结论：如果排除证据存在，必须排除此假设
```

---

### 5. 一致性输出 > 模块割裂

**原则：**
- ❌ 功能判断说"逃避"，心伴解读说"心理理论发展"
- ✅ 功能判断、归因、解读三者一致
- ✅ 讲述同一个专业的、自洽的、关于孩子发展的"故事"

## 🧪 测试要求

### 核心测试用例

**1. 薯片盒子游戏（观点采择）**
```
输入：她以为我看到盒子里还是糖，不太理解我看到的是薯片的意思。
预期：
- 功能判断：观点采择能力发展中的挑战（或"需要进一步观察"）
- 行为背后的原因：心理理论发展节奏
- 心伴解读：心理理论/换位思考
- 能力缺口：已掌握 + 卡点 + 发展阶段
```

**2. 等待提示案例（提示依赖）**
```
输入：遇到困难就说"我不会"，等着大人教，其实都会做。
预期：
- 功能判断：提示依赖下的逃避（有明确证据）
- 行为背后的原因：工作记忆/执行功能
- 心伴解读：执行功能发展中的挑战
```

**3. 捣乱案例（寻求关注）**
```
输入：老师看她时她就故意发出奇怪声音，老师不看她就不发。
预期：
- 功能判断：寻求关注
- 行为背后的原因：社交沟通技能
- 心伴解读：社交技能发展中的挑战
```

## 🚫 常见错误与避免

### 错误 1：功能判断误判

**错误：** 薯片盒子游戏 → "提示依赖下的逃避"  
**原因：** 基于关键词（"不明白"、"不太理解"）误判  
**避免：** 执行证据审查，有排除证据时必须排除

---

### 错误 2：笼统归因

**错误：** "执行功能不足"、"工作记忆有限"  
**原因：** 缺少具体卡点描述  
**避免：** 描述"卡在哪个具体规则/概念/步骤"

---

### 错误 3：临床术语

**错误：** "临床诊断"、"症状"、"障碍"  
**原因：** 未执行去临床化  
**避免：** 使用发展性、教育性语言

---

### 错误 4：模块割裂

**错误：** 功能判断说"逃避"，心伴解读说"心理理论"  
**原因：** 模块孤立，无一致性检查  
**避免：** 执行"假设 - 验证"流程，确保三者一致
