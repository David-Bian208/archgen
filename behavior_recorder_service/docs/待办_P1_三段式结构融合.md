## 问题描述

### 当前报告结构（割裂）

```
📘 行为观察助手 V4.11.0
分析日期：2026-04-08

孩子的表现：...
周围的反应：...

🔍 行为背后的原因          ← 模块 A
```

**问题：**
- 三个模块孤立运行，像"拼接"的
- 家长需要自己整合信息
- 缺乏流畅的阅读体验

## 目标报告结构（叙事式）

```
📘 行为观察助手 V4.11.0
分析日期：2026-04-08

---

**这是一个关于心理理论（换位思考）正在发展的典型观察。**

在薯片盒子游戏中，OK 能记住盒子里实际是糖果（自己的知识），但当被问"妈妈会看到什么"时，她很难抑制"我知道是糖"这个优势反应，去推断"妈妈没看到被换，所以妈妈会以为是薯片"。

**这反映了心理理论发展节奏的个体差异。** 大脑中负责理解他人心理状态的区域（颞顶联合区）在 4-6 岁还在快速发育中。OK 目前的观点采择能力大约处于 4 岁水平（已能理解"知识状态不同"），但正在攻克"视角由位置决定"这条具体规则——她应用了简单规则"妈妈总是看到电脑"，而非"坐在电脑椅的人会看到电脑"。这恰恰是从"理解知识状态不同"迈向"掌握复杂视角采择"的关键过渡阶段。

**当前阶段定位：**
- ✅ **已掌握**：理解"每个人知道的信息可能不同"（一级错误信念，4-5 岁典型里程碑）
- 🎯 **卡点**：在需要处理多重变化要素（谁、在哪、看什么）的即时任务中，会因规则不清晰而退回静态认知
- 🚀 **发展方向**：从"人物 - 物体"固定链接转向"位置 - 视角"动态链接

**家庭支持策略：**
干预重点是用身体化、游戏化的方式，帮她发现并内化"位置决定视角"这条规则。例如：
- "指挥家"游戏：用手指指向"妈妈眼睛看到的那个东西"
- "照相馆"游戏：用相机验证"站在这里会拍到什么"
- "一步交换"游戏：只交换位置，先忽略"爸爸/妈妈"是谁

**反思与展望：**
OK 不是"不会"换位思考，而是这个技能还在发展中。就像学走路需要一步步练习，换位思考也需要在真实情境中反复练习。下次游戏时，试着多说"坐在哪里，就看到什么"，少说"谁看到什么"，用位置词汇作为思考的锚点。

---

💬 帮助我们改进
这份分析对你有帮助吗？...
```

## 修复方案

### 方案 A：后端生成叙事式报告（推荐）

**修改文件：** `app/agents/insight_analyzer.py`

**修改位置：** 系统提示词中的 JSON 输出格式

**当前输出格式：**
```json
{
  "functional_judgment": "...",
  "core_insight": "...",
  "capability_hypothesis": "...",
  "clinical_differential": "...",
  "narrative_analysis": "...",
  "intervention_principle": "..."
}
```

**修改为：**
```json
{
  "narrative_report": "完整的叙事式报告（Markdown 格式）",
  "structured_data": {
    "capability_gaps": [...],
    "developmental_stage": {...},
    "intervention_strategies": [...]
  }
}
```

**叙事报告生成提示词：**
```python
【叙事式报告生成 - V4.10.4 P1】

请基于以下信息生成一份连贯的叙事式报告：
- 孩子的具体行为表现
- 功能判断（已验证的假设）
- 能力缺口分析（已掌握 + 卡点 + 发展阶段）
- 神经/发展归因
- 干预策略方向

**报告结构：**
1. **开篇定调**（1 句）：用一句话命名这个观察的发展意义
   - 示例："这是一个关于心理理论（换位思考）正在发展的典型观察。"

2. **行为描述 + 归因**（1-2 段）：将行为表现与发展归因自然融合
   - 示例："在薯片盒子游戏中，OK 能记住...但当被问...时，她很难...这反映了..."

3. **当前阶段定位**（分点显示）：
   - ✅ 已掌握（具体里程碑）
   - 🎯 卡点（具体规则/概念）
   - 🚀 发展方向（从 X 向 Y）

4. **家庭支持策略**（2-3 个具体游戏）：
   - 游戏名称 + 玩法 + 思维逻辑

5. **反思与展望**（1 段）：温暖、赋能的结语
   - 示例："OK 不是'不会'，而是这个技能还在发展中..."

**语言风格：**
- 用"我们"视角，拉近距离
- 用"还没学会"而非"不会"
- 用"练习"而非"训练"
- 短句优先（每句<30 字）
- 像聊天不像写论文
```

### 方案 B：前端重组现有字段（快速）

**修改文件：** `frontend/App.vue`

**不修改后端**，前端将现有字段重新组织成叙事式：

```vue
<div class="narrative-report">
  <!-- 开篇定调 -->
  <p class="opening-statement">
    这是一个关于{{ insightReport.capability_hypothesis }}的典型观察。
  </p>
  
  <!-- 行为描述 + 归因 -->
  <div class="behavior-attribution">
    <p>在{{ reportData.context }}中，{{ reportData.child_behavior }}。这反映了{{ insightReport.capability_hypothesis }}。</p>
  </div>
  
  <!-- 当前阶段定位 -->
  <div class="developmental-stage">
    <h4>当前阶段：</h4>
    <ul>
      <li>✅ 已掌握：...</li>
      <li>🎯 卡点：...</li>
      <li>🚀 发展方向：...</li>
    </ul>
  </div>
  
  <!-- 家庭支持策略 -->
  <div class="support-strategies">
    <h4>家庭支持策略：</h4>
    <p>{{ interventionPlan.strategy_details }}</p>
  </div>
  
  <!-- 反思与展望 -->
  <div class="reflection">
    <p>{{ insightReport.reflection }}</p>
  </div>
</div>
```

## 验收标准

### 叙事连贯性
- [ ] 开篇定调清晰（1 句话命名发展意义）
- [ ] 行为描述与归因自然融合
- [ ] 段落之间过渡流畅
- [ ] 无"拼接感"

### 信息完整性
- [ ] 包含"已掌握 + 卡点 + 发展阶段"
- [ ] 包含具体游戏策略（2-3 个）
- [ ] 包含温暖结语

### 语言风格
- [ ] 用"我们"视角
- [ ] 用"还没学会"而非"不会"
- [ ] 短句优先（每句<30 字）
- [ ] 像聊天不像写论文
