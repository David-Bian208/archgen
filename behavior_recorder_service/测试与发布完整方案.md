## 🎯 核心问题诊断

您当前的困境：
> "几千轮对话 + 目测完成工具，但这是盲盒。对外发布需要什么验证方法？"

**本质问题：**
1. ✅ 功能可用 ≠ 产品可靠
2. ✅ 个人使用 ≠ 多人并发
3. ✅ 目测正常 ≠ 无隐藏 bug
4. ✅ 一次成功 ≠ 持续稳定

**解决思路：** 三层验证体系
```
技术测试（AI + 自动化） → 用户测试（真实场景） → 数据驱动迭代
```

## 🧪 第一阶段：技术验证（AI 主导）

### 1.1 AI 生成完整测试清单

**目标：** 让 AI 根据您的 PRD 生成系统性测试清单

**测试维度：**
| 维度 | 说明 | 示例 |
|------|------|------|
| **功能测试** | 核心功能是否正常 | 输入行为描述 → 生成报告 |
| **边界测试** | 极端输入处理 | 空输入、超长文本、特殊字符 |
| **异常测试** | 错误场景处理 | 网络中断、API 超时、数据库异常 |
| **性能测试** | 响应速度与并发 | 10 人同时使用是否卡顿 |
| **安全测试** | 数据保护 | SQL 注入、XSS 攻击防护 |

**AI 指令模板：**
```markdown
请根据以下 PRD 生成完整测试清单：

【产品描述】
行为观察伙伴：家长输入孩子行为描述，AI 基于 ABC 理论生成分析报告

【核心功能】
1. 行为描述输入（文本）
2. AI 分析生成（功能判断 + 能力缺口 + 干预建议）
3. 报告展示与反馈

【测试要求】
- 功能测试：20 个典型场景
- 边界测试：10 个极端场景
- 异常测试：5 个错误场景
- 性能测试：并发 10 人响应时间<3s
- 安全测试：输入验证、数据加密

请输出为表格格式，包含：测试编号、测试场景、输入数据、预期结果、优先级
```

**执行方式：**
```bash

# 2. 保存为 tests/test_checklist.md
# 3. 逐项执行并记录结果
```

```bash
# 2. 录制操作
playwright codegen http://localhost:3000

# 3. 自动生成测试代码
# 录制完成后自动生成 test_e2e.py
```

**示例测试脚本：**
```python
# tests/test_e2e_core.py
from playwright.sync_api import sync_playwright

def test_core_workflow():
    """测试核心流程：输入 → 分析 → 报告 → 反馈"""
    with sync_playwright() as p:
        browser = p.chromium.launch()
        page = browser.new_page()
        
        # 访问页面
        page.goto("http://localhost:3000")
        
        # 输入行为描述
        page.fill("textarea", "孩子在课堂上故意制造声音")
        page.click("button:has-text('发送')")
        
        # 等待报告生成
        page.wait_for_selector(".report-content", timeout=10000)
        
        # 验证报告内容
        assert page.is_visible(".functional-judgment")
        assert page.is_visible(".intervention-suggestions")
        
        # 提交反馈
        page.click(".rating-star:nth-child(5)")  # 5 星
        page.fill(".feedback-text", "分析很准确")
        page.click("button:has-text('提交反馈')")
        
        # 验证反馈成功
        assert page.is_visible(".feedback-success")
        
        browser.close()
```

**自动化测试执行：**

```bash
# 生成测试报告
```

```python
# tests/test_security.py

def test_xss_protection():
    """测试 XSS 防护"""
    malicious_input = "<script>alert('xss')</script>"
    response = post_api("/api/v4/chat", {"user_input": malicious_input})
    assert "<script>" not in response.text  # 应该被转义

def test_sql_injection():
    """测试 SQL 注入防护"""
    malicious_input = "' OR '1'='1"
    response = post_api("/api/v4/chat", {"user_input": malicious_input})
    assert response.status_code == 200  # 不应报错

def test_api_rate_limit():
    """测试 API 限流"""
    for i in range(100):
        response = post_api("/api/v4/chat", {"user_input": "test"})
        if response.status_code == 429:
            print(f"限流触发于第 {i} 次请求")
            break
```

**埋点分析看板：**
```sql
-- 每日活跃用户
SELECT DATE(timestamp) as date, COUNT(DISTINCT session_id) as users
FROM test_logs
GROUP BY DATE(timestamp)
ORDER BY date DESC;

-- 功能使用率
SELECT action_type, COUNT(*) as count
FROM user_actions
GROUP BY action_type
ORDER BY count DESC;

-- 平均评分趋势
SELECT DATE(timestamp) as date, AVG(rating) as avg_rating
FROM user_feedback
GROUP BY DATE(timestamp)
ORDER BY date DESC;

-- 问题高发场景
SELECT user_input, COUNT(*) as count
FROM test_logs t
JOIN user_feedback f ON t.session_id = f.session_id
WHERE f.rating <= 2
GROUP BY user_input
ORDER BY count DESC;
```

### 3.7 线上监控配置

**Sentry 错误监控（推荐）：**

**1. 注册 Sentry：**
```
访问 https://sentry.io/
注册免费账号
创建新项目（选择 Vue.js）
获取 DSN 密钥
```

**2. 前端集成：**
```javascript
// frontend/main.js
import * as Sentry from "@sentry/vue";

Sentry.init({
  app,
  dsn: "你的 Sentry DSN",
  integrations: [
    new Sentry.BrowserTracing(),
    new Sentry.Replay(),
  ],
  tracesSampleRate: 1.0,
  replaysSessionSampleRate: 0.1,
  replaysOnErrorSampleRate: 1.0,
});
```

**3. 后端集成：**
```python
# app/main.py
import sentry_sdk
from sentry_sdk.integrations.fastapi import FastApiIntegration

sentry_sdk.init(
    dsn="你的 Sentry DSN",
    integrations=[FastApiIntegration()],
    traces_sample_rate=1.0,
)
```

**监控效果：**
- 实时捕获前端/后端错误
- 自动记录错误堆栈
- 用户行为回放（可选）
- 错误告警通知

### 4.2 版本管理

**版本号规则：** `主版本。次版本.修订号`
- `1.0.0` - 首次发布
- `1.0.1` - Bug 修复
- `1.1.0` - 新功能
- `2.0.0` - 重大更新

**更新频率：**
- **修订版（x.x.1）：** 每周（Bug 修复）
- **次版本（x.1.0）：** 每月（新功能）
- **主版本（2.0.0）：** 每季度（重大更新）

### 4.3 迭代优先级

**P0（紧急）：** 24 小时内修复
- 核心功能不可用
- 数据丢失风险
- 安全漏洞

**P1（重要）：** 1 周内修复
- 影响用户体验的 Bug
- 高频功能问题
- 性能问题

**P2（优化）：** 纳入下个版本
- 体验优化
- 新功能建议
- 界面改进
