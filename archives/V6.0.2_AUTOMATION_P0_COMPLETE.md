### 任务 2：安全扫描脚本

**文件：** `scripts/security_scan.py` (11KB)

**功能：**
1. 读取 SECURITY_CHECKLIST.yml
2. 遍历项目文件（.py/.js/.vue）
3. 执行正则匹配
4. 生成结构化报告（按严重级别分组）
5. 告警阈值检测

**预计耗时：** <10 秒

**使用方法：**
```bash
python scripts/security_scan.py
```

**输出：**
- `security_reports/security_scan_YYYYMMDD_HHMMSS.txt`（人类可读）
- `security_reports/security_scan_YYYYMMDD_HHMMSS.json`（JSON 格式）

### 任务 5：提交信息草案生成器

**文件：** `scripts/commit_msg_generator.py` (6KB)

**功能：**
1. 读取任务文档
2. 提取任务信息（ID、描述、改动文件）
3. 生成 COMMIT_MSG_DRAFT.md
4. 生成 Git 提交任务文档

**原则：**
- ✅ 战舰生成"文本建议"
- ✅ 小治保留最终修改权和执行权

**使用方法：**
```bash
python scripts/commit_msg_generator.py /workspace/TASK_*.md
```

**输出：**
- `git_tasks/{TASK_ID}_COMMIT_MSG_DRAFT.md`
- `git_tasks/{TASK_ID}_GIT_TASK.md`

## 🔄 新工作流程

### 流程 1：自动化验证流程

```
trae 开发完成 
  → 战舰运行 auto_test.py（1 分钟）
  → 生成验证报告
  → ✅ 通过 → 创建 Git 提交任务
  → ❌ 失败 → 创建修复任务
```

### 流程 2：Git 提交流程

```
战舰验证通过
  → 运行 commit_msg_generator.py
  → 生成 COMMIT_MSG_DRAFT.md + GIT_TASK.md
  → 派发给小治
  → 小治复核 + 修改 + 执行 git commit
  → 通知战舰验证
  → 战舰验证提交完成，闭环
```

### 流程 3：知识沉淀流程

```
小治执行 Git 提交
  → 填写开发总结（必填）
  → 提交到 memory/lessons/
  → 战舰归档到 MEMORY.md
  → 下次派发任务时自动注入相关经验
```
