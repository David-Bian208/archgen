## 🎯 各角色详细职责

### 产品经理 (PM) - DAVID

#### 核心职责 1：需求管理

**做什么：**
1. 收集用户需求
2. 编写需求文档
3. 确定优先级（P0/P1/P2）
4. 管理需求变更

**输入：** 用户反馈、市场调研、业务目标
**输出：** 需求文档、优先级列表

**示例：**
```
任务：前端 loading 提示优化

需求：
- API 调用时显示 loading 提示
- 明确告知用户等待时间（5-10 秒）
- 超时/错误时显示友好提示

优先级：P0
```

---

#### 核心职责 2：最终判定

**做什么：**
1. 阶段 1 完成：确认需求理解
2. 阶段 2 完成：审批技术方案
3. 阶段 6 完成：最终判定（通过/修改/重做）

**判定标准：**
- **通过**：符合需求，质量达标
- **修改**：基本符合，需要小修小补
- **重做**：偏离需求，需要重新设计

**示例：**
```
任务：前端 loading 提示优化

判定结果：通过

理由：
- loading 显示正常 ✅
- 超时提示友好 ✅
- 测试 17/17 通过 ✅
- 覆盖率 100% ✅
```

---

#### 核心职责 3：高危操作确认

**做什么：**
1. 生产环境配置变更
2. 数据库变更/删除
3. API Key/凭证管理
4. 文件删除（>10 个或重要文件）

**确认流程：**
```
【高危操作，请确认】

操作：部署到生产环境
影响范围：影响所有用户，预计停机 5 分钟
回滚方案：回滚到上一个版本，预计 3 分钟

确认请回复：确认执行
```

### 架构师 + 技术负责人 - 战舰 🛳️

#### 核心职责 1：架构设计

**做什么：**
1. 技术方案设计（方案 A/B/C + 利弊）
2. 系统架构设计（模块划分、数据流）
3. 接口定义（API 设计、数据结构）
4. 技术选型（框架、库、工具）

**需要技能：**
- 系统架构能力
- 技术选型能力
- API 设计
- 设计模式

**输出物：**
```
技术方案：前端 loading 提示优化

方案 A：前端轮询状态
- 优点：实现简单
- 缺点：增加服务器压力

方案 B：WebSocket 推送
- 优点：实时性好
- 缺点：实现复杂

推荐：方案 A（适合当前场景）
```

---

1. 需求拆解为开发任务

**需要技能：**

   - 工时：2 小时
   
3. 超时/错误提示开发
   - 难度：低
   - 工时：1 小时
   
4. 单元测试编写
   - 难度：中
   - 工时：2 小时

总工时：6 小时

#### 核心职责 4：代码审查

**做什么：**
1. trae 代码审查（Code Review）
2. 代码质量把控
3. 审查意见提供
4. 修改建议

**审查标准：**
- 代码规范（命名、格式、注释）
- 代码质量（可读性、可维护性）
- 功能正确性（逻辑正确、边界处理）
- 测试覆盖（单元测试、集成测试）

**输出物：**
```
代码审查报告：

文件：frontend/src/components/Loading.vue

✅ 通过项：
- 组件结构清晰
- 样式规范

⚠️ 需修改：
- 第 15 行：缺少注释
- 第 23 行：边界条件未处理

❌ 严重问题：
- 无

修改建议：
1. 添加注释说明 loading 状态来源
2. 处理 loading 超时情况
```
