# 引导式行为记录员 V2.0 使用指南

## 快速开始

### 1. 访问界面

打开浏览器访问：**http://localhost:3000**

### 2. 开始对话

你会看到 AI 教练的欢迎消息：

> 🤖 你好！我是你的行为记录 AI 教练 👋
> 
> 我会通过几个问题，帮你完成一次专业的行为记录。
> 
> 💡 请简单描述一下孩子最近的一个行为，我们从那里开始。

### 3. 输入描述

在底部输入框输入简单描述，例如：

```
他今天打自己头
```

按 **Enter** 发送（Shift+Enter 换行）。

### 4. 回答 AI 提问

AI 会评估你的描述，然后提出引导性问题：

> 🤖 谢谢你的描述，我理解你观察到孩子打自己头的行为。
> 
> 在孩子出现这个行为之前，你们正在做什么？或者你正在要求他做什么？

**进度指示器**会显示当前收集状态：
- ✓ 前因 (A) - 已收集
- ✓ 行为 (B) - 已收集  
- ○ 后果 (C) - 待收集

继续回答：

```
因为不给他手机
```

### 5. 完成记录

AI 会继续提问，直到 ABC 三要素完整：

> 🤖 谢谢你的补充。
> 
> 当孩子打自己头之后，你或周围的人第一时间是怎么做的？

回答最后一个问题：

```
我赶紧把手机给他了
```

### 6. 查看报告

AI 会输出完整的行为分析报告：

```
📊 行为分析报告

前因 (A): 不给他手机
行为 (B): 打自己头
后果 (C): 我赶紧把手机给他了

假设功能：实物 (Tangible)
推理：行为后获得了手机这一具体物品，符合实物获取特征。
```

你可以：
- 🔄 **开始新记录** - 清除当前会话，开始新的行为记录
- 📋 **复制报告** - 将报告复制到剪贴板

## 功能说明

### 进度指示器

对话过程中，AI 会显示 ABC 收集进度：

```
✓ 前因 (A)  |  ✓ 行为 (B)  |  ○ 后果 (C)
```

- **灰色 ○** - 未收集
- **绿色 ✓** - 已收集

### 会话 ID

每次对话都有一个唯一的会话 ID（如 `6819e551`），用于：
- 保持对话连续性
- 多轮对话引用同一会话

### 消息类型

| 类型 | 说明 |
|------|------|
| 💬 用户消息 | 右侧蓝色气泡 |
| 🤖 AI 提问 | 左侧灰色气泡，带进度指示器 |
| 📊 分析报告 | 左侧灰色气泡，带完整报告卡片 |

## API 使用

### 通过 curl 调用

**第一轮**:
```bash
curl -X POST http://localhost:8000/api/v2/record \
  -H "Content-Type: application/json" \
  -d '{"user_input": "他今天打自己头"}'
```

**后续轮次**（使用返回的 session_id）:
```bash
curl -X POST http://localhost:8000/api/v2/record \
  -H "Content-Type: application/json" \
  -d '{"session_id": "6819e551", "user_input": "因为不给他手机"}'
```

### 通过 JavaScript 调用

```javascript
async function startRecord() {
  // 第一轮
  const response1 = await fetch('/api/v2/record', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      user_input: '他今天打自己头'
    })
  });
  const data1 = await response1.json();
  console.log(data1.session_id); // 保存会话 ID
  
  // 第二轮
  const response2 = await fetch('/api/v2/record', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      session_id: data1.session_id,
      user_input: '因为不给他手机'
    })
  });
  const data2 = await response2.json();
  
  if (data2.status === 'completed') {
    console.log('分析完成:', data2.data);
  }
}
```

## 常见问题

### Q: 我可以一次性提供所有信息吗？

**A**: 可以！如果你一次性提供了完整的 ABC 信息，AI 会直接进行分析，不再提问。

例如：
```
不给他手机，他打自己头，我马上把手机给他了。
```

### Q: 对话中断了怎么办？

**A**: 只要会话 ID 还在，就可以继续对话。如果页面刷新了，点击"开始新记录"重新开始。

### Q: 如何查看历史对话？

**A**: 当前版本暂不支持历史记录。V2.1 将添加会话持久化功能。

### Q: 报告可以导出吗？

**A**: 目前支持复制文本（点击"复制报告"）。V2.1 将添加 PDF 导出功能。

## 最佳实践

### 1. 简单开始

不需要一次性提供所有信息，从简单描述开始：
- ✅ "他今天哭闹了"
- ✅ "他不肯写作业"
- ✅ "她打小朋友"

### 2. 具体回答

回答 AI 提问时，尽量具体：
- ✅ "我正在要求他收拾玩具"
- ❌ "没什么"

### 3. 确认信息

如果 AI 总结的信息有误，可以纠正：
```
不对，不是这样，其实是...
```

### 4. 善用复制

完成报告后，点击"复制报告"保存结果，方便：
- 分享给其他家长/老师
- 存档记录
- 制作行为日志

## 技术信息

### 系统架构

```
┌─────────────┐     ┌──────────────┐     ┌─────────────┐
│   前端界面   │────▶│  后端 API    │────▶│  LLM 服务    │
│  (Vue 3)    │◀────│  (FastAPI)   │◀────│ (DeepSeek)  │
└─────────────┘     └──────────────┘     └─────────────┘
                          │
                          ▼
                    ┌──────────────┐
                    │  会话管理     │
                    │ (内存存储)    │
                    └──────────────┘
```

### API 端点

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/v2/record` | POST | 引导式记录（主要接口） |
| `/api/v2/session/{id}` | GET | 查看会话状态 |
| `/api/v2/session/{id}` | DELETE | 清理会话 |
| `/api/analyze` | GET | 单次分析（V1.1 兼容） |

### 数据格式

**功能类型**:
- `tangible` - 实物获取
- `attention` - 关注获取
- `escape` - 逃避/终止
- `automatic` - 自我刺激
- `inconclusive` - 无法确定
