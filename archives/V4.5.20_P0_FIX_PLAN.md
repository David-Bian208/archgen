# V4.5.20 P0 紧急修复计划 - 临床逻辑断裂修复

## 问题描述

**症状：** 诊断是"社交认知不足"，干预却是"寻求关注的行为干预"

**根因：** 场景选择逻辑只有两个分支（逃避难度/提示依赖），没有社交技能不足的分支

**风险：** 干预建议与诊断完全不符，可能误导家长，造成二次伤害

---

## 修复方案

### 1. 修改场景选择逻辑（structured_assessor_v4.py）

**当前逻辑：**
```python
if top_hypothesis[0] == "escape_difficulty" or "难" in behavior:
    scenario_key = "task_refusal"  # 逃避难度
else:
    scenario_key = "task_disengagement"  # 提示依赖（默认）
```

**修复后逻辑：**
```python
# 优先使用 functional_judgment 选择场景
functional_judgment = insight.get("functional_judgment", "")
capability_gap = insight.get("capability_hypothesis", "")

if "逃避" in functional_judgment and "难度" in functional_judgment:
    scenario_key = "task_refusal"
    hypothesis_id = "H_ESCAPE_DIFFICULTY"
    
elif "提示依赖" in functional_judgment:
    scenario_key = "task_disengagement"
    hypothesis_id = "H_PROMPT_DEPENDENCE"
    
elif "社交" in functional_judgment or "社交" in capability_gap or "换位思考" in capability_gap:
    scenario_key = "social_skills_deficit"  # 新增：社交技能不足
    hypothesis_id = "H_SOCIAL_SKILLS"
    
elif "关注" in functional_judgment:
    scenario_key = "attention_seeking"
    hypothesis_id = "H_ATTENTION"
    
else:
    # 默认：根据行为特征推断
    if "难" in behavior or "不会" in behavior:
        scenario_key = "task_refusal"
    else:
        scenario_key = "social_skills_deficit"  # 默认指向社交技能（更安全）
```

### 4. 修复信息整合逻辑（解决 P1 问题）

**根因：** `_generate_summary` 方法机械拼接 `child_age` + `inferred_setting` + `antecedent`

**修复：**

def _generate_summary(self, state, inferred_env):

    if daily_env and event_context:
        return f"{child_age}孩子（{daily_env}），在{event_context}时..."

### 5. 优化对话流程（解决 P2 问题）

**问题：** 用户已提供详细场景，系统还机械追问

**修复方案：**

#### 5.1 增强 LLM 提取提示词
```python
# 在 extraction_prompt 中强调：
【重要】如果用户输入已包含具体场景（时间 + 地点 + 事件），直接提取到 specific_example 和 antecedent 字段，不要追问！
```

#### 5.2 添加"信息充分性检测"
```python
def _is_information_sufficient(self, state):
    """检查是否已有足够信息生成报告"""
    required = ["problem_description", "specific_example", "child_age", "antecedent", "behavior_detail", "immediate_consequence"]
    filled = sum(1 for f in required if state.is_field_filled(f))
    return filled >= len(required) * 0.8  # 80% 即可结束
```

#### 5.3 智能追问策略
```python
# 如果用户已提供 specific_example，但 antecedent 为空
# → 从 specific_example 中提取 antecedent，不要重复问
if state.get_field_value("specific_example") and not state.get_field_value("antecedent"):
    example = state.get_field_value("specific_example")
    # 用 LLM 从例子中提取 antecedent
    antecedent = self._extract_antecedent_from_example(example)
    state.set_field_value("antecedent", antecedent)
```
