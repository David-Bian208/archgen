## 📊 错误分级标准

### P0 级 - 核心功能不可用 🔴

**定义：** 核心功能完全无法使用，阻塞测试和部署

**示例：**
```
❌ API 调用失败（秒回无结果）
❌ 服务无法启动
❌ 数据库连接失败
❌ 认证系统故障
```

**响应时间：** 立即修复（<1 小时）

**修复流程：**
```
1. 立即通知开发者
2. 查看后端日志
3. 诊断根本原因
4. 修复并验证
5. 执行 P0 测试
```

---

### P1 级 - 重要功能有问题 🟡

**定义：** 重要功能有问题，但可以临时绕过

**示例：**
```
⚠️ UI 不自适应
⚠️ 加载状态缺失
⚠️ 错误提示不友好
⚠️ 部分功能异常
```

**响应时间：** 24 小时内修复

**修复流程：**
```
1. 记录问题详情
2. 分析影响范围
3. 安排修复计划
4. 修复并验证
5. 更新测试用例
```

---

### P2 级 - 体验问题 🟢

**定义：** 不影响功能，但影响用户体验

**示例：**
```
💡 文字大小不合适
💡 颜色搭配不佳
💡 按钮位置不优化
💡 动画效果缺失
```

**响应时间：** 本周内修复

**修复流程：**
```
1. 收集用户反馈
2. 评估改进价值
3. 批量修复
4. 用户验证
```

## 📋 常见错误模式库

### 1. API 调用失败

**症状：**
```
- 用户提交后秒回
- 无实际分析结果
- 前端显示"信息已收集"但无后续
```

**可能原因：**
```
1. ❌ 后端未正确调用 LLM API
2. ❌ API Key 配置错误
3. ❌ 前端未正确处理响应
4. ❌ 网络超时
```

**诊断步骤：**
```bash
# 1. 检查 API Key 配置
cat .env | grep LLM_API_KEY

# 2. 检查后端日志
tail -f /tmp/backend.log | grep -i "error\|api"

# 3. 测试 API 端点
curl -X POST http://localhost:8000/api/v4/analyze \
  -H "Content-Type: application/json" \
  -d '{"records":[{"antecedent":"测试","behavior":"测试","consequence":"测试"}],"context":"测试"}'

# 4. 检查前端网络请求
# 浏览器开发者工具 → Network → 查看 API 请求
```

**修复建议：**
```
根据诊断结果：
- 如 API Key 错误 → 更新 .env 文件
- 如后端未调用 → 检查后端代码
- 如前端未处理 → 检查前端代码
- 如网络超时 → 增加超时时间
```

---

### 2. UI 不自适应

**症状：**
```
- 移动端显示异常
- 文字溢出
- 按钮无法点击
- 布局错乱
```

**可能原因：**
```
1. ❌ 缺少媒体查询
2. ❌ 固定宽度布局
3. ❌ 未使用响应式框架
4. ❌ 测试不足
```

**诊断步骤：**
```bash
# 1. 检查 CSS 文件
grep -r "@media" frontend/

# 2. 检查布局方式
grep -r "width:" frontend/ | grep -v "max-width"

# 3. 多设备测试
# 使用浏览器开发者工具的 Device Mode
```

**修复建议：**
```
1. 添加媒体查询
2. 使用 Flexbox/Grid布局
3. 使用相对单位（rem、%）
4. 添加多设备测试
```

---

### 3. 加载状态缺失

**症状：**
```
- 用户提交后无反馈
- 不知道是否在加载
- 误以为系统卡死
```

**可能原因：**
```
1. ❌ 未添加 loading 状态
2. ❌ 未显示进度提示
3. ❌ 超时时间设置不当
```

**诊断步骤：**
```javascript
// 检查前端代码
grep -r "loading" frontend/App.vue
grep -r "isLoading" frontend/
```

**修复建议：**
```
1. 添加 loading 状态
2. 显示"分析中，预计 5-10 秒"
3. 添加进度条或动画
4. 设置合理的超时时间
```
