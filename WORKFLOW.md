## 🎭 角色定义（标准开发团队）

| 角色 | 人员 | 职责 |
|------|------|------|
| **产品经理 (PM)** | DAVID | 需求、优先级、最终判定 |
| **架构师 + 技术负责人** | 战舰 🛳️ | 架构设计、技术方案、代码审查、测试、部署 |
| **开发工程师 (Dev)** | trae | 编码实现、单元测试 |
| **测试工程师 (QA)** | 战舰 🛳️ | 测试用例、集成测试、覆盖率审计 |
| **运维工程师 (Ops)** | 战舰 🛳️ | 部署、监控、配置管理 |

**说明：** 战舰兼任架构师/技术负责人/QA/Ops 多职，trae 专注编码。

## 🔄 协作流程（6 阶段）

### 流程图

```
┌─────────────────────────────────────────────────────────────────┐
│  阶段 1：需求（PM）                                               │
│  DAVID：需求文档 + 优先级                                         │
│  战舰：需求澄清 + 技术可行性                                       │
│  DAVID：确认需求                                                  │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  阶段 2：架构设计（架构师）                                        │
│  战舰：技术方案 + 架构设计 + 接口定义                               │
│  DAVID：审批方案                                                  │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  阶段 3：开发（Dev）                                              │
│  战舰：任务拆解 + 技术指导                                        │
│  trae：编码实现 + 单元测试                                        │
│  战舰：代码审查（Code Review）                                    │
│  ❌ 不通过 → trae 修复                                            │
│  ✅ 通过 → 进入阶段 4                                              │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  阶段 4：测试（QA）                                               │
│  战舰：集成测试 + 回归测试 + 覆盖率审计                             │
│  ❌ 失败 → 返回阶段 3 修复                                          │
│  ✅ 通过 → 进入阶段 5                                              │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  阶段 5：部署（Ops）                                              │
│  战舰：部署检查 + 执行部署 + 监控                                   │
│  【高危操作，请确认】→ DAVID 确认                                   │
│  战舰：执行部署                                                   │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  阶段 6：判定（PM）                                               │
│  DAVID：最终判定（通过/修改/重做）                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 🚦 触发转换规则

### 你 → 战舰 的指令时机

| 时机 | 指令 | 战舰执行 |
|------|------|----------|
| **新需求** | "我想做..." | 需求分析 + 执行计划 |
| **问题反馈** | "XX 有问题" | 根因分析 + 修复 |
| **最终判定** | "通过"/"修改"/"重做" | 执行判定结果 |

### 战舰 → 你 的汇报时机

| 时机 | 汇报 | 你需要 |
|------|------|--------|
| **需求确认** | 需求理解 + 执行计划 | 确认/修改 |
| **方案选择** | ≥2 个方案 + 利弊 | 选一个 |
| **高危操作** | 【高危操作，请确认】 | 确认/拒绝 |
| **任务完成** | 验证报告 | 判定（通过/修改/重做） |

### trae 卡壳处理（战舰负责）

| 卡壳类型 | 战舰处理 |
|----------|----------|
| **编码超时** | 战舰接管，重新分析需求 |
| **逻辑错误** | 战舰根因分析 + 修复 |
| **依赖问题** | 战舰解决依赖或调整方案 |
| **反复失败** | 战舰切换方案或求助 DAVID |

## 🛡️ 安全协议

### 高危操作定义

以下操作必须【高危操作，请确认】：

1. **生产环境** - 修改生产配置、API Key、数据库
2. **删除操作** - 删除文件、数据库记录、日志
3. **外部发送** - 邮件、消息、公开帖子
4. **权限变更** - 修改用户权限、访问控制

### 确认格式

```
【高危操作，请确认】

**操作：** [具体操作]
**影响范围：** [影响哪些系统/数据]
**回滚方案：** [如何撤销]
**确认请回复：** 确认执行
```
