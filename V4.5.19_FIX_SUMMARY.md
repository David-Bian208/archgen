# V4.5.19 修复总结 - 模糊回答检测与问题澄清

## 修复日期
2026-03-15

## 问题背景

用户在对话中表达困难或使用模糊标签时，系统机械重复问题或跳过话题，缺乏共情和引导：

**典型案例：**
```
💬 系统：当孩子出现这个行为时，您或周围人第一时间是怎么回应的？
👤 用户：不知道怎么回答你，要根据实际情况
💬 系统：（重复同样的问题）❌

💬 系统：（跳到感官敏感问题）❌
👤 用户：要看具体事情
```

**根因分析：**
1. 框架设计缺失：没有"问题澄清"阶段，模糊的行为标签（如"深度关注有问题"）直接进入 ABC 收集
2. 模糊回答检测缺失：没有识别"不知道"、"看情况"等困难表达
3. 共情回应缺失：没有先认可用户感受，再换方式问
4. 机械推进：工作流按阶段顺序推进，不问用户是否困惑

#### 2.1 新增模糊回答模式定义

```python
VAGUE_ANSWER_PATTERNS = {
    "不知道": {
        "keywords": ["不知道", "不清楚", "没注意", "忘了", "说不清", "不确定"],
        "empathy": "我理解，确实每个情况都不一样，一时说不清楚很正常。",
        "reframe": "那不如这样——您能回想一个最近发生的具体例子吗？..."
    },
    "看情况": {
        "keywords": ["看情况", "要看", "不一定", "不好说", "这个嘛"],
        "empathy": "您说得对，不同事情确实处理方式会不一样。",
        "reframe": "那我们就拿一个具体事情来说——最近一次让您注意到这个行为的是什么事？..."
    },
    "模糊标签": {
        "keywords": ["深度关注", "社交互动", "有问题", "不对劲", "不正常"],
        "empathy": "我明白您想表达的意思，不过这些词确实比较抽象。",
        "reframe": "能具体描述一下孩子做了什么吗？比如：不看人？不回应名字？..."
    },
    "抗拒回答": {
        "keywords": ["不想说", "没必要", "说了你也不懂", "太复杂了"],
        "empathy": "我理解，这种情况确实可能比较复杂，或者让您不太舒服。",
        "reframe": "没关系，我们可以从最简单的开始——比如最近一次发生这个行为是什么时候？..."
    },
}
```

#### 2.2 新增检测方法
```python
def _is_vague_answer(self, user_input: str) -> Optional[str]:
    """检测模糊回答/困难表达"""
    for vague_type, data in self.VAGUE_ANSWER_PATTERNS.items():
        if any(kw in user_input for kw in data["keywords"]):
            return vague_type
    return None

def _generate_empathetic_response(self, vague_type: str, field_id: str = None) -> str:
    """生成共情 + 换问法的回应"""
    data = self.VAGUE_ANSWER_PATTERNS.get(vague_type)
    if not data:
        return "能再详细描述一下吗？"
    return f"{data['empathy']} {data['reframe']}"
```

#### 2.3 修改 process 方法
在字段提取之前先检测模糊回答：
```python
# V4.5.19 新增：模糊回答检测
vague_type = self._is_vague_answer(user_input)
if vague_type:
    logger.info(f"🔍 检测到模糊回答：{vague_type}，生成共情回应")
    # 获取当前需要问的字段
    current_field = ...
    empathetic_response = self._generate_empathetic_response(vague_type, current_field)
    
    # 记录共情回应到对话历史
    state.conversation_history.append({
        "role": "ai",
        "text": empathetic_response,
        "turn": state.turn_count,
        "field_id": current_field,
        "is_empathetic": True,  # 标记这是共情回应，不是新问题
    })
    
    # 返回共情回应，不推进工作流
    return {
        "session_id": state.session_id,
        "status": "in_progress",
        "response_type": "empathetic_follow_up",
        "message": empathetic_response,
        "vague_type": vague_type,
    }
```

### 3. 工作流层：已问字段跟踪与跳过逻辑

**修改 `_decide_next_action_with_reasoning` 方法：**

#### 3.1 计算已问字段（排除共情回应）
```python
asked_fields = {}
for msg in state.conversation_history:
    if msg.get("role") == "ai" and msg.get("field_id"):
        # 跳过共情回应（不计入已问次数）
        if msg.get("is_empathetic"):
            continue
        asked_fields[field_id] = asked_fields.get(field_id, 0) + 1
```

#### 3.2 跳过已问过 2 次的字段
```python
for f in current_stage.get("fields", []):
    field_id = f["field_id"]
    if not state.is_field_filled(field_id):
        asked_count = asked_fields.get(field_id, 0)
        if asked_count >= 2:
            logger.warning(f"⚠️ 字段{field_id}已问过{asked_count}次，跳过")
            skipped_fields.append(field_id)
            continue
        # 正常提问逻辑...
```

#### 3.3 强制结束逻辑
```python
# 如果所有字段都已问过 2 次，强制结束
if skipped_fields:
    logger.warning(f"⚠️ 有{len(skipped_fields)}个字段已问过 2 次仍未填充，强制生成报告")
    return {"type": "GENERATE_REPORT", "source": "skipped_fields"}
```
