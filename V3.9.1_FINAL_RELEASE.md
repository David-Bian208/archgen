## 核心改进

### 1️⃣ 彻底根治数据乱码（最高优先级）

**问题**: 报告中可能出现 `%!(string=`、`(MISSING)`、`&#039;` 等调试信息或未渲染代码

**解决方案**: 添加 `sanitize_report_data()` 净化函数，在返回前彻底清理所有乱码。

**后端实现** (`app/agents/guided_recorder_agent_v3.py`):
```python
import re
import html

def _sanitize_report_data(self, obj):
    """V3.9 Final 彻底净化报告数据 - 递归清理所有乱码"""
    if isinstance(obj, dict):
        return {key: self._sanitize_report_data(value) for key, value in obj.items()}
    elif isinstance(obj, list):
        return [self._sanitize_report_data(item) for item in obj]
    elif isinstance(obj, str):
        # 1. HTML 实体解码
        text = html.unescape(obj)
        
        # 2. 移除调试信息模式
        text = re.sub(r'%!\([^)]*\)', '', text)  # 移除 %!(...) 模式
        text = text.replace('(MISSING)', '')  # 移除 (MISSING)
        text = re.sub(r'!?,\s*\(MISSING\)', '', text)  # 移除 !,(MISSING)
        
        # 3. 清理多余空格
        text = re.sub(r'\s+', ' ', text)
        text = text.strip()
        
        return text
    else:
        return obj

def process_input(self, session_id, user_input):
    # ... 生成响应 ...
    
    # V3.9 Final 彻底净化：清理所有乱码和调试信息
    response = self._sanitize_report_data(response)
    
    return response
```

**清理规则**:
1. ✅ 移除 `%!(...)` 模式（调试信息）
2. ✅ 移除 `(MISSING)` 字符串
3. ✅ 移除 `!,(MISSING)` 模式
4. ✅ HTML 实体解码（`&#039;` → `'`）
5. ✅ 清理多余空格

**效果**: 
- ✅ 零容忍乱码：在任何输入案例下，绝不允许出现乱码
- ✅ 特殊字符 100% 正确显示（%, ≥, ', "）
- ✅ 无调试信息残留

## 技术实现细节

### 数据净化流程

```
LLM 生成 JSON 响应
    ↓
可能包含调试信息或 HTML 实体
    ↓
调用 _sanitize_report_data(response)
    ↓
递归遍历所有字段（字典 → 列表 → 字符串）
    ↓
对每个字符串执行：
  1. html.unescape() 解码
  2. re.sub(r'%!\([^)]*\)', '', text) 清理调试信息
  3. text.replace('(MISSING)', '') 清理缺失标记
  4. re.sub(r'\s+', ' ', text) 清理空格
    ↓
返回净化后的响应 → 前端渲染
```

### 文案精简原则

1. **摘要**:
   - ❌ 删除：情境重复描述
   - ✅ 保留：核心结论 + 干预策略 + 价值

2. **第一步行动**:
   - ❌ 删除：冗余条件句（"如果...如果..."）
   - ✅ 保留：核心动作 + 鼓励语

3. **全文统一**:
   - 中文引号：`"` `'`
   - 标点：全角中文标点
   - 空格：标点后无多余空格
