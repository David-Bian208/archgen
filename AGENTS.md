**版本：** V4.12.1

**项目：** 行为观察助手（AI 儿童行为分析工具）

---

## 🎯 项目定位

**行为观察助手** 是一个专业临床评估工具，帮助家长理解孩子行为背后的心理意义。

**核心价值：**
- 不是诊断工具，是"理解桥梁"
- 温暖、专业、可操作
- 像朋友聊天，不是写论文

**目标用户：**
- 自闭症儿童家长
- 幼儿园老师
- 特教从业者

---

## 📋 核心报告结构（三部分）

### 第一部分：ABC 场景描述（Skills 1）
**功能：** 收集和组织家长提供的行为场景信息

**ABC 逻辑：**
- **A (Antecedent) 前因** - 行为发生前的情境
- **B (Behavior) 行为** - 孩子的具体表现
- **C (Consequence) 结果** - 周围的反应和环境

**输出格式：**
```json
{
  "antecedent": "情境描述",
  "behavior": "孩子的具体表现",
  "consequence": "周围的反应和环境",
  "child_age": "年龄信息"
}
```

**Skills 实现：**
- 输入：家长的自然语言描述
- 输出：结构化的 ABC 场景信息（JSON）
- 验证：检查 ABC 信息是否完整、准确

---

### 第二部分：行为分析（Agent 1，理性）
**功能：** 基于 ABC 场景，理性分析问题

**分析维度：**
1. **行为功能判断** - 寻求关注/逃避/获得物品/自我刺激
2. **能力缺口分析** - 心理理论/执行功能/视觉空间/语言理解
3. **问题识别** - 列出观察到的具体问题（3-5 个）

**输出格式：**
```json
{
  "behavior_function": "寻求关注/逃避/获得物品/自我刺激",
  "capability_gaps": [
    {"dimension": "心理理论", "description": "具体描述"},
    {"dimension": "执行功能", "description": "具体描述"}
  ],
  "identified_problems": ["问题 1", "问题 2", "问题 3"],
  "analysis_summary": "简短总结"
}
```

**Agent 实现：**
- 输入：ABC 场景描述（Skills 1 输出）
- 处理：专业判断和推理（需要理解复杂情境）
- 输出：结构化的行为分析结果（JSON）
- 要求：客观、专业、基于证据

**为什么是 Agent 而不是 Skills：**
- 需要理解复杂的 ABC 情境
- 需要多维度分析和专业判断
- 需要推理能力（从行为推断功能和能力缺口）
- 输出需要结构化但依赖上下文理解

---

### 第三部分：心伴解读（Agent 2，感性 + 建议 + 优先级）
**功能：** 根据行为分析结果，感性化表达 + 提供建议 + 优先级排序

**核心内容：**
1. **感性解读** - 像朋友聊天，温暖、共情
2. **问题排序** - 优先解决什么问题（卡点）
3. **行动建议** - 针对优先级问题提供建议

**输出格式：**
```markdown
## 💡 心伴解读

**我们觉得：** [温暖的感性解读，像朋友聊天]

**优先解决的卡点：**
1. **第一优先：** [最重要的问题]
2. **第二优先：** [次重要的问题]

**建议：**
- [针对优先级的具体建议]
```

**Agent 实现：**
- 输入：行为分析 Agent 的结果
- 处理：理解问题优先级 + 感性表达 + 建议生成
- 输出：自然语言报告（温暖、像聊天）
- 要求：共情、温暖、可操作

**为什么是 Agent 而不是 Skills：**
- 需要综合理解行为分析结果
- 需要决策能力（优先级排序）
- 需要感性表达（不是简单的信息转换）
- 需要根据情境调整语言风格

## 🚫 已删除的内容

以下内容**不再包含**在报告中：

- ❌ 干预建议（独立模块）
- ❌ 幼儿园配合建议（合并到心伴解读的建议中）
- ❌ 家庭引导建议（合并到心伴解读的建议中）
- ❌ 临床鉴别思考（简化到行为分析中）
- ❌ 其他冗余内容

**核心原则：** 精简、聚焦、可操作

## 📐 技术架构

### 前端
- **框架：** Vue 3 + Vite
- **端口：** 5174
- **核心页面：** 对话界面、报告展示（三部分结构）
- **API 调用：** 相对路径 `/api/v4/analyze`

### 后端
- **框架：** FastAPI
- **端口：** 8001
- **核心 API：**
  - `POST /api/v4/analyze` - 行为分析（ABC → 行为分析 → 心伴解读）
  - `POST /api/v4/feedback` - 反馈收集
  - `GET /health` - 健康检查

### LLM 服务
- **服务：** DeepSeek
- **API Key:** `LLM_API_KEY`（环境变量）
- **Base URL:** `https://api.deepseek.com`

## 💡 核心原则

### 文本 > 大脑 📝
- 所有重要决策必须写入文件
- "心理笔记"无法在 session 重启后存活

### 安全 > 完成
- 临床系统优先保证输出安全
- 不确定时降级到保守方案

### 自动化 > 手动
- 关键测试必须自动化

### 用户预期管理
- 提前告知等待时间（5-10 秒）
- 像朋友聊天，不是写论文

## 🔄 报告生成流程

```
用户输入
  ↓
ABC 场景描述（Skills 1）
  ↓
行为分析 Agent（理性分析，复杂推理）
  ↓
心伴解读 Agent（感性表达 + 建议 + 优先级）
  ↓
前端展示（三部分结构）
```

**架构说明：**
- **Skills 1（ABC 场景描述）** - 简单信息收集和结构化
- **Agent 1（行为分析）** - 复杂推理和专业判断
- **Agent 2（心伴解读）** - 感性表达和决策能力
