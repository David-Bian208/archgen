## 临床知识库原则

### 干预模板映射规则
1. **强制一对一映射** - 每个临床假设只能关联到一组预设的干预策略
2. **禁止跨模板调用** - 社交技能不足不能用寻求关注模板，坚持同一性不能用冷静角
3. **安全优先于功能分析** - 危险行为先处理安全，再处理行为功能

### 报告语言风格原则（V4.5.21 新增）
1. **避免通用比喻** - 不用"社交雷达"等模板化表述，用具体行为细节
2. **避免说教感** - 不用"临床上通常考虑几种可能性"，直接分析具体行为
3. **用"我们"视角** - 拉近距离，如"从玥玥的表现来看，我们觉得..."
4. **能力发展视角** - 用"还没学会"而不是"不会"，用"练习"而不是"训练"
5. **短句优先** - 每句<30 字，像聊天不像写论文

### 测试覆盖标准
- 每类别 ≥15 题
- 整体通过率 ≥95% 才可部署
- P0 案例必须 100% 通过
- 回归测试自动化集成到 CI/CD

## 开发原则

### 文本 > 大脑 📝
- 所有重要决策必须写入文件
- "心理笔记"无法在 session 重启后存活
- 错误必须文档化，防止未来重复

### 安全 > 完成
- 临床系统优先保证输出安全
- 不确定时降级到保守方案
- 人工审核队列是必要的安全网

### 自动化 > 手动
- 关键测试必须自动化
- 回归测试集成到 CI/CD
- 手动测试无法保证每次都执行

### V4.10.0 报告质量优化 (2026-03-17)

**核心成就：** 修复 V4.6.6 版本报告质量问题，达到"优化版示例"水准

**问题诊断（来自全方位对比分析）：**
1. **P0 致命问题**：摘要归因"工作记忆"与后文"社交认知"分析矛盾
2. **P1 结构问题**：缺少"鉴别与排除"临床推理过程展示
3. **P2 表达问题**：比喻普通，未能构建深刻理解框架

**关键修复：**

1. **P0 修复 - 摘要生成逻辑** (`structured_assessor_v4.py`)
   - `_generate_summary()`: 优先使用 `insight_result.capability_hypothesis` 而非 `inferred_hypotheses`
   - `_generate_core_challenge()`: 增强社交技能不足的细分匹配，避免错误归因
   - 核心原则：摘要中的能力归因必须与后文分析严格一致

2. **P1 修复 - 临床推理结构** (`insight_analyzer.py`)
   - 强化 `ANALYSIS_SYSTEM_PROMPT` 中的"鉴别诊断结构要求"
   - 强制三模块结构：鉴别与排除 → 核心假设 → 能力缺口分析
   - 更新 JSON 输出格式要求，`clinical_differential` 必须包含完整三模块

3. **P2 修复 - 比喻生成优化** (`structured_assessor_v4.py`)
   - `_generate_ability_pattern()`: 建立"问题类型→比喻库"映射
   - 社交误解类 → "沉浸在自己剧本里的导演"
   - 规则僵化类 → "只能播放三首歌的收音机/单曲循环"
   - 提示依赖类 → "建筑需要脚手架支撑"
   - 寻求关注类 → "需要站在聚光灯下的表演者"

**预期效果：**
- 摘要准确率：从 80% 提升至 95%+
- 临床推理展示：从"只列结论"到"展示推理过程"
- 比喻质量：从"描述问题"到"构建理解框架"

**文件位置：**
- 摘要修复：`behavior_recorder_service/app/agents/structured_assessor_v4.py`
- 临床推理修复：`behavior_recorder_service/app/agents/insight_analyzer.py`
- 对比分析报告：`/home/admin/.openclaw/workspace/V4.6.6_报告对比分析与优化指令.md`

**核心教训：**
- 临床系统中，摘要与后文矛盾会直接破坏专业信任
- 家长需要的不仅是"结论"，更是"理解问题的框架"
- 比喻不是装饰，而是帮助家长建立心理模型的认知工具

### V4.10.1 场景化干预修复 (2026-03-17)

**问题诊断：** 新场景测试发现"分析正确但建议脱节"问题
- **案例：** 相互介绍场景（任务化执行，忽略他人反应）
- **分析质量：** ✅ 优秀（"任务化执行"概括精准，能力缺口聚焦正确）
- **建议质量：** ❌ 严重脱节（照搬"开始信号"游戏，与介绍场景不匹配）

**关键修复：**

1. **场景分类引擎** (`intervention_planner.py`)
   - 新增 5 类场景识别：共同游戏、对话/介绍、规则变化、身体边界、情绪识别
   - 基于情境和行为描述自动分类
   - 关键词匹配 + 优先级排序

2. **场景→干预映射库** (`intervention_planner.py`)
   - 共同游戏 → "开始信号"游戏（判断是否共同游戏）
   - 对话/介绍 → "观察 - 反应"游戏（介绍接力赛）
   - 规则变化 → "预告 + 小变化"游戏（规则变变变）
   - 身体边界 → "身体地图"游戏（身体边界小卫士）
   - 情绪识别 → "情绪侦探"游戏（表情大侦探）

3. **场景化衔接说明**
   - 新增 `scene_bridge` 字段，解释干预与场景的关联
   - 例："针对'介绍时忽略他人反应'这一具体困难，这个练习直接训练'社交信号监测'能力"

**预期效果：**
- 建议匹配准确率：从 60% 提升至 95%+
- 家长实用性评分：从 B+ 提升至 A
- 场景泛化能力：支持 5 类核心社交场景

**文件位置：**
- 场景分类：`behavior_recorder_service/app/agents/intervention_planner.py`
- 映射库文档：`behavior_recorder_service/app/knowledge/intervention_scene_mapping.md`

**核心教训：**
- 分析引擎成熟后，建议生成的精准匹配成为关键瓶颈
- 场景→干预映射需要显式建模，不能依赖隐式泛化
- "诊断正确但开错药"会严重削弱专业可信度

### V4.10.2 干预降级为通用框架 (2026-03-17)

**定位调整：** 工具核心价值是"评估诊断"，干预建议为"辅助参考"

**问题诊断：**
- 干预建议投入产出比低（场景适配复杂，但用户核心需求是评估）
- 具体游戏步骤容易脱节，反而降低专业可信度
- 家长可能误解为"专业干预方案"

**关键修复：**
1. **移除具体游戏步骤** → 保留干预方向和原则
2. **添加免责声明** → 明确"仅供参考，建议咨询专业治疗师"
3. **简化场景衔接** → 通用说明（如"干预重点通常是 X 能力的培养"）

**修改内容：**
- `intervention_planner._get_scene_intervention()`: 简化为通用框架
- `intervention_planner._generate_rigidity_plan()`: 简化为通用框架
- `frontend/App.vue`: 添加免责声明框（红色渐变）
- `frontend/App.vue`: 标题改为"可尝试的行动方向（仅供参考）"

**预期效果：**
- 聚焦核心价值：评估诊断（A+ 级）
- 降低期望：干预建议为参考框架，非具体方案
- 避免风险：明确建议咨询专业治疗师

**文件位置：**
- `behavior_recorder_service/app/agents/intervention_planner.py`
- `behavior_recorder_service/frontend/App.vue`

**核心教训：**
- 工具定位决定功能优先级：评估为主，干预为辅
- "少即是多"：通用框架比具体但脱节的建议更有价值
- 明确边界：免责声明保护用户和开发者
