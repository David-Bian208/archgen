## 临床知识库原则

### 干预模板映射规则
1. **强制一对一映射** - 每个临床假设只能关联到一组预设的干预策略
2. **禁止跨模板调用** - 社交技能不足不能用寻求关注模板，坚持同一性不能用冷静角
3. **安全优先于功能分析** - 危险行为先处理安全，再处理行为功能

### 报告语言风格原则（V4.5.21 新增）
1. **避免通用比喻** - 不用"社交雷达"等模板化表述，用具体行为细节
2. **避免说教感** - 不用"临床上通常考虑几种可能性"，直接分析具体行为
3. **用"我们"视角** - 拉近距离，如"从玥玥的表现来看，我们觉得..."
4. **能力发展视角** - 用"还没学会"而不是"不会"，用"练习"而不是"训练"
5. **短句优先** - 每句<30 字，像聊天不像写论文

### 测试覆盖标准
- 每类别 ≥15 题
- 整体通过率 ≥95% 才可部署
- P0 案例必须 100% 通过
- 回归测试自动化集成到 CI/CD

## 开发原则

### 文本 > 大脑 📝
- 所有重要决策必须写入文件
- "心理笔记"无法在 session 重启后存活
- 错误必须文档化，防止未来重复

### 安全 > 完成
- 临床系统优先保证输出安全
- 不确定时降级到保守方案
- 人工审核队列是必要的安全网

### 自动化 > 手动
- 关键测试必须自动化
- 回归测试集成到 CI/CD
- 手动测试无法保证每次都执行

## 记忆系统设计（2026-04-02）

**问题背景：** OpenClaw"失忆"，昨天的工作大部分丢失，依赖对话上下文作为记忆不可靠

**核心洞察：**

1. **三层记忆架构**
   - **OpenClaw（战舰）** → 永久记忆（协调中心，必须记住项目全貌）
   - **TRAE（小治/小测试）** → 弱记忆（可访问共享文档，不需要记住跨会话内容）
   - **共享记忆库** → 所有人可访问（`docs/`、`memory/`、`skills/`）

2. **记录时机判断**
   - ✅ **需要记录：** 最终决策、达成的规范、定义的"事实"、已确认的待办
   - ❌ **无需记录：** 头脑风暴、被否决的路径、辩论细节、简单指令确认
   - 🎯 **核心原则：** 记录价值，而非过程

3. **自动化触发机制（黄金规则）**
   - **指令下达 = 记录触发信号**
   - 派发任务给 TRAE 时，自动记录决策摘要
   - 条件反射流程：派发任务 → 自动记录 → 通知 TRAE 参阅文档

4. **6 词框架作为软件设计核心原则**
   - **Scope（范围）** - 做什么，不做什么
   - **Priority（优先级）** - 先做什么，后做什么
   - **Constraint（约束）** - 必须用什么，不能改什么
   - **Quality（质量）** - 什么叫好、易维护
   - **Risk（风险）** - 哪些坑必须防
   - **Validation（验证）** - 怎么知道做对了
