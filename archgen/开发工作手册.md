## 工作流说明

### 前端 7 步状态机

| Step | 页面/组件 | 操作 |
|------|----------|------|
| 0 | MCPSearchView → TopicSuggestView | MCP 素材扫描 → 选题推荐 |
| 1 | DirectionSuggestView → WorkflowView(Step 1) | 方向分析 → 角度选择 |
| 2 | WorkflowView(Step 2) | 框架匹配 → 第1次补充 |
| 3 | WorkflowView(Step 3 / ThreeColumnWorkbench) | 槽位生成 → 素材匹配 → 填充 → 确认 |
| 4 | WorkflowView(Step 4) | 提纲整合 → 第2次补充 |
| 5 | WorkflowView(Step 5) | 文章生成 |
| 6 | WorkflowView(Step 6) | 配图生成（旧功能保留） |

### Session 状态管理

前端通过 `useWorkflowState.js` composable 管理全部状态，通过 `session_id` 关联后端。

## 常见问题排查

### 启动问题

```
# 后端
python main.py  # 默认端口 8953

# 前端开发
cd frontend && npm run dev  # 默认端口 3014
```

### LLM 调用失败

1. 检查 config.yaml 中 `llm.api_key` 是否存在
2. 检查 API 端点是否可达：`curl https://api.deepseek.com/v1/models`
3. 检查超时设置（默认 120s）

### AI-Pulse 连接失败

1. 检查 AI-Pulse 服务是否运行：`curl http://8.130.148.166:8887/api/health`
2. 检查 config.yaml 中 `ai_pulse.enabled` 是否为 true
3. 不影响主流程，素材匹配会走关键词降级

### Session 找不到

- 前端报错"会话不存在"：检查 session_id 是否正确传递
- 后端 session 存储在内存，服务重启后所有 session 丢失
- 需要重启前导出 session，或接受丢失

### 素材匹配不准确

1. 检查素材池内容：`session["material_pool"]`
2. 检查槽位 label 和 desc 是否足够具体
3. 关键词匹配使用拆词+bigram+单字重叠度三特征
4. AIPULSE 素材用 LLM 语义分配
