## 一、产品概述

### 1.1 产品名称
**ArchGen v4.0 — 三列分析工作台**（Step 3 of 5）

### 1.2 一句话描述
用户提交创作选题后，AI 智能拆解分析维度（槽位），自动匹配素材并生成提纲，用户在三列视图中查看全局、精调细节，一键生成全文。

### 1.3 目标用户

| 用户画像 | 典型场景 | 核心需求 |
|----------|----------|----------|
| 独立创作者 | 写行业分析文章 | 不知道怎么拆解话题，需要 AI 给框架 |
| 内容运营 | 周报/月报/专题 | 素材分散（本地文件+搜索+手写），需要统一视图 |
| 研究员 | 商业分析报告 | 关注来源可追溯，不想 AI 编造内容 |

## 二、完整工作流（5 步）

```
① 选题 → ② 补充（写作角度+素材管理） → ③ 槽位工作台 → ④ 生成文章 → ⑤ 生成配图
```

### 2.1 各步骤职责

| 步骤 | 名称 | 做什么 | 产出 |
|------|------|--------|------|
| ① | 选题 | KB 扫描 → 方向评分 → 用户确认 | 选定的写作方向 |
| ② | 补充 | 推荐写作角度 → 素材增删 → AI补充 → 缺失检测 | 写作角度 + 素材池 |
| ③ | 槽位工作台 | 按写作角度拆槽位 → 素材填充 → 细则修改 | 填充完成的框架 |
| ④ | 生成文章 | 提纲展开为全文 | 完整文章 |
| ⑤ | 配图 | 生成配图并嵌入 | 配图文章 |

### 2.2 Step 2（补充）与 Step 3（槽位工作台）的边界

| 维度 | Step 2 补充 | Step 3 槽位工作台 |
|------|-----------|-----------------|
| 粒度 | 粗：决定"文章什么类型" | 细：决定"每个段落写什么" |
| 类比 | 选户型（三室/两室/复试） | 填装修细节（客厅墙色/厨房台面） |
| 产出 | 写作策略 + 匹配的素材集 | 填充完成的框架模板 |
| 关系 | 选了角度 → 决定了素材需求 → 决定了框架倾向 | 框架定了 → 用素材逐槽填充 |

**具体例子**：选题"AI赋能中层管理者"

| 写作角度 | 主旨 | Step 3 框架倾向 |
|----------|------|----------------|
| 工具盘点型 | 推荐10款AI工具 | 功能清单结构 |
| 案例研究型 | 拆解3家企业实践 | 背景→案例对比→关键发现→落地建议 |
| 方法论型 | 5步落地框架 | 步骤拆解结构 |

### 2.3 Step 3 的 3 个阶段

```
阶段 1：流式推理槽位（AI 打字机输出 + 用户增删改）
  ↓ 用户点「确认槽位」
阶段 2：素材匹配 + 提纲生成（槽位独立、逐行出现）
  ↓ 全部槽位生成完成
阶段 3：精调（编辑面板：写作方案 + 补充素材 + 微调提纲）
  ↓ 用户点「生成全文」
进入 Step 4
```

## 三、核心功能列表

### MVP（必须做）

| ID | 功能 | 说明 | 交互入口 |
|----|------|------|---------|
| F1 | AI 流式推理槽位 | 进入 Step 3 后，AI 打字机效果输出分析思路 → 推荐 4-6 个维度 | 自动触发 |
| F2 | 槽位增删改 | 用户修改维度名称 / 删除不需要的 / 新增手动维度 | 槽位列表行内编辑 |
| F3 | 素材自动匹配 | 确认槽位后，AI 从素材池（知识库+搜索+手动）按关键词匹配到各槽位 | 自动触发 |
| F4 | 提纲自动生成 | 确认槽位后，AI 根据主题+匹配的素材生成提纲要点 | 自动触发 |
| F5 | 三列表格展示 | 槽位 / 素材（来源标签）/ 提纲 三列并行展示 | 确认槽位后渲染 |
| F6 | 编辑面板 | 右侧滑出：写作方案（textarea）+ 补充素材（文本+文件上传）+ 微调提纲 | 每行 [编辑] 按钮 |
| F7 | 槽位独立填充 | 每个槽位的素材+提纲独立生成，不互相等待（先完成的先展示） | 自动触发 |
| F8 | 文件/图片上传 | 编辑面板内上传文件（.txt/.md/.pdf/.docx），图片用 PaddleOCR 提取文字 | 编辑面板按钮 |
| F9 | 步骤条精简 | 8 步 → 5 步（选题/补充/分析工作台/文章/配图） | 左侧固定 |
| F10 | 生成全文跳转 | 全部槽位确认后，点按钮跳 Step 4 | 底部按钮 |
| F11 | 槽位间关系图谱 | 流式推理完成后，展示槽位间逻辑依赖关系（简易连线图），用户确认前可看到 | 槽位清单上方（可折叠） |
| F12 | 编辑面板追问 AI | 编辑面板内追问，AI 回复后可采纳/忽略 | 编辑面板追问区 |
| F13 | 槽位确认阶段追问 | 流式输出完、确认前追问 AI（如"为什么是5个维度？""XX和YY是否重叠？"）| 槽位清单下方追问输入框 |
| F14 | 素材可行性预检 | 确认槽位前，快速统计每个槽位能匹配到几条素材。素材不足的槽位 🟡/🔴 显示，AI 推荐替代维度 | 流式推理完成后自动触发 |

## 四、界面设计要求

### 4.1 布局规范

```
┌──────────────────────────────────────────────────────────────┐
│ 左侧步骤条 (180px)  │  主内容区 (flex:1)    │  编辑面板(420px)│
│                     │                       │  absolute 滑出  │
│ ○ 选题              │  Step 3/5 三列工作台  │                 │
│ ✅ 补充              │  ┌──────────────────┐ │  ┌───────────┐  │
│ 🔄 分析工作台        │  │ AI 流式分析区域   │ │  │ 编辑面板  │  │
│ ○ 文章              │  ├──────────────────┤ │  │           │  │
│ ○ 配图              │  │ 三列表格         │ │  │ 写作方案  │  │
│                     │  │ 槽位│素材│提纲    │ │  │ 补充素材  │  │
│                     │  └──────────────────┘ │  │ 微调提纲  │  │
└──────────────────────────────────────────────┴──────────────┘
```

### 4.2 三列宽度

| 列 | 宽度 | 出界处理 |
|----|------|---------|
| # + 状态灯 | 60px | 固定 |
| ① 槽位 | 15% (min 120px) | 文字过长用省略号 + tooltip |
| ② 素材 | 35% | 每条素材一行，超 4 条可折叠 |
| ③ 提纲 | 35% | 每条要点一行，超 3 条可折叠 |
| 操作 | 120px | 固定 |

### 4.3 状态标识

| 状态 | 图标 | 条件 |
|------|------|------|
| 完整 | 🟢 | 素材 ≥ 1 条 且 提纲 ≥ 1 条 且 缺口 = 0 |
| 缺部分 | 🟡 | 素材或提纲不足 |
| 空 | 🔴 | 素材 = 0 且 提纲 = 0 |
| 已确认 | ✅ | 用户点过「确认」 |

### 4.4 素材列来源标签

| 来源 | 图标 | 说明 |
|------|------|------|
| 知识库 | 📄 | 本地文档匹配 |
| 用户输入 | ✏️ | 手动补充文本或上传文件 |
| 网络搜索 | 🌐 | MCP 或网络搜索结果 |
| AI 推断 | 🤖 | AI 推理生成（标注为推断，需用户审核） |

### 4.5 响应式

- **≥ 1024px**：左侧步骤条 + 三列表格
- **768-1024px**：隐藏步骤条，表格完整显示
- **< 768px**：退化为纵向卡片模式（每槽位一张卡片）

## 五、交互规范

### 5.1 流式推理

- 进入 Step 3 后自动触发
- 打字机速度：**每行间隔 200-400ms**（模拟思考节奏）
- 用户可随时**打断**（点「确认槽位」按钮，流式提前结束）
- 流式完成后，推理文本保留在页面上方（可折叠）

### 5.2 槽位编辑

- 每个槽位名是 `inline-edit` 输入框
- 焦点离开时自动保存
- 删除需二次确认（"确定删除此分析维度？"）

### 5.3 槽位独立生成

- 确认槽位后，每个槽位**独立调用** `batch_fill_v4`
- 生成完成的槽位**立刻显示**内容，其他槽位显示 `a-spin` 加载
- 不阻塞已完成槽位的编辑操作

### 5.4 编辑面板

- `@click="openEditPanel(slotKey)"` → 右侧 `transform: translateX` 滑入
- 面板内三个区域：
  1. **写作方案**：textarea，自动高度
  2. **补充素材**：textarea + 文件上传按钮 + 图片上传按钮
  3. **微调提纲**：可编辑列表（每项可改文字、可删、可加）
- 点[保存]：写入 session，关闭面板，表格行刷新
- 点[取消]或点面板外：关闭面板，不保存

### 5.5 错误处理

| 场景 | 处理 |
|------|------|
| SSE 断连 | 前端自动重试 2 次，失败后显示「连接中断」+ 手动重试按钮 |
| 素材匹配为空 | 素材列显示「暂无匹配素材」+ [手动补充] 按钮 |
| 提纲生成为空 | 提纲列显示「AI 未能生成提纲」+ [手动编写] 按钮 |
| LLM 返回超时 (30s) | 显示「生成超时，请重试」 |

## 七、验收标准（MVP）

- [ ] F1: 进入 Step 3 → AI 打字机流式输出推理文本
- [ ] F2: 槽位可增删改（改名称 / 删 / 新建）
- [ ] F3: 确认槽位后 → 素材列自动填充（带 📄✏️🌐 来源标签）
- [ ] F4: 确认槽位后 → 提纲列自动填充（带有序要点）
- [ ] F5: 三列同时可见（槽位 / 素材 / 提纲）
- [ ] F6: 点击 [编辑] → 右侧滑出编辑面板，含写作方案 / 补充素材 / 微调提纲三个区域
- [ ] F7: 槽位 1 生成完即显示，不等待槽位 2
- [ ] F8: 编辑面板内上传文件成功，素材列刷新
- [ ] F9: 左侧步骤条 5 步：选题 → 补充 → 分析工作台 → 文章 → 配图
- [ ] F10: 全部确认后 [生成全文] 可用，点击跳 Step 4
- [ ] F11: 流式完成后显示槽位间关系图谱（简易连线图，可折叠）
- [ ] F12: 编辑面板内可追问 AI，回复后可采纳/忽略
- [ ] F13: 槽位确认阶段可追问 AI（如"为什么是5个维度？"）
- [ ] 响应式：768px 以下退化为卡片模式
