## 二、已完成模块

### W1: AI-Pulse 素材分配改用 LLM 语义匹配

**问题**：`_match_materials_internal` 用 label/desc 关键词匹配 AI-Pulse 素材。AI-Pulse 新闻标题「AI 行业周报第 47 期」与槽位 label「风险管理」无关键词重叠 → 20 条素材全部分配失败。

**方案**：在 `batch_fill_v4` 中新增 LLM 语义分配调用。

**改动文件**：

| 文件 | 改动 |
|------|------|
| `archgen/api.py` L7362-L7472 | 新增 `_match_aipulse_by_llm()` 函数 |
| `archgen/api.py` L8036 | `batch_fill_v4` 中调用 `await _match_aipulse_by_llm(...)` |

**技术细节**：
- 收集所有 AIPULSE 素材标题+摘要 + 所有槽位 label
- 一次 LLM 调用完成所有分配（temperature=0, seed=42）
- LLM 分配的分值设为 99（最高优先级）
- 失败不影响主流程（返回原 slot_materials）

### W3: 联网兜底改为按槽位按需搜索

**问题**：联网搜索只在 batch_fill_v4 触发，编辑面板内无法使用，场景不明确。

**方案**：联网兜底整合到 EditPanel 弹窗内。

**新端点**：`POST /api/workflow/slot/web_search`

**触发条件**：槽位素材数 < 3 时显示按钮和提示。

| 素材数 | 显示 |
|--------|------|
| ≥ 3 | ✅ 素材充足 |
| 1-2 | 🟡 素材偏少，建议 🌐 联网搜索补充 |
| 0 | 🔴 无素材，强烈建议 🌐 联网搜索补充 |

**流程**：点击 → 用方向标题+槽位 label 搜 DuckDuckGo → 返回结果 → 逐条 [➕ 添加]

| 文件 | 改动 |
|------|------|
| `archgen/api.py` L8120-L8168 | 新增 `web_search_slot` 端点 |
| `frontend/src/utils/api.js` | 新增 `searchWebForSlot()` |
| `frontend/src/components/workflow/EditPanel.vue` | 联网搜索按钮 + 结果列表 + 素材评估 |
| `frontend/src/composables/useStep3_Workbench.js` | 新增 `handleWebSearchSlot()` |

3. **前端代理**：`vite.config.js` 已改为 `http://localhost:8952`，部署时根据实际后端端口调整。

4. **LLM API Key**：`integrate_outline` 和 `_match_aipulse_by_llm` 依赖 config.yaml 或环境变量中的 `llm.api_key`。
