## 问题诊断（来自临床 - 工程双维度评估）

### 核心问题：两个"大脑"在打架

**大脑 A（临床推理引擎）**：能做出正确诊断（如"寻求关注"）  
**大脑 B（对话状态机）**：机械执行鉴别问题收集

**症状**：
- 用户说"故意捣乱吸引注意"（强烈指向寻求关注）
- 系统仍反复问"如果有人提醒或示范，他能做好吗？"（提示依赖鉴别题）
- 像"事后诸葛亮"——分析正确，但问诊过程笨拙

### 工程表现

```
案例 12/13 日志：
asked_fields={'immediate_consequence': 2}  ← 已问过 2 次
但仍继续提问同一个字段  ← 状态机无法否决推理引擎
```

## V4.5.11 修复方案

### 1. 假设驱动的动态提问策略

**新增逻辑**：当某假设置信度 > 0.8 时，切换到假设驱动模式

```python
# 检测强假设
strong_hypotheses = [(hid, h.current_confidence) 
                     for hid, h in self.network.hypotheses.items() 
                     if h.current_confidence > 0.8]

if strong_hypotheses:
    # 直接追问该假设的特异性问题
    strong_hyp_id, strong_conf = max(strong_hypotheses, key=lambda x: x[1])
    specific_question = self._find_hypothesis_specific_question(strong_hyp_id, ...)
```

**效果**：
- 用户说"吸引注意" → attention_seeking 置信度飙升 → 直接问"您关注他后，他是什么反应？"
- 不再机械执行通用鉴别流程

### 2. 安全出口机制

**触发条件**：
- 基础字段填充 ≥ 3 个（antecedent, behavior, consequence, age）
- 某问题问过 2 次以上

```python
basic_fields = ["antecedent", "behavior_detail", "immediate_consequence", "child_age"]
basic_filled = sum(1 for f in basic_fields if filled_data.get(f))
max_asked_count = max(asked_fields.values()) if asked_fields else 0

if basic_filled >= 3 and max_asked_count >= 2:
    logger.warning("⚠️ 安全出口触发，强制生成报告")
    return {"type": "GENERATE_REPORT", "reason": "safety_exit"}
```

### 3. 强制尊重 asked_fields

**策略**：1 次警告，2 次跳过，优先选择未问过的问题

```python
# 排序：先按 asked_count 升序（未问过的优先），再按 uncertainty 降序
candidate_questions.sort(key=lambda x: (x[3], -x[2]))
```

### 4. 传递 conversation_history

让推理引擎能全面分析对话上下文，做出更智能的决策。
