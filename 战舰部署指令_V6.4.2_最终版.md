## 🔧 部署步骤

### Step 1：停止旧服务

```bash
# 停止 3000 端口的 Chainlit 服务
fuser -k 3000/tcp 2>/dev/null
sleep 2

# 停止 8000 端口的中台服务（如果存在）
fuser -k 8000/tcp 2>/dev/null
sleep 2
```

### Step 2：上传文件

将以下文件上传到服务器对应目录：

| 文件 | 服务器路径 | 说明 |
|------|-----------|------|
| `app.py` | `~/behavior_recorder_service/chainlit_v62/app.py` | V6.4.2 主应用 |
| `abc_prompts.py` | `~/behavior_recorder_service/chainlit_v62/abc_prompts.py` | ABC 引导 prompt 模板（⚠️ 必须同步上传） |
| `admin_dashboard.py` | `~/behavior_recorder_service/admin_dashboard.py` | 中台管理服务 |
| `config.toml` | `~/behavior_recorder_service/chainlit_v62/.chainlit/config.toml` | Chainlit 配置（含白色主题设置） |

### Step 3：清理 Python 缓存

```bash
cd ~/behavior_recorder_service

# 确保日志目录存在
mkdir -p logs

# 清理缓存
find . -type d -name '__pycache__' -exec rm -rf {} + 2>/dev/null
find . -name '*.pyc' -delete 2>/dev/null
echo "缓存已清理，logs 目录已确认"
```

### Step 4：启动中台服务（端口 8000）

```bash
cd ~/behavior_recorder_service

# 确保 data 目录存在
mkdir -p data

# 启动中台服务
nohup python3 admin_dashboard.py > logs/admin_dashboard.log 2>&1 &
echo "中台服务已启动，PID: $!"

# 验证
sleep 3
curl http://localhost:8000/api/stats
```

**预期输出：**
```json
{"total": 0, "today": 0, "avg_time": 0, "users": 0}
```

### Step 5：启动 Chainlit 服务（端口 3000）

```bash
cd ~/behavior_recorder_service/chainlit_v62

# 启动 Chainlit
nohup chainlit run app.py --port 3000 > logs/chainlit.log 2>&1 &
echo "Chainlit 服务已启动，PID: $!"

# 验证
sleep 5
curl http://localhost:3000/health
```

**预期输出：**
```json
{"status":"ok"}
```

### Step 6：验证服务

```bash
# 检查端口占用
echo "=== 端口 3000 ==="
lsof -i:3000

echo "=== 端口 8000 ==="
lsof -i:8000

# 查看日志（最近 20 行）
echo "=== Chainlit 日志 ==="
tail -20 logs/chainlit.log

echo "=== 中台日志 ==="
tail -20 logs/admin_dashboard.log
```

## 📊 中台功能说明

### 统计数据
| 指标 | 说明 |
|------|------|
| 总对话数 | 历史所有对话总数 |
| 今日对话 | 今天的对话数 |
| 平均响应 | 平均每次分析的响应时间（秒） |
| 用户数 | 不同的 session 数量 |

### 数据列表
- 卡片式会话展示（用户提问 + AI 回复配对显示）
- 支持搜索（问题/回答内容）
- 支持按场景类型筛选（A/B/C/D/E）
- 分页显示（每页 10 条）
- 未生成回答的对话会显示 "⚠️ 未生成回答"

### 数据导出
- 导出为 CSV 格式
- 包含：时间、场景类型、假设、用户问题、助手回答、响应时间

```bash
# 检查数据库文件
```
