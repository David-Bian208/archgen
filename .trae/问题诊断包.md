# 🚨 问题诊断包（给 trae 的指令）

**当你和 DAVID 沟通遇到以下情况时，使用本诊断包：**

1. 沟通超过 3 轮仍未解决问题
2. 代码修改后问题依然存在
3. DAVID 要求你总结问题
4. 你需要第三方（OpenClaw/战舰）协助

## 🛠️ 自动收集脚本

**运行此脚本自动收集诊断信息：**

```bash
#!/bin/bash
# 问题诊断包自动收集脚本

OUTPUT_DIR="/tmp/trae-diagnosis-$(date +%Y%m%d-%H%M%S)"
mkdir -p $OUTPUT_DIR

echo "🔍 收集诊断信息..."

# 1. Git 历史
echo "📝 收集 Git 历史..."
git log --oneline -10 > $OUTPUT_DIR/git-history.txt
git diff HEAD~5 > $OUTPUT_DIR/git-diff.txt

# 2. 最近修改的文件
echo "📁 收集最近修改的文件..."
git status > $OUTPUT_DIR/git-status.txt
find . -name "*.py" -o -name "*.vue" -o -name "*.js" | xargs ls -lt | head -20 > $OUTPUT_DIR/recent-files.txt

# 3. 日志
echo "📋 收集日志..."
tail -100 server.log > $OUTPUT_DIR/server.log
tail -100 frontend/vite.log > $OUTPUT_DIR/frontend.log
tail -100 analytics.log > $OUTPUT_DIR/analytics.log

# 4. 测试报告
echo "🧪 运行测试..."
python3 -m pytest tests/ -v --tb=short > $OUTPUT_DIR/test-report.txt 2>&1

# 5. 覆盖率审计
echo "📊 运行覆盖率审计..."
python3 tests/parity_audit.py > $OUTPUT_DIR/coverage-audit.txt 2>&1

# 6. 创建诊断包
echo "📦 创建诊断包..."
cat > $OUTPUT_DIR/问题诊断包.md << EOF
# 问题诊断包

**生成时间：** $(date)
**生成方式：** 自动收集

## 包含的文件

1. git-history.txt - Git 提交历史
2. git-diff.txt - 最近代码变更
3. git-status.txt - 当前 Git 状态
4. recent-files.txt - 最近修改的文件
5. server.log - 后端日志
6. frontend.log - 前端日志
7. analytics.log - 分析日志
8. test-report.txt - 测试报告
9. coverage-audit.txt - 覆盖率审计

## 下一步

1. 填写问题描述（见模板）
2. 打包发送给 DAVID/战舰
3. 等待进一步指示

## 打包命令

\`\`\`bash
cd $OUTPUT_DIR
tar -czf diagnosis.tar.gz *
\`\`\`

**诊断包位置：** $OUTPUT_DIR/diagnosis.tar.gz
EOF

echo "✅ 诊断包生成完成：$OUTPUT_DIR/问题诊断包.md"
echo "📦 压缩包位置：$OUTPUT_DIR/diagnosis.tar.gz"
```

## 📤 发送方式

### 方式 1：直接发送 Markdown

**适用于：** 对话不太长（<1000 行）

**步骤：**
1. 填写诊断包模板
2. 复制粘贴到对话

### 方式 2：发送压缩包

**适用于：** 对话很长或包含多个文件

**步骤：**
1. 运行自动收集脚本
2. 填写问题描述部分
3. 发送压缩包路径给 DAVID

### 方式 3：请求 OpenClaw 协助

**适用于：** 复杂问题需要第三方分析

**步骤：**
1. 运行自动收集脚本
2. 发送命令给 OpenClaw：
   ```
   请分析这个诊断包：[路径]
   问题：[简要描述]
   ```
