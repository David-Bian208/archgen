# 多窗口协作规范

**用途：** 规范 trae 在多个任务窗口中并行工作

**重要性：** ⭐⭐⭐⭐⭐ **多窗口时必须遵守**

---

## 🎯 核心原则

**每个窗口都是独立的会话，有独立的上下文。**

**为避免混淆，必须：**
1. 明确身份（我是谁）
2. 明确任务（在做什么）
3. 明确状态（进度如何）

---

## 📋 身份标识规范

### 窗口命名

**格式：**
```
[角色] 任务名称
```

**示例：**
```
[Dev-Frontend] 干预建议 UI 优化
[Dev-Backend] API 接口修复
[Dev-Test] 测试用例编写
```

---

### 回复格式

**每次回复开头必须说明：**

```markdown
**[窗口身份]** [任务名称]

[回复内容]
```

**示例：**
```markdown
**[Dev-Frontend] 干预建议 UI 优化**

已完成 UI 简化，删除了底部评价区域...
```

## 📊 任务拆解示例

### 示例：干预建议优化

**总任务：** 优化干预建议功能

**拆解为 3 个窗口：**

#### 窗口 1：[Dev-Frontend] UI 优化

**任务：**
- 简化干预建议 UI
- 删除复杂装饰
- 优化加载状态

**指令：**
```markdown
# 任务：干预建议 UI 优化

**窗口身份：** [Dev-Frontend]

**目标：** 简化 UI，删除装饰

**禁令：**
1. 不修改后端接口
2. 不删除功能

**验收：**
- UI 简洁
- 功能正常
```

---

#### 窗口 2：[Dev-Backend] API 优化

**任务：**
- 优化 API 响应速度
- 添加缓存
- 优化字段格式

**指令：**
```markdown
# 任务：干预建议 API 优化

**窗口身份：** [Dev-Backend]

**目标：** 提升响应速度到<5 秒

**禁令：**
1. 不改变 API 契约
2. 不影响现有功能

**验收：**
- 响应时间<5 秒
- 测试通过
```

---

#### 窗口 3：[Dev-Test] 测试覆盖

**任务：**
- 编写 UI 测试
- 编写 API 测试
- 编写集成测试

**指令：**
```markdown
# 任务：干预建议测试覆盖

**窗口身份：** [Dev-Test]

**目标：** 覆盖率>90%

**禁令：**
1. 不修改业务逻辑
2. 测试必须独立运行

**验收：**
- 覆盖率>90%
- 测试全部通过
```

## 🎯 窗口间协调

### 需要协调的场景

**当前端依赖后端时：**

```markdown
**[Dev-Frontend] UI 优化**

**需要协调：**
- 需要 [Dev-Backend] 提供新的 API 字段
- 字段名：`intervention.goal`
- 期望格式：字符串

**请 [Dev-Backend] 确认是否可以提供**
```

---

**当后端变更影响前端时：**

```markdown
**[Dev-Backend] API 优化**

**API 变更：**
- 新增字段：`intervention.goal`
- 类型：字符串
- 必填：是

**请 [Dev-Frontend] 更新 UI 以显示新字段**
```

---

### 协调格式

**发起协调：**
```markdown
**[窗口身份] [任务名称]**

**需要协调：**
- 需要 [目标窗口] 配合
- 配合内容：[详细说明]
- 期望时间：[如有时间要求]

**请确认：**
- [ ] [目标窗口] 可以配合
- [ ] 需要讨论
- [ ] 无法配合（原因）
```

---

**响应协调：**
```markdown
**[目标窗口身份] [任务名称]**

**协调响应：**
- 可以配合：是/否
- 预计完成：[时间]
- 需要说明：[如有]
```

## 🚫 多窗口禁令

### 禁止事项

1. ❌ **禁止混淆窗口身份**
   - 每个窗口必须明确自己的身份
   - 回复时必须说明身份

2. ❌ **禁止跨窗口修改代码**
   - 前端窗口只修改前端代码
   - 后端窗口只修改后端代码

3. ❌ **禁止不协调就变更接口**
   - API 变更必须通知前端窗口
   - 前端变更必须通知后端窗口

4. ❌ **禁止不等待依赖窗口**
   - 如有依赖，必须等待对方完成
   - 不要假设对方的实现
