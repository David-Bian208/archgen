## 架构对比

### V3.9.2 (旧架构)

```
用户输入 → _update_state(LLM) → _decide_next_question(代码) → 
           _evaluate_conversation(LLM) → 生成回应
           
问题：
- 2 次 LLM 调用/轮 (效率低)
- 状态更新依赖 LLM (不稳定)
- is_complete 由 LLM 主观判断 (不可控)
- 状态转移与回应生成脱节
```

### V4.0 (新架构)

```
用户输入 → _update_collected_data(代码) → 
           state.transition(代码) → 
           _build_prompt → LLM.generate(1 次) → 
           _should_generate_report(代码) → 生成回应/报告
           
优势:
- 1 次 LLM 调用/轮 (高效)
- 状态转移由代码控制 (确定性)
- 完成条件明确 (ABC 完整 + 证据充足)
- 集权式主控流程
```

## 核心文件

### 1. app/agents/guided_recorder_v4.py

**核心类**:
- `InterviewState`: 状态枚举 (WELCOME, GATHER_CONTEXT, ..., COMPLETE)
- `ClinicalInterviewState`: 状态机数据类
  - `should_transition_to()`: 纯代码逻辑的状态转移
  - `is_data_complete()`: ABC 信息完整性检查
- `GuidedRecorderV4`: 主控制器
  - `process()`: 集权式主控流程 (6 步)
  - `_should_generate_report()`: 代码级完成条件

**关键代码片段**:

```python
# 状态转移逻辑 (纯代码，不调用 LLM)
def should_transition_to(self, user_input: str) -> Optional[InterviewState]:
    if self.current_state == InterviewState.GATHER_CONTEXT:
        if self.collected_data.get("context", "").strip():
            return InterviewState.GATHER_BEHAVIOR
    
    if self.current_state == InterviewState.GATHER_RESPONSE:
        if self.collected_data.get("response", "").strip():
            if self._should_deep_dive():
                return InterviewState.HYPOTHESIS_DRIVEN
            else:
                return InterviewState.READY_TO_ANALYZE
    
    return None

# 完成条件 (代码逻辑)
def _should_generate_report(self, state) -> bool:
    abc_complete = state.is_data_complete()
    enough_evidence = len(state.evidence_log) >= 3
    max_turns = state.turn_count >= 5
    return abc_complete and (enough_evidence or max_turns)
```

## 状态流转图

```
WELCOME
   ↓ (用户描述行为)
GATHER_CONTEXT
   ↓ (情境已收集)
GATHER_BEHAVIOR
   ↓ (行为已收集)
GATHER_RESPONSE
   ↓ (回应已收集)

   │        ↓ (证据充足)
   └─→ READY_TO_ANALYZE
            ↓
       ANALYZING
            ↓
       COMPLETE
```

## 技术亮点

### 1. 确定性状态转移

```python
# V3.9.2: 依赖 LLM 主观判断
result = self.llm.generate_json(...)

# V4.0: 代码逻辑控制
if self.collected_data.get("response", "").strip():
    return InterviewState.READY_TO_ANALYZE  # ✅ 确定性
```

### 2. 数据收集归类

```python
# 根据当前状态，将用户输入归类到对应字段
if current == InterviewState.GATHER_CONTEXT:
    state.collected_data["context"] = user_input
elif current == InterviewState.GATHER_BEHAVIOR:
    state.collected_data["behavior"] = user_input
elif current == InterviewState.GATHER_RESPONSE:
    state.collected_data["response"] = user_input
```
