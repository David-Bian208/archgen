## 一、问题诊断回顾

基于《V4.6.6_报告对比分析与优化指令.md》中的全方位对比分析，识别出三个层次的问题：

### 🔴 P0 致命问题：摘要矛盾
- **症状**：摘要归因"工作记忆挑战"，后文分析"社交认知"
- **后果**：破坏专业信任，家长困惑
- **根因**：`_generate_summary()` 使用 `inferred_hypotheses` 而非 `insight_result.capability_hypothesis`

### 🟠 P1 结构问题：临床推理缺失
- **症状**：只有结论，没有展示"如何排除其他可能性"
- **后果**：家长无法理解推理过程，降低报告可信度
- **根因**：Prompt 中缺少强制的三模块结构要求

### 🟡 P2 表达问题：比喻普通
- **症状**："读懂其他小朋友"vs"沉浸在自己剧本里的导演"
- **后果**：未能构建深刻的理解框架
- **根因**：缺少"问题类型→比喻库"映射机制

## 二、修复详情

### 1. P0 修复 - 摘要生成逻辑

**文件：** `behavior_recorder_service/app/agents/structured_assessor_v4.py`

#### 1.1 `_generate_summary()` 方法优化

**修复前：**
```python
# 获取首要假设
top_hypothesis = max(state.inferred_hypotheses.items(), key=lambda x: x[1], default=(None, 0))
# 问题：inferred_hypotheses 可能是 prompt_dependence，但实际是社交技能不足
```

**修复后：**
```python
# V4.10.0 P0 修复：优先使用 insight_result 中的能力缺口和功能判断
insight = state.insight_result or {}
capability_gap = insight.get("capability_hypothesis", "")
functional_judgment = insight.get("functional_judgment", "")

# 确定用于摘要生成的"首要假设"（优先级：capability_gap > functional_judgment > inferred_hypotheses）
if capability_gap and any(kw in capability_gap for kw in ["社交", "换位思考", "灵活性", "情绪识别", "观点采择"]):
    primary_hypothesis = "社交技能不足"
elif capability_gap and any(kw in capability_gap for kw in ["规则", "僵化", "同一性"]):
    primary_hypothesis = "坚持同一性"
# ... 更多判断逻辑
```

**核心原则：** 摘要中的能力归因必须与后文分析严格一致

#### 1.2 `_generate_core_challenge()` 方法增强

**新增支持：**
- 社交技能不足的细分匹配（社交信号监测弱、认知灵活性不足、身体边界感、观点采择）
- 坚持同一性/刻板行为的独立处理
- 避免错误归因（如社交技能不足案例返回"工作记忆挑战"）

### 2. P1 修复 - 临床推理结构

**文件：** `behavior_recorder_service/app/agents/insight_analyzer.py`

#### 2.1 强化 SYSTEM PROMPT 中的鉴别诊断要求

**新增三模块强制结构：**

```markdown
【鉴别诊断结构要求 - V4.10.0 P1 强制】
**必须**包含以下三个子模块，缺一不可：

**1. 鉴别与排除**（临床推理过程）
- 格式：使用"为何不是 XXX"的明确排除句式
- 内容：针对至少 2 个竞争性假设，说明排除理由

**2. 核心假设**（专业概括）
- 格式：用专业术语命名行为模式
- 内容：一句话概括核心行为模式 + 简要解释

**3. 能力缺口分析**（从现象到归因）
- 格式：每个缺口必须包含"现象→归因"的完整链条
- 内容：从具体行为表现推导出 2-3 个最关键的能力缺口
```

#### 2.2 更新 JSON 输出格式要求

**`clinical_differential` 字段新格式：**
```json
{
  "clinical_differential": "**必须包含三模块结构**：\n\n1. **鉴别与排除**：\n   • 为何不是'提示依赖'：[具体理由]\n   • 为何不是'寻求关注'：[具体理由]\n\n2. **核心假设**：[专业术语命名行为模式 + 简要解释]\n\n3. **能力缺口分析**：\n   1. [缺口名称]：[具体行为现象]→[能力归因]\n   2. [缺口名称]：[具体行为现象]→[能力归因]"
}
```

### 3. P2 修复 - 比喻生成优化

**文件：** `behavior_recorder_service/app/agents/structured_assessor_v4.py`

#### 3.1 `_generate_ability_pattern()` 方法比喻增强

**建立"问题类型→比喻库"映射：**

| 问题类型 | 比喻 | 示例场景 |
|----------|------|----------|
| 社交信号监测弱 | "沉浸在自己剧本里的导演" | "以为别人在和她玩" |
| 认知灵活性不足 | "只能播放三首歌的收音机/单曲循环" | "规则只有 3 条，不停重复" |
| 坚持同一性 | "严格按剧本演出的演员" | "固定路线、仪式行为" |
| 提示依赖 | "建筑需要脚手架支撑" | "有提示则执行，无提示则中断" |
| 寻求关注 | "需要站在聚光灯下的表演者" | "成人注意力分散时制造问题" |

**匹配优先级优化：**
```python
# 认知灵活性不足（规则僵化）- 优先级最高
if "规则" in behavior or "僵化" in behavior or "重复" in behavior:
    return "就像一台只能播放三首歌的收音机..."

# 社交信号监测弱 - 次优先级
elif "以为" in behavior or "监测" in behavior:
    return "就像一位沉浸在自己剧本里的导演..."
```

## 五、核心教训

1. **摘要一致性是专业信任的基石**
   - 临床系统中，"分析正确但摘要错误"比"分析一般但谨慎"风险更高
   - 摘要必须与后文分析严格一致，这是 P0 级要求

2. **家长需要"理解框架"而非"结论"**
   - 比喻不是装饰，而是帮助家长建立心理模型的认知工具
   - "沉浸在自己剧本里的导演"比"社交技能不足"更能帮助家长理解

3. **临床推理过程必须可视化**
   - 展示"如何排除其他可能性"比"给出结论"更有教育价值
   - 鉴别诊断三模块结构是专业性的体现
