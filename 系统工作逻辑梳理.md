## 第一步：ABC 信息收集

### 用户如何输入

**用户输入类型：**
1. **完整 ABC 描述**（理想）
   ```
   情境（A）：在幼儿园集体课上
   行为（B）：孩子不停玩衣服，不看老师
   结果（C）：老师提醒多次，效果不明显
   ```

2. **行为片段**（常见）
   ```
   孩子今天不想吃饭
   ```

3. **复杂场景描述**（最佳）
   ```
   老师好，想跟您分享一下 OK 在家的小趣事...
   （200 字详细描述，包含 A、B、C 信息）
   ```

### 系统如何提取 ABC

**提取逻辑（优先级顺序）：**

```python
def extract_abc(user_input):
    # 1. LLM 智能提取（主要方式）
    extracted = llm_extract(user_input)
    
    # 2. 规则匹配 fallback（补充）
    rule_extracted = rule_based_extract(user_input)
    
    # 3. 上下文推断（增强）
    context_extracted = extract_from_context(user_input)
    
    # 4. 合并结果
    abc = merge(extracted, rule_extracted, context_extracted)
    
    return abc
```

**提取的字段：**
- `antecedent`（前因/情境）
- `behavior_detail`（行为细节）
- `immediate_consequence`（即时结果）
- `environment`（环境：家庭/幼儿园/学校）
- `child_age`（孩子年龄）

### 当前问题与优化

**问题：**
- 长文本提取不完整（部分字段为空）
- 环境推断错误（显示"幼儿园"但实际是"在家"）

**优化方案：**
- 增强规则提取 fallback
- 添加环境推断关键词匹配
- 调试日志确认提取结果

## 第二步：场景识别与假设生成

### 场景识别逻辑

**系统如何判断问题类型：**

```python
def identify_scenario(user_input):
    # 1. 提取关键词
    keywords = extract_keywords(user_input)
    # 如：舞蹈、游泳、美术 → 集体学习场景
    # 如：薯片盒、妈妈看到什么 → 观点采择场景
    # 如：介绍、名字、说话 → 对话/介绍场景
    
    # 2. 匹配场景类型
    if "舞蹈" in keywords or "集体" in keywords:
        return "集体学习场景"
    elif "以为" in keywords or "看到什么" in keywords:
        return "观点采择场景"
    elif "介绍" in keywords or "名字" in keywords:
        return "对话/介绍场景"
    elif "规则" in keywords or "必须" in keywords:
        return "规则变化场景"
    else:
        return "通用场景"
```

**5 大场景类型：**

| 场景类型 | 核心挑战 | 典型表现 |
|----------|----------|----------|
| 集体学习 | 视觉信息筛选 | 1v1 好，集体课不集中 |
| 观点采择 | 视角切换 | 以为别人知道自己知道的 |
| 社交互动 | 双向意识 | 说完就好，不等回应 |
| 认知灵活性 | 规则变化适应 | 坚持同一路线/程序 |
| 情绪识别 | 非语言线索解读 | 看不出别人不高兴 |

### 假设生成逻辑

**系统如何生成竞争性假设：**

```python
def generate_hypotheses(scenario_type):
    # 根据场景类型，生成 2-3 个竞争性假设
    
    if scenario_type == "观点采择场景":
        return [
            "H1: 心理理论发展延迟（无法表征他人心理状态）",
            "H2: 工作记忆/抑制控制短板（记不住关键信息）",
            "H3: 语言理解偏差（将'看到'理解为'知道'）"
        ]
    
    elif scenario_type == "对话/介绍场景":
        return [
            "H1: 社交技能缺失（不懂互动规则）",
            "H2: 程序化理解（将互动理解为单向任务）",
            "H3: 认知超载（认知资源被占用，无暇监测他人）"
        ]
```

**核心原则：**
- ❌ 不预设单一答案
- ✅ 生成 2-3 个竞争性假设
- ✅ 每个假设都针对具体场景

## 第三步：临床推理与鉴别

### 推理逻辑（核心）

**系统如何像专家一样思考：**

```python
def clinical_reasoning(user_input, hypotheses):
    # 1. 为每个假设寻找支持/反对证据
    for hypothesis in hypotheses:
        supporting_evidence = find_supporting_evidence(user_input, hypothesis)
        opposing_evidence = find_opposing_evidence(user_input, hypothesis)
        
        # 2. 评估证据强度
        confidence = calculate_confidence(supporting_evidence, opposing_evidence)
        
        # 3. 排除证据最弱的假设
        if confidence < threshold:
            exclude_hypothesis(hypothesis)
    
    # 4. 保留证据最强的 1-2 个假设
    core_challenge = select_best_hypothesis(hypotheses)
    
    return core_challenge
```

### 推理示例（薯片盒案例）

**用户输入：**
```
OK 可以准确说出盒子里是糖果，但问她"妈妈会看到什么"时，
她以为我看到盒子里还是糖，不太理解我看到的是薯片的意思。
```

**推理过程：**

```
假设 H1：心理理论发展延迟
- 支持证据：她以为妈妈看到的是糖（无法推断他人信念）
- 反对证据：无
- 置信度：高 ✅

假设 H2：工作记忆短板
- 支持证据：需要同时记住 3 个信息（真相、关键信息、他人信念）
- 反对证据：她能准确说出盒子里是糖果（记忆力正常）
- 置信度：中 ⚠️

假设 H3：语言理解偏差
- 支持证据：无
- 反对证据：她能理解"看到"的字面意思
- 置信度：低 ❌

结论：H1（心理理论发展延迟）证据最强
```

## 第四步：干预建议生成

### 多段式设计逻辑

**你提到的"多段式"设计：**

```
能回答的我们就回答
    ↓
不能回答的我们先提示
    ↓
知识库里没有对应的东西
    ↓
回归到大模型通用推理
```

**实际实现逻辑：**

```python
def generate_intervention(core_challenge):
    # 1. 检查知识库是否有匹配场景
    if scenario_in_knowledge_base(core_challenge):
        # 能回答的：直接返回知识库干预
        return knowledge_base_intervention(core_challenge)
    
    # 2. 知识库没有，但有场景映射
    elif scenario_has_mapping(core_challenge):
        # 不能回答的：使用场景映射生成
        return scene_mapped_intervention(core_challenge)
    
    # 3. 完全新场景，使用大模型推理
    else:
        # 回归通用推理
        return llm_reasoned_intervention(core_challenge)
```

### 场景 - 干预映射

**5 类场景的干预映射：**

| 场景类型 | 匹配干预 | 核心思路 |
|----------|----------|----------|
| 共同游戏 | "开始信号"游戏 | 将抽象的"是否在一起玩"转化为具体动作信号 |
| 对话/介绍 | "观察 - 反应"游戏 | 将"说完话要看对方反应"转化为可练习步骤 |
| 规则变化 | "预告 + 小变化"游戏 | 通过预告增加可预测性，通过微小变化培养灵活性 |
| 身体边界 | "身体地图"游戏 | 将抽象的"身体边界"转化为可视化概念 |
| 情绪识别 | "情绪侦探"游戏 | 将情绪识别转化为可练习的"侦探游戏" |
