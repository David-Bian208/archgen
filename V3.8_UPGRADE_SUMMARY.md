## 核心改进

### 1️⃣ 彻底修复 HTML 实体乱码

**问题**: 报告中出现 `&#039;`、`&ge;` 等 HTML 实体乱码

**后端修复** (`app/agents/intervention_planner.py`):
```python
import html

def decode_html_entities(self, text: str) -> str:
    """V3.8 HTML 实体解码 - 使用 html.unescape()"""
    if not text:
        return ""
    return html.unescape(text)

def sanitize_plan(self, plan: dict) -> dict:
    """对计划字典中的所有文本字段进行 HTML 实体解码"""
    if "four_step_plan" in plan:
        for key, value in plan["four_step_plan"].items():
            if isinstance(value, str):
                plan["four_step_plan"][key] = self.decode_html_entities(value)
    return plan
```

**前端加固** (`frontend/utils/filters.js`):
```javascript
// V3.8 新增：HTML 实体解码函数（前端双保险）
export function decodeHTMLEntities(text) {
  const entities = {
    '&amp;': '&',
    '&lt;': '<',
    '&gt;': '>',
    '&quot;': '"',
    '&#39;': "'",
    '&ge;': '≥',
    '&le;': '≤',
  }
  let result = String(text)
  for (const [entity, char] of Object.entries(entities)) {
    result = result.replace(new RegExp(entity, 'g'), char)
  }
  return result
}
```

**效果**: 特殊字符（%, ≥, ", '）在任何位置均正确显示，无乱码。

### 3️⃣ 重构第四部分为通用方法论

**核心要求**: 避免仅适用于当前案例的具体名词（如"大鹏展翅"）和情境（如"下雨的教室"）。

**V3.8 通用四步结构**:

```
四、我们可以这样开始

1. 💡 核心思路
针对孩子"需要看着老师才能记住动作"的情况，我们将帮助他把需要紧盯的"外部提示"，转化为他自己能使用的"内部密语"。就像给他装一个"动作小闹钟"，不需要看老师，自己也能想起来。

2. 🎮 我们的计划：动作密语游戏
我们可以从当前的任务情境中，选择一个有代表性的步骤，将它转化为您和孩子之间的趣味"暗号"。通过精心设计的小游戏，在轻松的氛围中反复练习，帮助孩子建立"暗号"与"行动"之间的快速链接。

3.  成功的画面
在 3 天内，当您使用这个"暗号"提示时，孩子能在 3 秒内做出正确反应，且尝试的成功率能够稳定达到 80% 以上。

4. 🚀 第一步行动
今天，您就可以尝试和孩子一起，为那个选定的步骤起一个有趣的名字，并像玩闯关游戏一样试一次。记住，第一次玩不重要，重要的是开始！
```

**关键改进**:
- ✅ 避免具体动作名称（如"大鹏展翅"）
- ✅ 避免具体情境（如"做操"、"下雨的教室"）
- ✅ 使用通用方法论描述（"动作密语游戏"、"任务情境"）
- ✅ 增强普适性和可复用性
