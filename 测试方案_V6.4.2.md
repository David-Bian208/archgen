## 测试用例

### 测试 1：ABC 智能引导 - B 中包含 C 信息

**目的：** 验证系统在 B 回答中识别到 C 信息时，不再机械追问

**输入：**
```
在一起上课的时候，他分不清严厉还是凶。然后会很生气，我一般两种处理方式，第一种是我退让一步，然后他会寻求安慰，想希望我安慰她，还有一种方式，我不退让，他就会生气发脾气，但发脾气的过程当中会克制，说我不想当好孩子啦，我想发脾气啦
```

**预期结果：**
- 系统识别到 B 中包含"处理方式"、"退让"、"安慰"、"发脾气"等关键词（≥2个）
- 系统直接进入推理，不再追问 C
- 生成完整的行为分析报告

**通过标准：** 系统不再重复追问"您当时是怎么处理的？最后怎么样了？"

---

### 测试 2：完整信息一次性提交

**目的：** 验证系统能处理一次性提供完整 ABC 信息的场景

**输入：**
```
昨天在超市，我看到一个玩具，我说不能买，他就开始大哭，哭得很大声，我在旁边劝他冷静，后来他慢慢停止了哭泣
```

**预期结果：**
- 系统识别到一次性完整输入（长度 > 80）
- 直接进入推理，不启动 ABC 引导流程
- 生成完整报告

**通过标准：** 直接生成报告，无引导对话

---

### 测试 3：标准 ABC 引导流程

**目的：** 验证标准的 A→B→C 引导流程正常工作

**步骤：**
1. 输入：`玥玥最近很暴躁`
2. 系统应启动 ABC 引导，先问 A
3. 输入 A：`昨天在画画课上`
4. 系统确认后问 B
5. 输入 B：`她把画笔都扔了`
6. 系统确认后问 C
7. 输入 C：`我让她道歉，她哭了`

**预期结果：**
- 系统按 A→B→C 顺序引导
- 每步确认后进入下一步
- C 收集完成后自动生成报告

**通过标准：** 引导流程完整，最终生成报告

---

### 测试 4：中台数据记录

**目的：** 验证所有问答数据被正确记录到中台

**步骤：**
1. 在 3000 端口完成测试 1 或测试 2
2. 打开 http://localhost:8000
3. 刷新页面

**预期结果：**
- 统计数据更新（总对话数 +1）
- 对话列表中出现新的记录
- 可以看到用户问题和助手回答
- 场景类型正确（A/B/C/D/E）
- 响应时间显示正常

**通过标准：** 中台显示完整的对话记录

**预期结果：**

- 场景筛选正常工作

**通过标准：** 搜索和筛选结果准确

### 测试 6：中台导出功能

**预期结果：**

**通过标准：** CSV 文件可正常打开且数据完整
