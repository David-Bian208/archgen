## 项目概述

**定位：** 文章转架构图的工业化流水线，核心理念"先逻辑，后视觉"

**核心架构（四步流水线）：**
1. **Router Agent** - 文章分类（主张/因果/系统/对比/流程 5 种类型）
2. **Extractor Agent** - 结构拆解（LLM 按 Schema 提取结构化 JSON）
3. **HTML Generator** - 视觉渲染（Jinja2 模板 + CSS 样式）
4. **Screenshot** - 截图出图（Playwright 无头浏览器）

**项目路径：** `/home/admin/.openclaw/workspace/behavior_recorder_service/archgen/`

## 常见问题排查

### 问题 1: 按钮点击无效

**排查流程：**
1. 打开浏览器控制台（F12）
2. 点击问题按钮
3. 观察是否有 🔴 [DEBUG] 日志
4. 观察是否有错误信息
5. 反馈截图给开发者

**常见根因：**
- `<a-spin>` 遮罩层覆盖
- v-if 条件不满足（按钮不在 DOM 中）
- disabled 状态卡住
- 事件被阻止（stopPropagation）

### 问题 2: 页面切换时报错

**错误信息：** `Cannot read properties of null (reading 'type')`

**根因：** Arco Design `<a-spin>` 与 Vue 响应式系统冲突

**修复方案：** 见 `frontend/重构方案.md`

### 问题 3: 进度条不显示当前阶段

**排查：** 检查 `currentStep` 值是否正确

**调试：**
```javascript
console.log('currentStep:', currentStep.value)
```

## 代码规范

### 前端规范

#### Loading 状态管理（重要）

所有涉及 `loading.value` + `currentStep` 切换的异步函数，必须遵循：

```javascript
async function someStep() {
  loading.value = true
  try {
    const res = await someApi()
    
    if (res.data.code === 0) {
      data.value = res.data.data
      
      // ✅ 先关闭 loading
      loading.value = false
      
      // ✅ 延迟切换 step（给 Arco 100ms）
      setTimeout(() => {
        currentStep.value = N
        Message.success('完成')
      }, 100)
    } else {
      loading.value = false
      Message.error(res.data.msg)
    }
  } catch (e) {
    loading.value = false
    Message.error(e.message)
  }
  // ❌ 不要用 finally
}
```

#### 调试日志规范

```javascript
// 🔴 调试日志（修复后保留）
console.log('🔴 [DEBUG] someFunction 被调用')
console.log('🔴 当前状态:', {
  currentStep: currentStep.value,
  loading: loading.value,
})
```

### 后端规范

- 严格遵循 PEP8 规范
- 所有函数必须加类型提示
- 核心逻辑必须加注释
